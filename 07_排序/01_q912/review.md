# LeetCode 912. 排序数组

## 题目理解

给你一个整数数组 `nums`，要求按升序排序后返回。

例如：

```text
nums = [5, 2, 3, 1]
```

返回：

```text
[1, 2, 3, 5]
```

这题表面是“把数组排好序”，但真正适合拿来做一件事：

```text
借 LeetCode 912 系统梳理排序算法大全
```

不过要注意，排序算法很多，面试优先级并不一样。

有些需要能手写，有些需要能解释，有些只需要知道它存在和适用场景。

---

## 排序算法全景图

常见排序算法可以分成三大类。

### 第一类：基础 O(n^2) 排序

这类排序主要用来理解“排序”这件事最朴素的过程。

```text
选择排序
插入排序
冒泡排序
```

它们的时间复杂度通常是：

```text
O(n^2)
```

不适合作为 LeetCode 912 的最终解法，但适合作为排序入门。

其中最值得掌握的是：

```text
插入排序
```

因为很多工业级排序算法会在小数组或局部有序数组上使用插入排序思想。

---

### 第二类：经典 O(n log n) 比较排序

这类是排序面试的核心。

```text
归并排序
快速排序
堆排序
```

它们都基于元素之间的比较。

这三个需要重点掌握。

```text
归并排序：稳定，最坏 O(n log n)，需要 O(n) 空间
快速排序：平均 O(n log n)，原地分区，但最坏 O(n^2)
堆排序：最坏 O(n log n)，O(1) 额外空间，但不稳定
```

如果面试官问 LeetCode 912，最稳的主解法是：

```text
归并排序
```

如果面试官要求原地排序，可以写：

```text
堆排序
```

如果面试官想考 partition 思想，可以写：

```text
随机三路快速排序
```

---

### 第三类：非比较排序

这类排序不依赖元素两两比较，而是利用数字范围、桶、位数等信息。

```text
计数排序
桶排序
基数排序
```

它们在特定条件下可以做到接近：

```text
O(n)
```

但都有额外前提。

```text
计数排序：适合数值范围不大的整数
桶排序：适合能合理分桶的数据
基数排序：适合整数或字符串，按位处理
```

其中 `桶排序` 会在后面的：

```text
LeetCode 164. Maximum Gap
```

里更自然地展开。

## 方法一：选择排序

### 思路

选择排序每轮从未排序部分里选出最小值，放到当前位置。

例如：

```text
[5, 2, 3, 1]
```

第 1 轮找出最小值 `1`，和第一个位置交换：

```text
[1, 2, 3, 5]
```

后续继续处理剩下的部分。

### 特点

```text
时间复杂度：O(n^2)
空间复杂度：O(1)
稳定性：不稳定
```

选择排序交换次数少，但比较次数固定很多。

### 代码

```text

```

### 面试定位

只适合理解排序基本过程，不推荐作为最终解法。

## 方法二：插入排序

### 思路

插入排序维护一个已经有序的前缀。

每次取当前元素，把它插入到前面有序区间的正确位置。

例如：

```text
[5, 2, 3, 1]
```

先看前两个：

```text
[5] 已经有序
```

把 `2` 插入到 `[5]` 前面：

```text
[2, 5, 3, 1]
```

再把 `3` 插入到 `[2, 5]` 中间：

```text
[2, 3, 5, 1]
```

最后把 `1` 插入到最前面：

```text
[1, 2, 3, 5]
```

### 特点

```text
时间复杂度：O(n^2)
最好情况：O(n)
空间复杂度：O(1)
稳定性：稳定
```

如果数组本来接近有序，插入排序会非常快。

### 代码

```text
07_排序/01_q912/solution_2_insertion_sort.py
```

### 面试定位

插入排序值得掌握。

它不是 912 的最终推荐解法，但它是排序基础里最有价值的 `O(n^2)` 排序。

## 方法三：冒泡排序

### 思路

冒泡排序不断比较相邻元素。

如果前一个比后一个大，就交换。

每一轮会把当前未排序区间中的最大值“冒泡”到末尾。

### 特点

```text
时间复杂度：O(n^2)
最好情况：O(n)，需要提前退出优化
空间复杂度：O(1)
稳定性：稳定
```

## 方法四：归并排序

### 思路

归并排序是分治排序。

核心步骤：

```text
1. 把数组拆成左右两半
2. 分别排序左右两半
3. 合并两个有序数组
```

例如：

```text
[5, 2, 3, 1]
```

拆分：

```text
[5, 2] 和 [3, 1]
```

继续拆：

```text
[5] [2] [3] [1]
```

合并：

```text
[5] + [2] -> [2, 5]
[3] + [1] -> [1, 3]
[2, 5] + [1, 3] -> [1, 2, 3, 5]
```

### 为什么是 O(n log n)

每一层都要合并所有元素一次，所以每层是：

```text
O(n)
```

数组每次减半，所以一共有：

```text
O(log n)
```

层。

总复杂度：

```text
O(n log n)
```

### 特点

```text
时间复杂度：O(n log n)
空间复杂度：O(n)
稳定性：稳定
```

### 代码

```text
07_排序/01_q912/solution_4_merge_sort.py
```

### 面试定位

这是 LeetCode 912 最推荐的面试写法。

原因：

```text
1. 最坏时间复杂度稳定是 O(n log n)
2. 不怕重复元素
3. 思路清晰，容易解释
4. 代码比堆排序更容易写对
5. 比快速排序更不容易被极端数据卡住
```

## 方法五：随机三路快速排序

### 思路

快速排序每次选一个基准值 `pivot`，把数组分成：

```text
小于 pivot
等于 pivot
大于 pivot
```

然后递归排序小于区间和大于区间。

三路分区适合处理大量重复元素。

例如：

```text
[2, 2, 2, 2, 2]
```

普通二路快排可能会反复处理重复元素。

三路快排会一次把所有等于 `pivot` 的元素放到中间，后续不再处理。

### 特点

```text
平均时间复杂度：O(n log n)
最坏时间复杂度：O(n^2)
空间复杂度：O(log n)
稳定性：不稳定
```

使用随机 `pivot` 可以显著降低遇到最坏情况的概率。

### 代码

如果面试官想考分区思想，优先写随机三路快排。

但如果只是问 912 的稳妥解法，归并排序更稳。

## 方法六：堆排序

### 思路

堆排序使用最大堆。

最大堆满足：

```text
父节点 >= 子节点
```

所以堆顶一定是当前最大值。

排序过程：

```text
1. 把数组建成最大堆
2. 把堆顶最大值和末尾交换
3. 缩小堆范围
4. 对新的堆顶做下沉
5. 重复直到排序完成
```

### 特点

```text
时间复杂度：O(n log n)
空间复杂度：O(1)
稳定性：不稳定
```

### 代码

```text
07_排序/01_q912/solution_6_heap_sort.py
```


时，堆排序是好选择。

但它的代码细节比归并排序更容易写错。

## 方法七：希尔排序

### 思路

希尔排序是插入排序的改进版。

它先按较大的间隔 `gap` 分组做插入排序，再逐渐缩小 `gap`。

最后当：

```text
gap = 1
```

时，就退化成普通插入排序。

但此时数组已经比较接近有序，所以插入排序会更快。

### 特点

```text
时间复杂度：取决于 gap 序列
空间复杂度：O(1)
稳定性：不稳定
```

## 方法八：计数排序

### 思路

计数排序不比较元素大小，而是统计每个数字出现了多少次。

例如：

```text
[5, 1, 1, 2, 0, 0]
```

统计：

```text
0 出现 2 次
1 出现 2 次
2 出现 1 次
5 出现 1 次
```

然后按数字从小到大还原：

```text
[0, 0, 1, 1, 2, 5]
```

### 特点

设数字范围大小是 `k`。

```text
时间复杂度：O(n + k)
空间复杂度：O(k)
稳定性：可以稳定，也可以不稳定
```

如果数字范围很大，计数排序会浪费大量空间。

### 代码


### 面试定位

它不是通用排序，但在数字范围较小的时候非常强。

## 方法九：基数排序

### 思路

基数排序按位排序。

以十进制整数为例：

```text
先按个位排序
再按十位排序
再按百位排序
...
```

每一位都使用稳定的分配过程。

对于负数，可以把负数部分单独处理：

```text
负数取相反数排序，再反向还原
非负数正常排序
```

### 特点

设最大数字有 `d` 位。

```text
时间复杂度：O(d * n)
空间复杂度：O(n)
稳定性：稳定
```

### 代码

### 面试定位

知道原理和适用场景即可。

它适合整数或固定长度字符串，不是通用比较排序。
