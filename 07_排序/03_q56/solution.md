# LeetCode 56 — 合并区间（Merge Intervals）

## 题目

以数组 `intervals` 表示若干个区间的集合，其中单个区间为 `intervals[i] = [start_i, end_i]` 。请合并所有重叠的区间，并返回一个不重叠的区间数组，该数组需恰好覆盖输入中的所有区间。

### 示例

```
输入: intervals = [[1,3],[2,6],[8,10],[15,18]]
输出: [[1,6],[8,10],[15,18]]
解释: 区间 [1,3] 和 [2,6] 重叠，合并为 [1,6]
```

```
输入: intervals = [[1,4],[4,5]]
输出: [[1,5]]
解释: 端点相接也算重叠
```

---

## 解法一览

| 解法 | 时间 | 空间 | 是否推荐面试 |
|---|---|---|---|
| 1. 暴力两两合并 | O(n²) | O(n) | 否 |
| 2. 排序 + 一次扫描 | O(n log n) | O(n) | **最佳 ✅** |

---

## 解法 1：暴力两两合并

**思路：** 不排序，每次遍历找所有重叠区间对并合并，反复直到没有可合并的。

每次遍历 O(n²)，最坏需要 n 轮。

**缺点：** 时间 O(n²)，面试中只作为铺垫。

---

## 解法 2：排序 + 一次扫描 ⭐ 面试推荐

**思路：**
1. 按区间**起点**升序排序
2. 从左到右扫描，维护当前合并区间 `[merged_start, merged_end]`
3. 遇到下一个区间 `[s, e]`：
   - 如果 `s <= merged_end` → 重叠，合并：`merged_end = max(merged_end, e)`
   - 否则 → 不重叠，把当前区间加入结果，更新当前区间

### 核心逻辑

排序后，重叠区间一定相邻，只需比较当前区间和已合并区间的右端点：

```
排序后: [1,3] [2,6] [8,10] [15,18]
         ↑     ↑
      当前    下一个

2 <= 3 → 重叠，合并为 [1, max(3,6)] = [1,6]
8 >  6 → 不重叠，[1,6] 入结果，当前更新为 [8,10]
15 > 10 → 不重叠，[8,10] 入结果，当前更新为 [15,18]
```

**复杂度：** 排序 O(n log n) + 扫描 O(n) = 总时间 O(n log n)；结果数组 O(n)。

---

## 面试建议

**解法 2（排序 + 一次扫描）是面试标准答案。**

面试时这样展开：
1. 先提一句"不排序的话需要反复两两合并，O(n²)"
2. 立刻转到排序思路："按起点排序后，重叠区间一定相邻"
3. 写出一次扫描的合并逻辑
4. 如果面试官追问边界情况，提到端点相接也算重叠（`s <= merged_end` 的 `=` 号）
