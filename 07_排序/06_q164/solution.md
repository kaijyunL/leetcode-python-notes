# LeetCode 164 — 最大间距（Maximum Gap）

## 题目

给定一个无序数组 `nums`，返回数组在排序后，**相邻元素之间的最大差值**。如果数组元素个数小于 2，则返回 0。

**要求：** 时间复杂度 O(n)，空间复杂度 O(n)。

### 示例

```
输入: nums = [3, 6, 9, 1]
输出: 3
解释: 排序后为 [1, 3, 6, 9]，相邻差值为 2, 3, 3，最大为 3
```

```
输入: nums = [10]
输出: 0
解释: 元素个数小于 2，返回 0
```

---

## 解法一览

| 解法 | 时间 | 空间 | 是否推荐面试 |
|---|---|---|---|
| 1. 排序后扫描 | O(n log n) | O(1) | 可接受，但不满足进阶要求 |
| 2. 桶排序 + 鸽笼原理 | O(n) | O(n) | **最佳 ✅** |

---

## 解法 1：排序后扫描

**思路：** 先对数组排序，然后遍历相邻元素求差值取最大值。

**缺点：** 时间 O(n log n)，不满足题目要求的 O(n) 时间复杂度。面试中可以作为第一步，但需继续优化。

---

## 解法 2：桶排序 + 鸽笼原理 ⭐ 面试推荐

这是本题的难点所在。题目明确要求 O(n) 时间，意味着不能用比较排序。

### 核心思路

利用**鸽笼原理**（抽屉原理）：

- 设数组有 n 个元素，最小值为 `minVal`，最大值为 `maxVal`
- 排序后有 n-1 个相邻差值
- **最大间距一定 ≥ ceil((maxVal - minVal) / (n - 1))**

把 `[minVal, maxVal]` 这个区间分成 `n-1` 个桶，每个桶的宽度为 `bucketSize = ceil((maxVal - minVal) / (n - 1))`。

### 关键推论

**同一个桶内的元素，差值一定小于 `bucketSize`，不可能成为最大间距。**

因为桶的宽度就是平均间距，桶内最大差值最多是 `bucketSize - 1`。所以最大间距只可能出现在**相邻桶之间**（前一个桶的最大值和后一个桶的最小值之间）。

### 算法步骤

1. 找 `minVal` 和 `maxVal`
2. 计算桶宽度 `bucketSize = ceil((maxVal - minVal) / (n - 1))`
3. 创建 n 个桶，每个桶只记录最小值和最大值
4. 把每个元素放入对应桶：`index = (num - minVal) // bucketSize`
5. 遍历所有桶，用**前一个非空桶的最大值**和**当前非空桶的最小值**求差值，取最大

### 逐步演示

```
nums = [3, 6, 9, 1]
minVal = 1, maxVal = 9, n = 4
bucketSize = ceil((9-1)/3) = 3

创建 4 个桶（编号 0~3），每个桶宽 3：
桶 0: [1, 3]  → 放 1, 3 → min=1, max=3
桶 1: (4, 6]  → 放 6     → min=6, max=6
桶 2: (7, 9]  → 放 9     → min=9, max=9
桶 3: 放 maxVal 本身

空桶：无

遍历相邻非空桶：
桶0.max=3 与 桶1.min=6 → 差值 3
桶1.max=6 与 桶2.min=9 → 差值 3

最大间距 = 3
```

### 为什么这个方法是 O(n)？

- 找 min/max：O(n)
- 分配元素入桶：O(n)，每个元素入桶 O(1)
- 遍历桶：O(n)，桶的数量等于 n

总时间 O(n)，总空间 O(n)。

## 面试建议

**解法 2（桶排序）是面试标准答案。**

面试时这样展开：
1. 先写排序法："排序后扫描，O(n log n)，但不满足进阶要求"
2. 引出桶排序："题目要求 O(n)，联想到用桶排序/鸽笼原理"
3. 重点讲清核心推论："同一个桶内不可能出现最大间距，只需比较相邻桶"
4. 写出桶的分配公式和遍历逻辑
5. 主动提到边界：空桶要跳过、n < 2 直接返回 0
