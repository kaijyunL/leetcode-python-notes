# LeetCode 179 — 最大数（Largest Number）

## 题目

给定一组非负整数 `nums`，重新排列每个数的顺序（每个数不可拆分），使之组成一个最大的整数。返回一个字符串。

### 示例

```
输入: nums = [10, 2]
输出: "210"
```

```
输入: nums = [3, 30, 34, 5, 9]
输出: "9534330"
```

```
输入: nums = [0, 0]
输出: "0"
```

---

## 解法一览

| 解法 | 时间 | 空间 | 是否推荐面试 |
|---|---|---|---|
| 1. 全排列枚举 | O(n!) | O(n) | 否 |
| 2. 自定义排序比较 | O(n log n) | O(n) | **最佳 ✅** |

---

## 解法 1：全排列枚举

**思路：** 生成 `nums` 的所有排列，每个排列拼接成字符串，取最大的那个。

**缺点：** O(n!) 复杂度，n=10 就已经不可接受，仅作为暴力思路展示。

---

## 解法 2：自定义排序比较 ⭐ 面试推荐

**核心思想：** 对于两个数字的字符串形式 `a` 和 `b`，比较 `a + b` 和 `b + a`：
- 如果 `a + b > b + a` → `a` 应该排在 `b` 前面
- 否则 `b` 排在 `a` 前面

例如 `a = "3"`, `b = "30"`：
- `a + b = "330"`, `b + a = "303"`
- `"330" > "303"` → 所以 3 排在 30 前面

### 为什么这个比较规则有效？

这本质上是在回答一个问题：**两个数谁放在前面能产生更大的结果？**

把 `+` 理解为字符串拼接，我们关心的是拼接后哪个更大。字符串比较时长度相同，逐字符比较，所以 `"330" > "303"` 意味着 3 放前面更好。

这个比较规则满足**传递性**（如果 a 优于 b，b 优于 c，则 a 优于 c），因此可以用来排序。

### 逐步演示

```
nums = [3, 30, 34, 5, 9]
转为字符串: ["3", "30", "34", "5", "9"]

按自定义规则排序（降序）:
"9" vs "5"  → "95" > "59" → 9 在前
"5" vs "34" → "534" > "345" → 5 在前
"34" vs "3" → "343" > "334" → 34 在前
"3" vs "30" → "330" > "303" → 3 在前

排序结果: ["9", "5", "34", "3", "30"]
拼接: "9534330"
```

### 特殊处理：全 0 情况

如果排序后最大的元素是 `"0"`，说明全是 0，直接返回 `"0"` 而不是 `"000"`。

**复杂度：** 时间 O(n log n × k)，其中 k 是数字字符串的最大长度；空间 O(n) 存储字符串列表和排序结果。

## 面试建议

**解法 2（自定义排序）是面试标准答案。**

面试时这样展开：
1. 先说暴力思路："全排列所有组合取最大，但 O(n!) 不可行"
2. 引出核心问题："两个数谁放前面更好？比较 a+b 和 b+a"
3. 写出自定义排序逻辑
4. 主动提到边界情况：`[0, 0]` → 返回 `"0"` 而非 `"000"`
5. 如果面试官追问，可以解释为什么这个比较规则具有传递性（可证明）
