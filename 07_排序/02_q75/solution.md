# LeetCode 75 — 颜色分类（Sort Colors）

## 题目

给定一个包含红色、白色和蓝色，一共 `n` 个元素的数组，**原地**对它们进行排序，使得相同颜色的元素相邻，并按照红色、白色、蓝色顺序排列。使用整数 `0`、`1`、`2` 分别表示红色、白色和蓝色。

**要求：不能使用代码库中的排序函数。**

### 示例

```
输入: nums = [2, 0, 2, 1, 1, 0]
输出: [0, 0, 1, 1, 2, 2]
```

```
输入: nums = [2, 0, 1]
输出: [0, 1, 2]
```

---

## 解法一览

| 解法 | 时间 | 空间 | 是否推荐面试 |
|---|---|---|---|
| 1. 内置排序 | O(n log n) | O(1) | 否 |
| 2. 两次计数填充 | O(n) | O(1) | 可作为过渡思路 |
| 3. 一趟三指针（荷兰国旗） | O(n) | O(1) | **最佳 ✅** |

---

## 解法 1：内置排序

最简单但最不符合题意的方法：直接调用 `sort()`。

**缺点：** 题目明确要求不能使用内置排序，面试中直接写会扣分。

```python
nums.sort()
```

> 时间复杂度 O(n log n)，空间 O(1)。仅作为 baseline 对比。

---

## 解法 2：两次计数填充

**思路：**
1. 第一遍遍历，统计 0、1、2 各自出现次数
2. 第二遍按统计结果重写数组

```
[2, 0, 2, 1, 1, 0]
  ↓ 计数
  cnt[0] = 2, cnt[1] = 2, cnt[2] = 2
  ↓ 重写
[0, 0, 1, 1, 2, 2]
```

**复杂度：** 时间 O(n)，需要两趟扫描；空间 O(1)。

---

## 解法 3：一趟三指针 — 荷兰国旗问题 ⭐ 面试推荐

这是本题的最优解，也是面试官期待的答案。

### 核心思想

维护三个指针，把数组划分为四个区域：

```
[0, p0)     → 全是 0（红色区）
[p0, i)     → 全是 1（白色区）
[i, p2]     → 待处理（未知区）
(p2, n-1]   → 全是 2（蓝色区）
```

### 指针含义

| 指针 | 含义 |
|---|---|
| `p0` | 下一个 0 应该放置的位置（从左开始） |
| `i` | 当前正在检查的位置 |
| `p2` | 下一个 2 应该放置的位置（从右开始） |

### 算法流程

从左到右扫描，`i` 从 0 走到 `p2`，循环条件 `i <= p2`：

```
while i <= p2:
    如果 nums[i] == 0:
        与 p0 位置交换，p0 右移，i 右移
        （因为 p0 位置换过来的只可能是 0 或 1，都已就位）

    如果 nums[i] == 2:
        与 p2 位置交换，p2 左移，i 不动
        （因为 p2 位置换过来的可能是 0/1/2，需要再检查）

    如果 nums[i] == 1:
        不用交换，i 直接右移
```

### 关键区别：为什么 `< pivot` 时 `i++`，`> pivot` 时 `i` 不动？

**交换 0（和 p0 换）后 `i++`：** p0 位置的元素要么来自已经检查过的 1，要么来自 i 自身（p0 == i），换过来的一定是已就位的 0 或 1。

**交换 2（和 p2 换）后 `i` 不动：** p2 来自右端未处理区域，换过来可能是 0/1/2 中任何一个，必须在下一轮重新检查。

### 逐步演示

```
初始: [2, 0, 2, 1, 1, 0]
      p0=0, i=0, p2=5

i=0, nums[0]=2 > 1 → 和 p2=5 交换, p2=4, i不动
[0, 0, 2, 1, 1, 2]
       ↑        ↑
       i        p2

i=0, nums[0]=0 == 0 → 和 p0=0 交换, p0=1, i=1
（原地交换，不变）

i=1, nums[1]=0 == 0 → 和 p0=1 交换, p0=2, i=2
（原地交换，不变）

i=2, nums[2]=2 > 1 → 和 p2=4 交换, p2=3, i不动
[0, 0, 1, 1, 2, 2]
           ↑     ↑
           i     p2

i=2, nums[2]=1 == 1 → i=3

i=3, nums[3]=1 == 1 → i=4
此时 i=4 > p2=3，循环结束 ✓

结果: [0, 0, 1, 1, 2, 2]
```

**复杂度：** 时间 O(n)，一趟扫描；空间 O(1)。

## 面试建议

**解法 3（三指针）是唯一推荐在面试中给出的答案。**

面试时可以这样展开：
1. 先说解法 2（两次计数）作为基础思路，展示你有分析能力
2. 再提出解法 3（一趟三指针），说明优化点：把两趟合并为一趟
3. 重点讲清楚为什么 `nums[i] == 2` 时 `i` 不前进——这是面试官最常追问的点
