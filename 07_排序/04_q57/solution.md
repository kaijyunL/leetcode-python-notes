# LeetCode 57 — 插入区间（Insert Interval）

## 题目

给你一个 **无重叠** 的区间列表 `intervals`，其中 `intervals[i] = [start_i, end_i]` 按起始端点**已升序排列**。另给你一个新区间 `newInterval = [start, end]`，请将 `newInterval` 插入到 `intervals` 中，使得插入后仍然有序且无重叠区间（必要时需要合并）。

### 示例

```
输入: intervals = [[1,3],[6,9]], newInterval = [2,5]
输出: [[1,5],[6,9]]
解释: [2,5] 与 [1,3] 重叠，合并为 [1,5]
```

```
输入: intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]], newInterval = [4,8]
输出: [[1,2],[3,10],[12,16]]
解释: [4,8] 与 [3,5]、[6,7]、[8,10] 都重叠，合并为 [3,10]
```

---

## 解法一览

| 解法 | 时间 | 空间 | 是否推荐面试 |
|---|---|---|---|
| 1. 追加后排序合并 | O(n log n) | O(n) | 否 |
| 2. 一趟扫描（三段法） | O(n) | O(n) | **最佳 ✅** |

---

## 解法 1：追加后排序合并

**思路：** 把 `newInterval` 追加到原列表，然后直接调用 Q56（合并区间）的解法——先按左端点排序，再合并。

**缺点：** 原数组已经有序，排序是多余的操作，时间 O(n log n)。面试中可作为铺垫。

---

## 解法 2：一趟扫描（三段法） ⭐ 面试推荐

**思路：** 利用原数组已排序的性质，一次遍历完成插入和合并。扫描过程中，区间自然分成三段：

```
第 1 段：在 newInterval 左边，完全不重叠（end < newStart）
         → 直接加入结果

第 2 段：与 newInterval 有重叠
         → 合并成一个区间：start = min, end = max

第 3 段：在 newInterval 右边，完全不重叠（start > newEnd）
         → 直接加入结果
```

### 判断逻辑

遍历每个区间 `[s, e]`：

```
if e < newStart:     → 在左边，不重叠，直接加入结果
elif s > newEnd:     → 在右边，不重叠，newInterval 先加入结果，再把当前区间及之后所有区间加入结果
else:                → 重叠，合并 newInterval
```

合并操作：
```
newStart = min(newStart, s)
newEnd = max(newEnd, e)
```

### 逐步演示

```
intervals = [[1,2],[3,5],[6,7],[8,10],[12,16]]
newInterval = [4,8]

[1,2]:   e=2 < newStart=4 → 左边，直接加入 ans = [[1,2]]
[3,5]:   s=3 <= 8 且 e=5 >= 4 → 重叠，合并为 [min(4,3), max(8,5)] = [3,8]
[6,7]:   s=6 <= 8 且 e=7 >= 3 → 重叠，合并为 [min(3,6), max(8,7)] = [3,8]
[8,10]:  s=8 <= 8 且 e=10 >= 3 → 重叠，合并为 [min(3,8), max(8,10)] = [3,10]
[12,16]: s=12 > newEnd=10 → 右边，先把 [3,10] 加入 ans，再追加 [12,16]

结果: [[1,2],[3,10],[12,16]]
```

**复杂度：** 时间 O(n)，只需遍历一次；空间 O(n) 存储结果。

---

## 面试建议

**解法 2（一趟扫描）是面试标准答案。**

面试时这样展开：
1. 先提一句"追加后用合并区间的做法能过，但浪费了数组已有序的条件"
2. 引出三段法："按与新区间的关系分成左、重叠、右三部分"
3. 写出判断逻辑，重点说明重叠时的合并规则（取 min start、取 max end）
4. 注意边界情况：newInterval 插在最前面、最后面、或数组为空
