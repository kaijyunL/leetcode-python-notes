# LeetCode 144 - 二叉树的前序遍历（Binary Tree Preorder Traversal）

## 题目

给你一棵二叉树的根节点 `root`，返回它的前序遍历结果。

前序遍历的顺序是：

```text
根 -> 左 -> 右
```

例如这棵树：

```text
    1
     \
      2
     /
    3
```

前序遍历结果是：

```text
[1, 2, 3]
```

---

## 这题本质是什么

这题本质不是“技巧题”，而是：

```text
如何按照 根 -> 左 -> 右 的顺序访问整棵树
```

所以第一性原理其实很简单：

1. 先处理当前节点
2. 再处理左子树
3. 最后处理右子树

只要你牢牢记住这个顺序，后面的递归、栈、Morris 都只是不同实现方式。

---

## 先把“前序”真正理解透

假设当前节点是 `root`，那么前序遍历做的事情就是：

```python
先把 root.val 放进答案
再遍历 root.left
再遍历 root.right
```

也就是说，这题不是在背模板，而是在执行下面这个定义：

```text
preorder(root) = [root] + preorder(root.left) + preorder(root.right)
```

递归版最贴近这个定义。

---

## 解法一览

| 解法 | 思路 | 时间复杂度 | 空间复杂度 | 是否推荐面试 |
|---|---|---|---|---|
| 1. 递归 | 直接按“根左右”的定义写 | `O(n)` | `O(h)` | 适合理解 |
| 2. 显式栈迭代 | 手动模拟递归调用栈 | `O(n)` | `O(h)` | **最适合面试 ✅** |
| 3. Morris 前序遍历 | 利用线索化思想做到 `O(1)` 额外空间 | `O(n)` | `O(1)` | 进阶加分 |

说明：

- `n` 是节点总数
- `h` 是树高
- 对平衡树，`h` 大约是 `log n`
- 对链式退化树，`h` 最坏是 `n`

---

## 解法 1：递归

对应文件：

```text
10_二叉树/01_q144/solution_1_recursive.py
```

### 思路

递归就是最自然的写法。

因为“前序遍历”的定义本身就长这样：

```text
访问当前节点
前序遍历左子树
前序遍历右子树
```

所以代码几乎就是把定义翻译成程序。

### 递归过程

```python
def dfs(node):
    if node is None:
        return

    ans.append(node.val)
    dfs(node.left)
    dfs(node.right)
```

### 为什么它最容易理解

因为每一层递归都在做完全相同的事：

1. 处理当前节点
2. 把同样的问题交给左子树
3. 再把同样的问题交给右子树

这其实就是“树问题分解成更小的树问题”。

### 优点

1. 最符合定义
2. 最容易写对
3. 最适合理解树遍历的本质

### 缺点

1. 依赖系统递归栈
2. 如果面试官要求非递归，这版不够
3. 有些面试官会继续追问“你能不能用迭代写”

### 复杂度

- 时间复杂度：`O(n)`，每个节点只访问一次
- 空间复杂度：`O(h)`，来自递归调用栈

## 解法 2：显式栈迭代

对应文件：

```text
10_二叉树/01_q144/solution_2_iterative_stack.py
```

## 这是最适合面试的方法

如果在面试里让我写第 144 题，我最推荐优先写这版。

原因很直接：

1. 它不依赖系统递归栈
2. 它清楚展示了你理解“递归其实就是栈”
3. 代码不复杂，现场手写稳定
4. 相比 Morris，出错率低很多

一句话总结：

```text
递归最直观，栈迭代最稳，Morris 最秀但不该先写。
```

---

### 从递归推到栈，怎么想

递归版做的是：

```text
先处理根
再处理左
再处理右
```

如果不用递归，我们就要自己维护“下一步该处理谁”。

这时最自然的数据结构就是：

```text
栈
```

因为递归的底层，本质上也是调用栈。

---

### 为什么要先压右子节点，再压左子节点

这是这题最关键的点。

栈是：

```text
后进先出
```

而前序遍历希望访问顺序是：

```text
根 -> 左 -> 右
```

当前节点出栈后：

1. 先记录当前节点值
2. 之后应该优先处理左子树
3. 但因为栈后进先出，所以必须先把右子节点压栈，再把左子节点压栈

这样下一次弹出的才会是左子节点。

这句一定要真正吃透：

```text
想让左边先处理，就要让右边先入栈。
```

---

### 过程演示

看这棵树：

```text
      1
     / \
    2   3
   / \   \
  4   5   6
```

初始：

```text
stack = [1]
ans = []
```

第一轮：

1. 弹出 `1`
2. 记录答案：`[1]`
3. 先压 `3`，再压 `2`

现在：

```text
stack = [3, 2]
```

第二轮：

1. 弹出 `2`
2. 记录答案：`[1, 2]`
3. 先压 `5`，再压 `4`

现在：

```text
stack = [3, 5, 4]
```

下一次弹出的就是 `4`，这正好符合前序的“先左后右”。

最终顺序：

```text
[1, 2, 4, 5, 3, 6]
```

---

### 最推荐背下来的代码骨架

```python
stack = [root]

while stack:
    node = stack.pop()
    ans.append(node.val)

    if node.right:
        stack.append(node.right)
    if node.left:
        stack.append(node.left)
```

这段骨架非常重要，因为它不只是第 144 题能用。

它体现的是一种通用思想：

```text
用栈控制遍历顺序
```

---

### 面试时怎么讲最顺

你可以这样讲：

1. 前序遍历顺序是根左右
2. 递归最好理解，但如果写迭代，本质上是在手动模拟递归栈
3. 我用一个栈先放根节点
4. 每次弹出一个节点就立刻记录，因为前序是先访问根
5. 然后先压右孩子，再压左孩子，这样左孩子会先被处理
6. 最终每个节点恰好进栈出栈一次，所以时间复杂度是 `O(n)`

这套表达很完整，而且很像你是真的想明白了。

---

### 常见错误

#### 错误 1：先压左，再压右

如果你写成：

```python
if node.left:
    stack.append(node.left)
if node.right:
    stack.append(node.right)
```

那么弹出顺序就会变成：

```text
根 -> 右 -> 左
```

这是错的。

---

#### 错误 2：把“入栈顺序”和“访问顺序”混了

要记住：

```text
栈里的压入顺序，和最后访问顺序是反着的
```

---

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`；最坏链式树时可到 `O(n)`

## 解法 3：Morris 前序遍历

对应文件：

```text
10_二叉树/01_q144/solution_3_morris.py
```

### 这版在优化什么

递归和显式栈，本质上都用了额外空间来记“回来的路”。

Morris 的想法是：

```text
既然遍历完左子树后还要回到当前节点，
那我能不能临时在树里造一条返回路径？
```

答案是可以。

于是我们会在遍历过程中，临时修改某些空指针，遍历完再恢复。

这样就能做到：

```text
额外空间 O(1)
```

---

### 前序版的关键点

对于当前节点 `cur`：

#### 情况 1：没有左子树

那就直接访问当前节点，然后去右边：

```python
ans.append(cur.val)
cur = cur.right
```

#### 情况 2：有左子树

我们要先找到左子树里“最右边的节点”，记作 `pred`。

这个节点是当前节点在前序遍历里，左子树部分最后会经过的位置之一，所以可以用它来临时连回 `cur`。

如果：

```python
pred.right is None
```

说明这条回路还没建：

1. 先访问 `cur`
2. 让 `pred.right = cur`
3. 再往 `cur.left` 走

如果：

```python
pred.right is cur
```

说明左子树已经遍历完了，现在是沿着临时线索回来的：

1. 把 `pred.right` 恢复成 `None`
2. 再去 `cur.right`

---

### 为什么这版不是面试首选

因为它虽然空间最优，但现场更容易写错：

1. 容易把“什么时候记录答案”搞错
2. 容易把前序版和中序版的 Morris 混掉
3. 容易忘记恢复指针

所以更好的策略通常是：

1. 先写栈迭代版稳稳拿下
2. 如果面试官追问空间还能不能优化，再补 Morris

---

### 复杂度

- 时间复杂度：`O(n)`，每条边最多被处理有限次
- 空间复杂度：`O(1)`，不算返回结果数组

---

## 三种写法怎么选

### 如果你现在是在“学懂这题”

先掌握递归。

因为它最贴近定义，最能帮你建立树遍历直觉。

---

### 如果你要准备面试

主练显式栈迭代。

这是第 144 题最稳的面试答案。

---

### 如果你想进一步拔高

再补 Morris。

它更像进阶武器，而不是首发武器。
