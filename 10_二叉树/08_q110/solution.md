# LeetCode 110 - 平衡二叉树（Balanced Binary Tree）

## 题目

给定一个二叉树，判断它是否是高度平衡的二叉树。

高度平衡的定义是：

```text
任意一个节点的左右两个子树的高度差不超过 1
```

注意，是任意节点，不只是根节点。

例如：

```text
      3
     / \
    9  20
       / \
      15  7
```

这棵树是平衡的，返回：

```text
True
```

再看这个例子：

```text
        1
       / \
      2   2
     / \
    3   3
   / \
  4   4
```

这棵树不是平衡的，返回：

```text
False
```

---

## 先说结论

这题和第 104 题“最大深度”关系很近。

要判断某个节点是否平衡，需要知道：

```text
左子树高度
右子树高度
```

然后判断：

```text
abs(left_height - right_height) <= 1
```

从第一性原理出发，真正自然的解法是：

1. 朴素自顶向下：每个节点都单独求左右高度，再递归判断左右子树
2. 后序 DFS：一边求高度，一边判断是否平衡，**最适合面试**

不需要为了凑方法写 BFS。

因为这题的核心信息是：

```text
子树高度
```

而高度天然适合用后序 DFS 从下往上返回。

---

## 这题本质是什么

一个节点是否平衡，需要满足三件事：

1. 左子树是平衡的
2. 右子树是平衡的
3. 左右子树高度差不超过 1

写成公式就是：

```text
balanced(root) =
    balanced(root.left)
    and balanced(root.right)
    and abs(height(root.left) - height(root.right)) <= 1
```

这句话非常重要。

它说明第 110 题不是只看根节点。

即使根节点左右高度差不超过 1，只要某个子树内部不平衡，整棵树也不是平衡二叉树。

---

## 解法一：朴素自顶向下

对应文件：

```text
10_二叉树/08_q110/solution_1_top_down.py
```

### 思路

最直观的想法是：

1. 对当前节点，分别计算左子树高度和右子树高度
2. 如果高度差超过 1，返回 `False`
3. 否则继续判断左子树是否平衡、右子树是否平衡

代码骨架：

```python
def isBalanced(root):
    if root is None:
        return True

    left_height = height(root.left)
    right_height = height(root.right)

    if abs(left_height - right_height) > 1:
        return False

    return isBalanced(root.left) and isBalanced(root.right)
```

### 问题在哪里

这个写法很好理解，但会重复计算高度。

比如判断根节点时，会计算整棵左子树高度。

接着递归判断左子树是否平衡时，又会重新计算左子树下面很多节点的高度。

也就是说：

```text
同一批节点的高度可能被反复算很多次
```

在退化成链表的树上，时间复杂度会变成：

```text
O(n^2)
```

### 复杂度

- 时间复杂度：平均接近 `O(n log n)`，最坏 `O(n^2)`
- 空间复杂度：`O(h)`，递归调用栈高度

## 解法二：后序 DFS

对应文件：

```text
10_二叉树/08_q110/solution_2_postorder.py
```

## 这是最适合面试的方法

如果面试里写第 110 题，我最推荐写后序 DFS。

原因是：

1. 它只计算一次每个节点的高度
2. 它从下往上判断平衡，符合题目定义
3. 一旦发现不平衡，可以用 `-1` 立刻向上传递失败
4. 时间复杂度是最优的 `O(n)`

一句话总结：

```text
后序 DFS 返回子树高度；如果子树不平衡，就返回 -1 作为失败信号。
```

---

### 从第一性原理推导

判断当前节点是否平衡，需要先知道：

```text
左子树高度
右子树高度
```

所以访问顺序应该是：

```text
左 -> 右 -> 根
```

这正是后序遍历。

我们定义一个函数：

```python
def height(node):
```

它有两个含义：

```text
如果以 node 为根的子树平衡，返回它的高度
如果这棵子树不平衡，返回 -1
```

为什么用 `-1`？

因为正常树高不会是负数。

所以 `-1` 可以作为一个清晰的失败信号。

---

### 递归过程

先算左子树：

```python
left_height = height(node.left)
if left_height == -1:
    return -1
```

如果左子树已经不平衡，当前树肯定也不平衡，直接返回 `-1`。

再算右子树：

```python
right_height = height(node.right)
if right_height == -1:
    return -1
```

然后判断当前节点：

```python
if abs(left_height - right_height) > 1:
    return -1
```

如果当前节点也平衡，就返回正常高度：

```python
return 1 + max(left_height, right_height)
```

最终：

```python
return height(root) != -1
```

---

### 为什么这比朴素法更好

朴素法的问题是：

```text
判断平衡时算高度
递归到子树时又重新算高度
```

后序 DFS 把两件事合并了：

```text
算高度的同时判断平衡
```

每个节点只处理一次。

所以时间复杂度从最坏 `O(n^2)` 优化到：

```text
O(n)
```

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(h)`，递归调用栈高度

## 面试推荐

第 110 题最适合面试的方法是：

```text
后序 DFS
```

面试时可以这样讲：

```text
判断一棵树是否平衡，需要先知道左右子树高度，
所以我用后序 DFS 从下往上计算。
递归函数返回当前子树高度。
如果某棵子树已经不平衡，就返回 -1 作为失败信号。
当前节点拿到左右高度后，如果高度差超过 1，也返回 -1。
否则返回 1 + max(left_height, right_height)。
最后判断 height(root) 是否等于 -1。
```

复杂度：

```text
每个节点只访问一次，时间复杂度 O(n)。
递归栈深度等于树高，空间复杂度 O(h)。
```

## 和第 104 题的关系

第 104 题是：

```text
只要求最大深度
```

所以每个节点直接返回：

```text
1 + max(left_height, right_height)
```

第 110 题是在 104 的基础上多做一步判断：

```text
左右高度差是否超过 1
```

所以可以把它理解为：

```text
104 的高度递归 + 平衡性检查 + 失败信号剪枝
```
