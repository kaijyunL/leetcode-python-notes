# LeetCode 103 - 二叉树的锯齿形层序遍历（Binary Tree Zigzag Level Order Traversal）

## 题目

给你一棵二叉树的根节点 `root`，返回它的锯齿形层序遍历结果。

也就是说：

```text
第一层：从左到右
第二层：从右到左
第三层：从左到右
第四层：从右到左
……
```

例如：

```text
      3
     / \
    9  20
       / \
      15  7
```

普通层序遍历是：

```text
[
  [3],
  [9, 20],
  [15, 7]
]
```

锯齿形层序遍历是：

```text
[
  [3],
  [20, 9],
  [15, 7]
]
```

---

## 先说结论

第 103 题是第 102 题的变体。

第 102 题是：

```text
每一层都从左到右
```

第 103 题是：

```text
层与层之间方向交替
```

从第一性原理出发，真正自然的解法是：

1. BFS 正常分层，奇数层反转，**最适合面试**
2. BFS + deque，根据方向从头或尾放入当前层
3. DFS 按深度分组，根据深度奇偶决定加入方向

这题不需要额外凑一个很绕的“暴力法”。

因为题目本质仍然是按层返回结果，最朴素的想法就是：

```text
先按第 102 题正常拿到每一层，再根据层号决定要不要反转这一层。
```

这个朴素想法本身已经是 `O(n)`，也是最优时间复杂度。

最推荐的写法是：

```text
复用第 102 题 BFS 模板，然后用 left_to_right 控制当前层方向。
```

---

## 这题本质是什么

第 103 题不是一种全新的二叉树遍历。

它还是层序遍历，只是每一层的输出方向不同。

可以记成：

```text
103 = 102 + 每一层方向交替
```

所以核心问题不是：

```text
怎么遍历二叉树？
```

而是：

```text
已经拿到当前层以后，应该按什么方向放进答案？
```

---

## 最容易错的地方

### 1. 改错了入队顺序

很多人会想：

```text
这一层从右到左，那我是不是应该先入右孩子，再入左孩子？
```

不建议这样写。

因为队列负责的是：

```text
保证下一层节点的自然顺序仍然是从左到右
```

如果随便改左右孩子入队顺序，后面层的结构很容易乱。

更稳的做法是：

```text
孩子永远按 left -> right 入队；
只改变当前 level 的输出顺序。
```

### 2. 忘记固定当前层长度

和第 102、107 题一样，BFS 分层时必须先写：

```python
level_size = len(queue)
```

否则当前层和下一层会混在一起。

### 3. 方向切换的位置写错

每处理完一整层，才切换方向：

```python
left_to_right = not left_to_right
```

不要在处理每个节点时切换。

## 解法一：BFS 正常分层，奇数层反转

对应文件：

```text
10_二叉树/14_q103/solution_1_bfs_reverse.py
```

## 这是最适合面试的方法

如果面试里写第 103 题，我最推荐这一版。

原因是：

1. 它最大程度复用第 102 题 BFS 模板
2. 队列逻辑始终简单：左孩子先入队，右孩子后入队
3. 只在当前层列表 `level` 上处理方向，责任很清楚
4. 代码现场手写稳定，不容易把层和方向搞乱

一句话总结：

```text
BFS 一层一层拿节点；如果当前层应该从右到左，就把这一层反转。
```

---

### 从第 102 题推过来

第 102 题的核心模板是：

```python
while queue:
    level = []
    level_size = len(queue)

    for _ in range(level_size):
        node = queue.popleft()
        level.append(node.val)

        if node.left:
            queue.append(node.left)
        if node.right:
            queue.append(node.right)

    ans.append(level)
```

第 103 题只多一个方向变量：

```python
left_to_right = True
```

如果当前层不是从左到右，就反转：

```python
if not left_to_right:
    level.reverse()
```

处理完这一层后切换方向：

```python
left_to_right = not left_to_right
```

---

### 为什么不要改孩子入队顺序

BFS 队列里的顺序，最好一直保持下一层的自然顺序。

也就是：

```text
父节点从左到右处理；
孩子也按 left -> right 入队。
```

这样下一层被取出来时，天然就是从左到右。

至于题目要求这一层从右到左，只需要在输出 `level` 时反转。

把“遍历顺序”和“输出顺序”分开，代码最稳。

---

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次；每层反转的总元素数不超过 `n`
- 空间复杂度：`O(w)`，队列最多保存一层左右的节点，不计返回结果

## 解法二：BFS + deque 按方向插入

对应文件：

```text
10_二叉树/14_q103/solution_2_bfs_deque.py
```

### 思路

解法一是：

```text
先正常 append，再根据方向 reverse
```

解法二是在构造当前层时就处理方向。

如果当前层从左到右：

```python
level.append(node.val)
```

如果当前层从右到左：

```python
level.appendleft(node.val)
```

这里的 `level` 用 `deque`。

最后转成列表：

```python
ans.append(list(level))
```

### 是否比解法一更适合面试

它也很好，少了一次显式 `reverse()`。

但我仍然更推荐解法一。

因为解法一更容易从第 102 题直接推出来，面试现场解释更顺。

解法二适合在你已经写熟 BFS 模板以后，用来展示你知道 `deque` 可以从两端插入。

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(w)`，不计返回结果

## 解法三：DFS 按深度分组

对应文件：

```text
10_二叉树/14_q103/solution_3_dfs.py
```

### 思路

和第 102 题一样，也可以递归时带上深度：

```python
dfs(node, depth)
```

`depth` 表示当前节点在第几层。

如果是偶数层：

```python
ans[depth].append(node.val)
```

如果是奇数层：

```python
ans[depth].appendleft(node.val)
```

最后把每一层的 `deque` 转成普通列表。

### 为什么它是补充方法

DFS 能做，是因为：

```text
层 = depth
```

但题目名字叫“层序遍历”，BFS 更贴近题意。

所以面试主写 BFS，DFS 作为补充即可。

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`，递归调用栈高度，不计返回结果

## 面试推荐

第 103 题最适合面试的方法是：

```text
BFS 正常分层，奇数层反转
```

面试时可以这样讲：

```text
这题是第 102 题的变体。
我先用队列做普通层序遍历。
每一轮 while 代表一层，先用 level_size = len(queue) 固定当前层节点数。
处理这一层时，孩子始终按 left -> right 入队，保证队列结构稳定。
当前层的节点值先正常加入 level。
如果这一层方向是从右到左，就 level.reverse()。
把 level 加入答案后，再切换 left_to_right。
```

复杂度：

```text
每个节点访问一次，每层反转加起来也只会覆盖所有节点一次。
所以时间复杂度 O(n)。
队列最多保存一层左右的节点，空间复杂度 O(w)，不计返回结果。
```

## 和第 102、107 题的关系

可以这样记：

```text
102：普通层序遍历
107：普通层序遍历 + 反转层的顺序
103：普通层序遍历 + 每一层方向交替
```

三题最核心的共同模板都是：

```python
level_size = len(queue)
```

区别只在于：

```text
拿到 level 以后，怎么放进 ans
```
