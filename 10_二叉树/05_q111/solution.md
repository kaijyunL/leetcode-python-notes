# LeetCode 111 - 二叉树的最小深度（Minimum Depth of Binary Tree）

## 题目

给定一个二叉树的根节点 `root`，返回它的最小深度。

最小深度是指：

```text
从根节点到最近叶子节点的最短路径上的节点数
```

注意，叶子节点必须是：

```text
没有左孩子，也没有右孩子的节点
```

例如：

```text
      3
     / \
    9  20
       / \
      15  7
```

最小深度是：

```text
2
```

因为最近的叶子节点是 `9`，路径是：

```text
3 -> 9
```

路径上有 2 个节点。

---

## 先说结论

这题和第 104 题“最大深度”长得很像，但不能简单照抄。

第 104 题是：

```text
最大深度 = 1 + max(左深度, 右深度)
```

第 111 题最容易写错的地方是误以为：

```text
最小深度 = 1 + min(左深度, 右深度)
```

这在某些树上是错的。

真正自然的解法有两个：

1. 递归 DFS：分情况处理空子树
2. BFS 层序遍历：从上到下找第一个叶子，**最适合面试**

---

## 这题本质是什么

最小深度问的是：

```text
从 root 出发，最快几层能走到一个叶子节点？
```

这里的关键词是：

```text
叶子节点
```

叶子节点必须同时满足：

```text
left is None
right is None
```

所以，如果一个节点只有一个孩子，不能把缺失的那边当成深度 `0` 的叶子路径。

---

## 最容易错的写法

很多人会从第 104 题直接迁移，写成：

```python
if root is None:
    return 0

return 1 + min(self.minDepth(root.left), self.minDepth(root.right))
```

这个写法是错的。

看这个例子：

```text
1
 \
  2
```

如果套上面的公式：

```text
left_depth = 0
right_depth = 1
return 1 + min(0, 1) = 1
```

但真实答案应该是：

```text
2
```

因为路径必须走到叶子节点 `2`：

```text
1 -> 2
```

空的左子树不是一条有效路径。

## 解法一览

| 解法 | 思路 | 时间复杂度 | 空间复杂度 | 是否推荐面试 |
|---|---|---|---|---|
| 1. 递归 DFS | 分情况计算左右子树最小深度 | `O(n)` | `O(h)` | 适合理解边界 |
| 2. BFS 层序遍历 | 第一次遇到叶子就返回深度 | `O(n)` | `O(w)` | **最适合面试 ✅** |

说明：

- `n` 是节点总数
- `h` 是树高
- `w` 是树的最大宽度
- 空间复杂度不计算输入树本身

## 解法 1：递归 DFS

对应文件：

```text
10_二叉树/05_q111/solution_1_recursive.py
```

### 思路

递归仍然是树题最自然的第一版。

但是这题必须分情况：

1. 空树：深度是 `0`
2. 叶子节点：深度是 `1`
3. 只有左子树：只能走左边
4. 只有右子树：只能走右边
5. 左右子树都有：取较小值

代码逻辑就是：

```python
if root is None:
    return 0

if root.left is None and root.right is None:
    return 1

if root.left is None:
    return 1 + self.minDepth(root.right)

if root.right is None:
    return 1 + self.minDepth(root.left)

return 1 + min(self.minDepth(root.left), self.minDepth(root.right))
```

### 为什么要这么写

关键在于：

```text
只有存在的子树，才可能通向叶子节点。
```

如果左子树为空、右子树不为空，那么最小深度必须来自右子树。

不能因为左边是 `0`，就把它当成更短路径。

### 复杂度

- 时间复杂度：`O(n)`，最坏情况下每个节点访问一次
- 空间复杂度：`O(h)`，递归调用栈高度

## 解法 2：BFS 层序遍历

对应文件：

```text
10_二叉树/05_q111/solution_2_bfs.py
```

## 这是最适合面试的方法

如果面试里写第 111 题，我最推荐写 BFS。

原因是：

1. 最小深度本质上是“离根最近的叶子”
2. BFS 天然按层从近到远搜索
3. 第一次遇到叶子节点就可以直接返回
4. 它能自然避开“空子树不是叶子路径”的坑

一句话总结：

```text
BFS 一层层找，第一个遇到的叶子一定就是最近叶子。
```

---

### 从第一性原理推导

最小深度要找的是：

```text
最近的叶子节点
```

既然要找最近，那就应该从根开始：

```text
第 1 层
第 2 层
第 3 层
...
```

一层一层往外扩。

这正是 BFS 的特点。

只要某一层第一次出现叶子节点，就可以立刻返回当前深度。

因为更深的层肯定不可能更短。

---

### BFS 过程

队列里保存：

```text
(node, depth)
```

意思是：

```text
当前节点 node 位于第 depth 层
```

每次从队列左侧弹出节点：

1. 如果它是叶子，直接返回 `depth`
2. 如果它有左孩子，把左孩子和 `depth + 1` 入队
3. 如果它有右孩子，把右孩子和 `depth + 1` 入队

核心代码：

```python
queue = deque([(root, 1)])

while queue:
    node, depth = queue.popleft()

    if node.left is None and node.right is None:
        return depth

    if node.left:
        queue.append((node.left, depth + 1))
    if node.right:
        queue.append((node.right, depth + 1))
```

---

### 用例走一遍

树是：

```text
      3
     / \
    9  20
       / \
      15  7
```

初始：

```text
queue = [(3, 1)]
```

弹出 `3`：

```text
3 不是叶子，把 9 和 20 入队
queue = [(9, 2), (20, 2)]
```

弹出 `9`：

```text
9 没有左孩子，也没有右孩子，是叶子
```

所以直接返回：

```text
2
```

不需要再看 `20、15、7`。

这就是 BFS 在最小深度题上的优势。

### 复杂度

- 时间复杂度：`O(n)`，最坏情况下仍可能访问所有节点
- 空间复杂度：`O(w)`，队列最多保存一层节点

## 面试推荐

第 111 题最适合面试的方法是：

```text
BFS 层序遍历
```

面试时可以这样讲：

```text
最小深度等价于离根最近的叶子节点所在层数。
所以我用 BFS 从根节点开始按层遍历。
队列中保存节点和它的深度。
每次弹出一个节点，如果它是叶子节点，就说明这是从上到下遇到的第一个叶子，
它的 depth 就是最小深度，可以直接返回。
如果不是叶子，就把存在的左右孩子加入队列，深度加 1。
最坏情况下每个节点访问一次，时间复杂度 O(n)，队列空间 O(w)。
```

如果面试官想看递归，再补 DFS：

```text
递归时要特别处理只有一个子树的情况，不能直接对 0 取 min。
```
