# LeetCode 145 - 二叉树的后序遍历（Binary Tree Postorder Traversal）

## 题目

给你一棵二叉树的根节点 `root`，返回它的后序遍历结果。

后序遍历顺序是：

```text
左 -> 右 -> 根
```

例如这棵树：

```text
    1
     \
      2
     /
    3
```

后序遍历结果是：

```text
[3, 2, 1]
```

---

## 先说结论

这题没有真正意义上的“暴力解法”。

原因很简单：

```text
你要返回整棵树的遍历结果，就必须至少看一遍每个节点。
```

所以时间复杂度下界就是 `O(n)`。

从第一性原理出发，真正自然的层次是：

1. 递归：直接翻译后序遍历定义，最适合理解
2. 显式栈 + 访问标记：手动模拟递归，**最适合面试**
3. Morris 后序遍历：额外空间 `O(1)`，理论最优，但代码复杂，属于进阶

---

## 这题本质是什么

后序遍历的定义是：

```text
postorder(root) = postorder(root.left) + postorder(root.right) + [root.val]
```

也就是说，对于任意节点 `root`：

1. 必须先处理完左子树
2. 再处理完右子树
3. 最后才能访问当前节点

后序遍历最难的地方就在这里：

```text
当前节点明明最先被看到，却必须最后被加入答案。
```

递归里，这件事由系统调用栈自动帮你完成。

迭代里，就需要我们自己设计一个栈，明确记录：

```text
这个节点是第一次遇到，还是它的左右子树都处理完了。
```

---

## 解法一览

| 解法 | 思路 | 时间复杂度 | 空间复杂度 | 是否推荐面试 |
|---|---|---|---|---|
| 1. 递归 | 直接按“左右根”的定义写 | `O(n)` | `O(h)` | 适合理解 |
| 2. 显式栈 + 访问标记 | 手动模拟递归调用栈 | `O(n)` | `O(h)` | **最适合面试 ✅** |
| 3. Morris 后序遍历 | 临时线索化二叉树 | `O(n)` | `O(1)` | 进阶加分 |

说明：

- `n` 是节点总数
- `h` 是树高
- 平衡树时，`h ≈ log n`
- 极端退化成链表时，`h = n`
- 空间复杂度默认不计算返回结果 `ans`

## 解法 1：递归

对应文件：

```text
10_二叉树/03_q145/solution_1_recursive.py
```

### 思路

后序遍历定义就是：

```text
左 -> 右 -> 根
```

所以递归写法几乎就是把定义翻译成代码：

```python
def dfs(node):
    if node is None:
        return

    dfs(node.left)
    dfs(node.right)
    ans.append(node.val)
```

### 为什么先从递归开始

因为树天然就是递归结构：

```text
一棵树 = 根节点 + 左子树 + 右子树
```

而后序遍历也是递归定义：

```text
后序遍历一棵树 = 后序遍历左子树 + 后序遍历右子树 + 访问根节点
```

所以递归版最适合作为第一版。

### 优点

1. 最符合定义
2. 代码最短
3. 最容易验证正确性

### 缺点

1. 依赖系统递归栈
2. 树特别深时可能触发 Python 的递归深度限制
3. 面试官经常会继续追问非递归写法

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(h)`，来自递归调用栈，不计返回结果

## 解法 2：显式栈 + 访问标记

对应文件：

```text
10_二叉树/03_q145/solution_2_iterative_stack.py
```

## 这是最适合面试的方法

如果面试里写第 145 题，我最推荐写这一版。

原因是：

1. 它不依赖 Python 的系统递归栈
2. 它不是投机技巧，而是在真正模拟递归
3. 它的逻辑能直接解释“为什么根节点最后访问”
4. 代码比 Morris 简单很多，现场手写更稳

一句话总结：

```text
递归版最适合理解，显式栈 + 访问标记最适合面试，Morris 是空间进阶。
```

---

### 从递归推到栈

递归版的核心是：

```python
dfs(node.left)
dfs(node.right)
ans.append(node.val)
```

注意，当前节点 `node` 不是马上加入答案。

它必须等两件事都完成：

```text
左子树处理完
右子树处理完
```

然后才能访问自己。

所以迭代版需要解决一个问题：

```text
第一次看到 node 时，不能访问它；
但以后必须回来访问它。
```

这正是栈适合做的事。

---

### 用 visited 标记表达状态

我们在栈里放二元组：

```text
(node, visited)
```

含义是：

- `visited = False`：第一次遇到这个节点，还不能加入答案
- `visited = True`：它的左右子树已经安排处理过了，现在可以加入答案

当第一次遇到一个节点时，后序顺序应该是：

```text
左 -> 右 -> 根
```

但是栈是后进先出，所以压栈顺序要反过来：

```text
先压 根(True)
再压 右(False)
最后压 左(False)
```

这样弹出时才会变成：

```text
左 -> 右 -> 根
```

核心代码：

```python
stack = [(root, False)]

while stack:
    node, visited = stack.pop()

    if visited:
        ans.append(node.val)
        continue

    stack.append((node, True))
    if node.right:
        stack.append((node.right, False))
    if node.left:
        stack.append((node.left, False))
```

---

### 用例走一遍

树是：

```text
    1
     \
      2
     /
    3
```

初始：

```text
stack = [(1, False)]
ans = []
```

第一次弹出 `1`：

```text
1 不能马上访问，先安排它最后访问
stack = [(1, True), (2, False)]
```

弹出 `2`：

```text
2 也不能马上访问
stack = [(1, True), (2, True), (3, False)]
```

弹出 `3`：

```text
3 没有左右子树，安排自己
stack = [(1, True), (2, True), (3, True)]
```

弹出 `(3, True)`：

```text
ans = [3]
```

弹出 `(2, True)`：

```text
ans = [3, 2]
```

弹出 `(1, True)`：

```text
ans = [3, 2, 1]
```

这正好就是后序遍历。

---

### 为什么它比“反转前序”更适合讲面试

还有一种常见写法是：

```text
先得到 根 -> 右 -> 左
再反转成 左 -> 右 -> 根
```

这很短，也能通过。

但它更像一个技巧。

面试里，`visited` 标记法更推荐，因为它直接回答了后序遍历的核心矛盾：

```text
第一次看到根节点时不能访问，要等左右子树都处理完再回来访问。
```

这比单纯背一个“反转前序”的技巧更扎实。

### 复杂度

- 时间复杂度：`O(n)`，每个节点最多入栈两次，仍然是线性
- 空间复杂度：`O(h)`，栈深度和树高相关，不计返回结果

## 解法 3：Morris 后序遍历

对应文件：

```text
10_二叉树/03_q145/solution_3_morris.py
```

### 它解决什么问题

递归和显式栈都需要 `O(h)` 的额外空间。

如果面试官继续追问：

```text
能不能做到 O(1) 额外空间？
```

这时才轮到 Morris。

Morris 的核心思想是：

```text
利用树里原本为空的右指针，临时建立“回来的路”，用完再恢复。
```

注意：

```text
Morris 会临时修改树结构，但遍历结束后会恢复。
```

---

### 为什么后序 Morris 更复杂

前序和中序 Morris 比较直接，因为当前节点的访问时机比较容易判断。

但后序是：

```text
左 -> 右 -> 根
```

根节点要最后访问。

所以 Morris 后序一般会加一个虚拟头节点 `dummy`：

```text
dummy.left = root
```

这样真正的 `root` 也能被当作某个节点的左子树处理。

当第二次从某个左子树回到当前节点时，说明这棵左子树已经处理完。

此时要把这棵左子树的右边界按反向加入答案。

---

### 不建议面试优先写 Morris 的原因

Morris 后序虽然空间最优，但它有几个现实问题：

1. 代码细节多
2. 需要反转右指针链
3. 写错后容易破坏树结构
4. 大多数面试并不要求你主动写到这个程度

所以推荐策略是：

```text
正常面试：先写显式栈 + visited 标记
追问 O(1) 空间：再讲 Morris 思路
```

### 复杂度

- 时间复杂度：`O(n)`，每条边只被有限次处理
- 空间复杂度：`O(1)`，不计返回结果

---

## 推荐记忆顺序

1. 先牢牢记住后序定义：

```text
左 -> 右 -> 根
```

2. 再写递归：

```text
dfs(left)
dfs(right)
visit(root)
```

3. 面试主写显式栈：

```text
第一次遇到节点：先别访问，安排它之后再访问
第二次遇到节点：左右子树已经处理完，可以访问
```

4. 最后了解 Morris：

```text
临时借用空右指针建立回路，做到 O(1) 额外空间
```
