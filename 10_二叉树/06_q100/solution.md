# LeetCode 100 - 相同的树（Same Tree）

## 题目

给你两棵二叉树的根节点 `p` 和 `q`，判断它们是否相同。

两棵二叉树相同，需要同时满足：

```text
结构相同
对应位置的节点值相同
```

例如：

```text
    1          1
   / \        / \
  2   3      2   3
```

这两棵树相同，返回：

```text
True
```

再看这个例子：

```text
    1          1
   /            \
  2              2
```

节点值看起来一样，但结构不同，所以返回：

```text
False
```

---

## 先说结论

这题不需要硬凑很多方法。

从第一性原理出发，真正自然的解法是：

1. 序列化后比较：直观但不推荐面试主写
2. 递归 DFS：同步比较两棵树，**最适合面试**
3. 迭代 BFS/队列：递归的非递归版本，作为补充

本题最重要的不是技巧，而是想清楚：

```text
比较两棵树，就是比较当前两个节点，再比较它们的左子树和右子树。
```

---

## 这题本质是什么

判断两棵树是否相同，本质上是同步遍历两棵树。

站在两个当前节点 `p` 和 `q` 上，只需要判断四种情况：

1. `p` 和 `q` 都是空：这一部分相同
2. `p` 是空、`q` 不是空：结构不同
3. `p` 不是空、`q` 是空：结构不同
4. `p` 和 `q` 都不是空：值要相同，并且左右子树也要相同

写成递归定义就是：

```text
isSameTree(p, q) =
    True                                      p 和 q 都为空
    False                                     p 和 q 只有一个为空
    p.val == q.val
      and isSameTree(p.left, q.left)
      and isSameTree(p.right, q.right)        p 和 q 都非空
```

这就是这题最核心的公式。

## 朴素思路：序列化后比较

对应文件：

```text
10_二叉树/06_q100/solution_1_serialize.py
```

### 思路

最直观的想法是：

```text
把两棵树都转换成一个列表
然后比较两个列表是否完全相等
```

比如用前序遍历，并且把空节点也记录下来：

```text
树 A: [1, 2, None, None, 3, None, None]
树 B: [1, 2, None, None, 3, None, None]
```

列表一样，说明树相同。

注意：

```text
空节点 None 必须记录。
```

否则下面两棵树会被误判：

```text
    1          1
   /            \
  2              2
```

如果不记录空节点，前序遍历值都是：

```text
[1, 2]
```

但它们结构不同。

### 为什么它不适合面试主写

这个方法很好理解，但不是最推荐的面试写法。

原因是：

1. 需要额外生成两个列表
2. 明明可以边比较边返回，却先完整遍历两棵树
3. 没有直接体现树的递归结构

它适合用来建立直觉：

```text
相同的树 = 结构和值都相同
```

但正式面试更推荐递归 DFS。

### 复杂度

- 时间复杂度：`O(n)`，需要遍历两棵树
- 空间复杂度：`O(n)`，需要保存序列化结果

## 解法 2：递归 DFS

对应文件：

```text
10_二叉树/06_q100/solution_2_recursive.py
```

## 这是最适合面试的方法

如果面试里写第 100 题，我最推荐写递归 DFS。

原因是：

1. 它直接表达了“相同的树”的定义
2. 代码短，现场手写稳定
3. 可以遇到不相同的位置立刻返回
4. 不需要额外保存遍历结果

一句话总结：

```text
两个当前节点相同，并且左子树相同、右子树相同，这两棵树才相同。
```

---

### 从第一性原理推导

比较两棵树，从根节点开始。

假设当前比较的是 `p` 和 `q`。

第一种情况：

```text
p is None and q is None
```

两边都是空，说明这一块结构一样，返回 `True`。

第二种情况：

```text
p is None or q is None
```

只有一边为空，说明结构不同，返回 `False`。

第三种情况：

```text
p.val != q.val
```

结构虽然当前都存在，但值不同，返回 `False`。

剩下的情况是：

```text
p 和 q 都存在，而且值相同
```

这时还不能直接说整棵树相同。

还必须继续比较：

```text
p.left 和 q.left 是否相同
p.right 和 q.right 是否相同
```

所以代码就是：

```python
return (
    self.isSameTree(p.left, q.left)
    and self.isSameTree(p.right, q.right)
)
```

---

### 为什么这个递归一定正确

因为二叉树本身就是递归结构：

```text
一棵树 = 当前节点 + 左子树 + 右子树
```

所以判断两棵树相同也应该递归地判断：

```text
当前节点相同
左子树相同
右子树相同
```

三件事同时成立，整棵树才相同。

### 复杂度

- 时间复杂度：`O(n)`，`n` 是两棵树中较小范围内需要比较的节点数；最坏情况要遍历所有对应节点
- 空间复杂度：`O(h)`，递归调用栈高度

## 解法 3：迭代 BFS / 队列

对应文件：

```text
10_二叉树/06_q100/solution_3_iterative_queue.py
```

### 思路

递归是在“同步比较两个节点”。

迭代版也做同一件事，只是把系统递归栈换成我们自己的队列。

队列里放：

```text
(p_node, q_node)
```

每次取出一对节点：

1. 两个都为空：继续
2. 只有一个为空：返回 `False`
3. 两个都非空但值不同：返回 `False`
4. 两个都非空且值相同：把左右孩子成对入队

核心代码：

```python
queue = deque([(p, q)])

while queue:
    node_p, node_q = queue.popleft()

    if node_p is None and node_q is None:
        continue
    if node_p is None or node_q is None:
        return False
    if node_p.val != node_q.val:
        return False

    queue.append((node_p.left, node_q.left))
    queue.append((node_p.right, node_q.right))

return True
```

### 什么时候写迭代版

如果面试官追问：

```text
能不能不用递归？
```

就写这一版。

但默认情况下，递归版更自然、更简洁。

### 复杂度

- 时间复杂度：`O(n)`，最坏情况下比较所有对应节点
- 空间复杂度：`O(w)`，队列最多保存一层节点对

## 面试推荐

第 100 题最适合面试的方法是：

```text
递归 DFS
```

面试时可以这样讲：

```text
我同步比较两棵树的当前节点。
如果两个节点都为空，说明这一部分相同。
如果只有一个为空，说明结构不同。
如果两个节点都存在但值不同，说明节点值不同。
否则继续递归比较左子树和右子树。
只有当前节点、左子树、右子树都相同，整棵树才相同。
```

复杂度可以这样说：

```text
最坏情况下需要比较所有对应节点，时间复杂度 O(n)。
递归栈深度等于树高，空间复杂度 O(h)。
```

---

## 推荐记忆顺序

1. 先记住判断标准：

```text
结构相同 + 对应节点值相同
```

2. 再记住递归三步：

```text
两个都空：True
只有一个空：False
值不同：False
继续比较左右子树
```

3. 最后知道迭代补充：

```text
队列里放成对节点，同步比较
```
