# LeetCode 129 - 求根节点到叶节点数字之和（Sum Root to Leaf Numbers）

## 题目

给你一棵二叉树的根节点 `root`，树中每个节点都存放一个 `0` 到 `9` 的数字。

每一条从根节点到叶子节点的路径，都代表一个数字。

例如路径：

```text
1 -> 2 -> 3
```

代表数字：

```text
123
```

返回所有根到叶路径代表的数字之和。

例如：

```text
    1
   / \
  2   3
```

有两条根到叶路径：

```text
1 -> 2  代表 12
1 -> 3  代表 13
```

所以答案是：

```text
12 + 13 = 25
```

---

## 先说结论

第 129 题和第 112、113 题一样，都是二叉树根到叶路径问题。

区别是：

```text
112：判断有没有路径和等于 targetSum
113：返回所有路径和等于 targetSum 的路径
129：把每条路径看成一个数字，然后求所有数字之和
```

从第一性原理出发，真正自然的解法是：

1. 朴素暴力：先收集所有根到叶路径，再把每条路径转成数字求和
2. DFS 递归：边走边累积当前数字，**最适合面试**
3. 迭代栈：递归思想的非递归版本

这题最核心的公式是：

```text
next_number = current_number * 10 + node.val
```

---

## 这题本质是什么

路径上的节点值不是普通求和，而是在构造十进制数字。

如果当前已经走出的数字是：

```text
12
```

下一个节点值是：

```text
3
```

那么新数字不是：

```text
12 + 3
```

而是：

```text
12 * 10 + 3 = 123
```

所以每往下一层，状态更新就是：

```text
当前数字左移一位十进制，再加上当前节点值
```

也就是：

```python
current = current * 10 + node.val
```

当走到叶子节点时，当前数字就是这条根到叶路径代表的完整数字。

---

## 最容易错的地方

### 1. 把路径当普通和来算

错误直觉：

```text
1 -> 2 -> 3  算成 1 + 2 + 3 = 6
```

正确含义：

```text
1 -> 2 -> 3  代表 123
```

### 2. 必须走到叶子节点才加入答案

和第 112、113 一样，路径必须是：

```text
root -> leaf
```

不能中途就把当前数字加入答案。

### 3. 空树答案是 0

如果 `root is None`，没有任何路径，所以返回：

```text
0
```

---

## 解法一：朴素暴力收集路径

对应文件：

```text
10_二叉树/11_q129/solution_1_bruteforce.py
```

### 思路

最直观的想法是：

1. 先枚举所有根到叶路径
2. 每条路径都是一个列表
3. 再把列表转成数字
4. 最后把所有数字加起来

例如：

```text
路径 [1, 2] -> 数字 12
路径 [1, 3] -> 数字 13
```

总和：

```text
25
```

### 为什么它不适合面试主写

这个方法很直观，但不够简洁。

它会额外保存所有路径：

```text
[[1, 2], [1, 3]]
```

然后再二次遍历这些路径转换数字。

其实我们可以在 DFS 的过程中直接维护当前数字，不需要先保存路径。

### 复杂度

- 时间复杂度：`O(n * h)`，路径转换数字需要遍历路径
- 空间复杂度：`O(n * h)`，需要保存所有根到叶路径

## 解法二：DFS 递归累积数字

对应文件：

```text
10_二叉树/11_q129/solution_2_recursive.py
```

## 这是最适合面试的方法

如果面试里写第 129 题，我最推荐写 DFS 递归累积数字。

原因是：

1. 它直接表达了根到叶路径的搜索过程
2. 不需要保存完整路径
3. 每个节点只访问一次
4. 代码短，现场手写稳定

一句话总结：

```text
每走到一个节点，就把当前数字更新为 current * 10 + node.val；
走到叶子时，把 current 加入答案。
```

---

### 从第一性原理推导

定义一个 DFS 函数：

```python
def dfs(node, current):
```

含义是：

```text
走到 node 之前，路径已经形成的数字是 current
```

进入当前节点后，更新数字：

```python
current = current * 10 + node.val
```

如果当前节点是叶子节点：

```python
if node.left is None and node.right is None:
```

说明一条完整路径结束了。

此时直接返回当前数字：

```python
return current
```

如果不是叶子，就继续向左右子树搜索：

```python
return dfs(node.left, current) + dfs(node.right, current)
```

为什么是加法？

因为题目要的是：

```text
所有根到叶数字之和
```

左子树产生的所有数字和，加上右子树产生的所有数字和，就是当前子树的答案。

---

### 为什么不需要 path

第 113 题必须返回完整路径，所以需要 `path`。

第 129 题只需要路径代表的数字之和。

所以只需要维护：

```text
当前路径已经形成的数字
```

不需要保存路径列表。

这和第 112 题很像：

```text
112 只维护 remain
129 只维护 current number
```

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(h)`，递归调用栈高度

## 解法三：迭代栈

对应文件：

```text
10_二叉树/11_q129/solution_3_iterative_stack.py
```

### 思路

递归版每一层保存两个信息：

```text
当前节点
当前路径已经形成的数字
```

迭代版用栈手动保存：

```text
(node, current)
```

每次弹出一个节点后：

1. 更新当前数字：`current = current * 10 + node.val`
2. 如果是叶子节点，把 `current` 加入答案
3. 否则把左右孩子和当前数字一起入栈

核心代码：

```python
stack = [(root, 0)]
ans = 0

while stack:
    node, current = stack.pop()
    current = current * 10 + node.val

    if node.left is None and node.right is None:
        ans += current
        continue

    if node.right:
        stack.append((node.right, current))
    if node.left:
        stack.append((node.left, current))
```

### 什么时候写迭代版

如果面试官追问：

```text
能不能不用递归？
```

就写这一版。

默认情况下，递归 DFS 更自然。

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(h)`，栈空间和树高相关

---

## 面试推荐

第 129 题最适合面试的方法是：

```text
DFS 递归累积数字
```

面试时可以这样讲：

```text
我用 DFS 从根节点走到叶子节点。
递归函数携带 current，表示走到当前节点之前已经形成的数字。
进入当前节点时，更新 current = current * 10 + node.val。
如果当前节点是叶子，说明一条完整路径结束，直接返回 current。
如果不是叶子，就递归计算左右子树产生的数字和，并把二者相加返回。
空节点返回 0。
```

复杂度：

```text
每个节点访问一次，时间复杂度 O(n)。
递归栈深度等于树高，空间复杂度 O(h)。
```

## 和 112 / 113 的关系

这三题都是根到叶路径题。

可以这样记：

```text
112：有没有一条路径满足目标和 -> DFS + remain
113：返回所有满足目标和的路径 -> DFS + remain + path
129：把每条路径当数字求和 -> DFS + current number
```

它们的共同点是：

```text
必须到叶子节点才判断或收集结果。
```
