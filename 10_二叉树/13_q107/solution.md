# LeetCode 107 - 二叉树的层序遍历 II（Binary Tree Level Order Traversal II）

## 题目

给你一棵二叉树的根节点 `root`，返回它自底向上的层序遍历结果。

也就是说：

```text
先按层序遍历得到每一层
最后从底层到顶层返回
```

例如：

```text
      3
     / \
    9  20
       / \
      15  7
```

普通层序遍历是：

```text
[
  [3],
  [9, 20],
  [15, 7]
]
```

第 107 题要返回：

```text
[
  [15, 7],
  [9, 20],
  [3]
]
```

---

## 先说结论

第 107 题是第 102 题的直接变体。

第 102 题是：

```text
从上到下返回层序遍历
```

第 107 题是：

```text
从下到上返回层序遍历
```

从第一性原理出发，真正自然的解法是：

1. BFS 层序遍历后反转结果，**最适合面试**
2. BFS + deque 头插，每层直接放到答案前面
3. DFS 按深度分组后反转，作为补充

这题不需要额外凑一个“暴力枚举路径”的方法。

因为题目要的是按层返回结果，最朴素的想法就是：

```text
先把每一层收集出来，再把层的顺序反过来。
```

这个朴素想法本身已经是 `O(n)`，也就是最优时间复杂度。

最推荐的写法是：

```text
先完全复用第 102 题模板，再把结果反转。
```

这比一开始就想各种技巧更稳。

---

## 这题本质是什么

第 107 题并不是一种全新的遍历。

它本质上还是层序遍历，只是最终答案顺序反过来。

所以可以分成两步：

1. 按第 102 题的方法得到自顶向下层序结果
2. 把层列表反转

也就是：

```python
ans.reverse()
return ans
```

或者：

```python
return ans[::-1]
```

---

## 最容易错的地方

### 1. 误以为节点内部也要反转

第 107 题只反转“层的顺序”。

每一层内部仍然是从左到右。

比如：

```text
      3
     / \
    9  20
       / \
      15  7
```

最后一层应该是：

```text
[15, 7]
```

不是：

```text
[7, 15]
```

所以不能把每一层也倒过来。

### 2. 忘记先固定当前层长度

和第 102 题一样，BFS 分层时必须先写：

```python
level_size = len(queue)
```

否则当前层和下一层会混在一起。

### 3. 空树返回空列表

如果 `root is None`，返回：

```text
[]
```

## 这是最适合面试的方法

如果面试里写第 107 题，我最推荐写这一版。

原因是：

1. 它直接复用第 102 题 BFS 模板
2. 逻辑最清楚：先正常层序，再反转层顺序
3. 代码稳定，现场不容易出错
4. 面试官一眼能看懂你的意图

一句话总结：

```text
先按 102 得到 [[3], [9, 20], [15, 7]]，再反转成 [[15, 7], [9, 20], [3]]。
```

---

### 从第 102 题推过来

第 102 题 BFS 模板是：

```python
ans = []
queue = deque([root])

while queue:
    level = []
    level_size = len(queue)

    for _ in range(level_size):
        node = queue.popleft()
        level.append(node.val)

        if node.left:
            queue.append(node.left)
        if node.right:
            queue.append(node.right)

    ans.append(level)
```

这会得到自顶向下的层序结果。

第 107 题只需要最后多一步：

```python
ans.reverse()
```

然后返回 `ans`。

---

### 为什么这版最适合面试

这版没有隐藏技巧。

它清楚表达了：

```text
我要先完成层序遍历，再改变层的顺序。
```

第 107 题本质就是第 102 题的输出顺序变形。

所以先写出 102，再反转，是最符合思维路径的写法。

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次；反转层列表最多 `O(h)`
- 空间复杂度：`O(w)`，队列最多保存一层节点，不计返回结果

## 解法二：BFS + deque 头插

对应文件：

```text
10_二叉树/13_q107/solution_2_bfs_deque.py
```

### 思路

既然最后要把层顺序反过来，也可以在遍历时就把每一层插到答案最前面。

用 `deque`：

```python
ans.appendleft(level)
```

这样最后就不用再 `reverse()`。

核心区别只是：

```python
ans.append(level)
```

变成：

```python
ans.appendleft(level)
```

最后返回：

```python
list(ans)
```

### 是否更优

它避免了最后一次反转，但实际差别不大。

因为反转的是“层数”，不是节点数。

面试里我仍然更推荐第一版，因为更直观。

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(w)`，不计返回结果

## 解法三：DFS 按深度分组后反转

对应文件：

```text
10_二叉树/13_q107/solution_3_dfs.py
```

### 思路

和第 102 题一样，也可以用 DFS：

```python
dfs(node, depth)
```

把每个节点放入：

```python
ans[depth]
```

最后再反转 `ans`。

这说明：

```text
层 = depth
```

但因为题目是层序遍历，BFS 仍然更自然。

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`，递归调用栈高度，不计返回结果

## 面试推荐

第 107 题最适合面试的方法是：

```text
BFS 层序遍历后反转
```

面试时可以这样讲：

```text
这题是第 102 题的变体。
我先用 BFS 队列做普通层序遍历。
每一轮 while 代表一层，先用 level_size = len(queue) 固定当前层节点数。
处理完一层后，把 level 加入 ans。
最后把 ans 反转，就得到自底向上的层序结果。
注意只反转层的顺序，不反转每一层内部节点的顺序。
```

复杂度：

```text
每个节点访问一次，时间复杂度 O(n)。
队列最多保存一层节点，空间复杂度 O(w)，不计返回结果。
```

## 和第 102 题的关系

可以直接记成：

```text
107 = 102 + reverse(ans)
```

但注意：

```text
reverse 的是 ans 这个“层列表”，不是每个 level。
```

```text
103 = 102 + 每一层方向交替
```
