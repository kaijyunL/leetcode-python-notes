# LeetCode 104 - 二叉树的最大深度（Maximum Depth of Binary Tree）

## 题目

给定一个二叉树的根节点 `root`，返回它的最大深度。

最大深度是指：

```text
从根节点到最远叶子节点的最长路径上的节点数
```

例如：

```text
      3
     / \
    9  20
       / \
      15  7
```

最大深度是：

```text
3
```

因为最长路径可以是：

```text
3 -> 20 -> 15
```

或者：

```text
3 -> 20 -> 7
```

路径上有 3 个节点。

---

## 先说结论

这题不需要为了“从简单暴力到最优”而硬凑很多方法。

从第一性原理出发，真正自然的解法只有两个：

1. 递归 DFS：把问题拆成左子树最大深度和右子树最大深度，**最适合面试**
2. BFS 层序遍历：一层一层数高度，适合作为迭代补充

这题没有比 `O(n)` 更低的时间复杂度。

原因是：

```text
你想知道整棵树最深到哪里，最坏情况下必须看完所有节点。
```

所以时间复杂度的下界就是 `O(n)`。

---

## 这题本质是什么

最大深度问的是：

```text
从当前节点往下走，最多能走多少层？
```

如果当前节点是空：

```text
深度 = 0
```

如果当前节点不是空：

```text
当前树的最大深度 = 1 + max(左子树最大深度, 右子树最大深度)
```

这就是本题最核心的递推关系。

写成公式就是：

```text
maxDepth(root) = 0                                      root 是空
maxDepth(root) = 1 + max(maxDepth(root.left),
                         maxDepth(root.right))          root 非空
```

只要你真正理解这个公式，代码就很自然。

---

## “暴力”思路是什么

如果完全从直觉出发，你可能会这样想：

```text
把所有从根到叶子的路径都找出来
看看哪条路径最长
```

比如：

```text
3 -> 9
3 -> 20 -> 15
3 -> 20 -> 7
```

最长路径长度是 3。

这个想法没有错，但没有必要真的把所有路径都保存下来。

因为题目只要最大深度，不要具体路径。

所以我们只需要在递归返回时保留一个数字：

```text
这棵子树的最大深度
```

这一步就是从朴素暴力想法走向最优递归解法的关键。

## 解法一览

| 解法 | 思路 | 时间复杂度 | 空间复杂度 | 是否推荐面试 |
|---|---|---|---|---|
| 1. 递归 DFS | 当前深度 = 1 + 左右子树最大深度 | `O(n)` | `O(h)` | **最适合面试 ✅** |
| 2. BFS 层序遍历 | 一层一层遍历，遍历完一层深度加 1 | `O(n)` | `O(w)` | 迭代补充 |

说明：

- `n` 是节点总数
- `h` 是树高
- `w` 是二叉树某一层的最大宽度
- 空间复杂度不计算输入树本身

## 这是最适合面试的方法

第 104 题面试时最推荐先写递归 DFS。

原因很简单：

1. 它直接表达了题目的数学定义
2. 代码非常短，不容易写错
3. 时间复杂度已经是最优的 `O(n)`
4. 它能体现你理解树问题的递归结构

一句话总结：

```text
二叉树最大深度，本质就是左右子树最大深度再加当前这一层。
```

---

### 从第一性原理推导

站在任意一个节点 `node` 上想：

```text
以 node 为根的树有多深？
```

如果 `node` 是空：

```text
这棵树没有节点，所以深度是 0。
```

如果 `node` 不是空：

```text
这棵树至少有当前节点这一层，所以先有 1。
```

然后它下面可能有左子树，也可能有右子树。

因为要的是“最大深度”，所以应该选择更深的那边：

```text
1 + max(左子树深度, 右子树深度)
```

代码就是：

```python
if root is None:
    return 0

left_depth = self.maxDepth(root.left)
right_depth = self.maxDepth(root.right)
return 1 + max(left_depth, right_depth)
```

---

### 为什么这已经是最优

每个节点只会被访问一次。

对每个节点做的事情也很少：

```text
计算左深度
计算右深度
取最大值
加 1
```

所以时间复杂度是：

```text
O(n)
```

而这题最坏情况下必须看完所有节点，因此 `O(n)` 已经不能再优化。

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(h)`，递归调用栈高度

其中：

- 平衡树：`h ≈ log n`
- 链式退化树：`h = n`

## 解法 2：BFS 层序遍历

对应文件：

```text
10_二叉树/04_q104/solution_2_bfs.py
```

### 思路

最大深度也可以理解成：

```text
这棵树一共有多少层？
```

既然是数层数，就可以用 BFS。

每次处理队列里当前这一整层的节点：

1. 当前层处理前，`depth += 1`
2. 把这一层所有节点的孩子加入队列
3. 队列不空就继续下一层

代码骨架：

```python
queue = deque([root])
depth = 0

while queue:
    depth += 1
    for _ in range(len(queue)):
        node = queue.popleft()
        if node.left:
            queue.append(node.left)
        if node.right:
            queue.append(node.right)
```

### 什么时候用 BFS

如果题目强调：

```text
按层处理
最短路径
每层信息
层序遍历
```

BFS 通常更自然。

但第 104 题只是问最大深度，递归 DFS 更直接。

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(w)`，队列最多保存一层节点

`w` 是树的最大宽度。对于满二叉树，最底层宽度可能接近 `n / 2`。

## 面试推荐

第 104 题最适合面试的方法是：

```text
递归 DFS
```

面试时可以这样讲：

```text
空树深度是 0。
对于非空节点，以它为根的最大深度等于：
1 + 左右子树最大深度中的较大值。
所以递归计算左子树深度和右子树深度，最后返回 1 + max(left, right)。
每个节点只访问一次，时间复杂度 O(n)。
递归栈深度等于树高，空间复杂度 O(h)。
```

如果面试官追问非递归，再给 BFS 层序遍历：

```text
用队列一层一层遍历，每处理完一层 depth 加 1。
最后 depth 就是最大深度。
```

```text

```
