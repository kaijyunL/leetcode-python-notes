# LeetCode 94 - 二叉树的中序遍历（Binary Tree Inorder Traversal）

## 题目

给你一棵二叉树的根节点 `root`，返回它的中序遍历结果。

中序遍历顺序是：

```text
左 -> 根 -> 右
```

例如这棵树：

```text
    1
     \
      2
     /
    3
```

中序遍历结果是：

```text
[1, 3, 2]
```

---

## 先说结论

这题不需要为了“从简单到最优”而硬凑一个很笨的暴力法。

从第一性原理出发，这题最自然的层次就是：

1. 递归版
2. 显式栈迭代版
3. Morris 版

原因很简单：

```text
中序遍历本身就是一个递归定义
```

所以第一版直接写递归最自然；
然后再把系统递归栈改成手动维护的栈；
最后如果还想继续压缩空间，再看 Morris。

也就是说，这题真正值得掌握的不是“花样”，而是：

```text
中序遍历到底为什么不能像前序那样一见到节点就访问
```

这才是核心。

---

## 这题本质是什么

中序遍历本质上只做一件事：

```text
每个节点都要在“左子树处理完之后、右子树处理之前”被访问
```

这句话非常重要。

因为它决定了三种遍历的根本区别：

- 前序：`根 -> 左 -> 右`，一见到节点就访问
- 中序：`左 -> 根 -> 右`，左边没处理完，当前节点不能访问
- 后序：`左 -> 右 -> 根`，左右都没处理完，当前节点更不能访问

所以第 94 题的难点，不是树本身，而是：

```text
当你还不能访问当前节点时，怎么先把它“记住”，等左边处理完再回来访问它
```

递归里，这件事由系统调用栈完成。
迭代里，就得我们自己用栈完成。

---

## 中序遍历的定义

对于任意节点 `root`：

```text
inorder(root) = inorder(root.left) + [root.val] + inorder(root.right)
```

这就是第 94 题最底层的定义。

只要你把这句话真正想明白，后面的代码都只是实现形式不同。

---

## 解法一览

| 解法 | 思路 | 时间复杂度 | 空间复杂度 | 是否推荐面试 |
|---|---|---|---|---|
| 1. 递归 | 直接按中序定义写 | `O(n)` | `O(h)` | 适合理解 |
| 2. 显式栈迭代 | 手动模拟递归过程 | `O(n)` | `O(h)` | **最适合面试 ✅** |
| 3. Morris | 利用线索化思想做到 `O(1)` 额外空间 | `O(n)` | `O(1)` | 进阶加分 |

说明：

- `n` 是节点总数
- `h` 是树高
- 平衡树时 `h ≈ log n`
- 极端退化成链表时 `h = n`

## 解法 1：递归

对应文件：

```text
10_二叉树/02_q94/solution_1_recursive.py
```

### 思路

递归是这题最自然的第一版。

因为它几乎就是把定义直接翻译成代码：

1. 先递归左子树
2. 再记录当前节点
3. 最后递归右子树

代码骨架就是：

```python
def dfs(node):
    if node is None:
        return

    dfs(node.left)
    ans.append(node.val)
    dfs(node.right)
```

### 为什么它适合作为第一版

因为你在写的不是模板，而是在执行中序遍历本身的定义。

它能帮助你真正理解：

```text
为什么当前节点不是一开始就访问，
而必须等左子树处理完之后再访问。
```

### 优点

1. 最符合定义
2. 最容易写对
3. 最适合理解树遍历的本质

### 缺点

1. 依赖系统递归栈
2. 如果面试官要求非递归，这版不够
3. 继续追问时，通常还是要落到迭代版

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(h)`，来自递归调用栈


## 这是最适合面试的方法

如果面试里让我写第 94 题，我会优先写这版。

原因很简单：

1. 它不依赖系统递归栈
2. 它清楚体现了你理解“递归的底层其实就是栈”
3. 代码结构稳定，现场手写不容易翻车
4. 相比 Morris，更符合大多数正常面试场景

一句话总结：

```text
递归版最直观，栈迭代版最稳，Morris 版最适合追求空间最优。
```

---

### 从递归推到栈，核心怎么想

递归版在做什么？

对于任意节点 `node`：

```text
先处理左子树
回来访问 node
再处理右子树
```

这里最关键的是“回来访问 node”。

因为你一路往左走时，这些节点还不能立刻进答案，但它们之后必须回来访问。

所以你需要一个地方，临时把这些节点存起来。

这个地方最合适的数据结构就是：

```text
栈
```

这就是这题迭代版的第一性原理。

---

### 迭代版到底在做什么

整个过程可以压缩成 3 句话：

1. 一路向左，把沿途节点全部压栈
2. 左边走到底后，弹出栈顶并访问它
3. 再转向它的右子树，重复同样过程

把这 3 句话吃透，第 94 题就真的懂了。

---

### 为什么一定是“一路向左压栈”

因为中序顺序是：

```text
左 -> 根 -> 右
```

所以在访问某个节点之前，必须先把它左边能到的节点全部处理完。

而这些沿途经过、但暂时还不能访问的节点，就应该先压进栈里保存。

换句话说：

- 往左走：是在不断深入，寻找当前最先该访问的节点
- 压栈：是在保存“这些节点以后还要回来访问”
- 弹栈访问：说明它的左子树已经处理完了，现在终于轮到它自己

---

### 过程演示

以这棵树为例：

```text
    1
     \
      2
     /
    3
```

初始：

```text
cur = 1
stack = []
ans = []
```

#### 第 1 步：一路向左压栈

`1` 没有左子节点，但它自己还不能立刻丢掉，所以先压栈：

```text
stack = [1]
cur = None
```

#### 第 2 步：左边走不动了，弹栈访问

弹出 `1`，说明它左边已经处理完：

```text
ans = [1]
```

然后转去它的右子树：

```text
cur = 2
```

#### 第 3 步：继续一路向左

先压 `2`：

```text
stack = [2]
cur = 3
```

再压 `3`：

```text
stack = [2, 3]
cur = None
```

#### 第 4 步：开始回退访问

弹出 `3`，访问：

```text
ans = [1, 3]
```

`3` 没有右子树，继续弹。

再弹出 `2`，访问：

```text
ans = [1, 3, 2]
```

这就是最终答案。

---

### 最推荐背下来的代码骨架

```python
stack = []
cur = root

while cur is not None or stack:
    while cur is not None:
        stack.append(cur)
        cur = cur.left

    cur = stack.pop()
    ans.append(cur.val)
    cur = cur.right
```

这段代码最关键的不是形式，而是这三步：

1. 向左压栈
2. 弹栈访问
3. 转向右边

---

### 面试时怎么讲最顺

你可以按这个顺序讲：

1. 中序遍历是左根右，所以当前节点不能一见到就访问
2. 递归版里，系统栈会帮我记住这些以后还要回来的节点
3. 如果改成迭代，我就自己用一个栈做同样的事
4. 我会一路往左，把沿途节点压栈
5. 左边走到底后，弹出栈顶访问它，再转去它的右子树
6. 这样每个节点恰好入栈出栈一次，所以时间复杂度是 `O(n)`

这套说法很完整，也能体现你是真的想明白了。

---

### 常见错误

#### 错误 1：把中序写成前序

如果你一拿到节点就立刻记：

```python
ans.append(cur.val)
```

那你写出来的其实更像前序，而不是中序。

---

#### 错误 2：忘了转向右子树

很多人写完：

```python
cur = stack.pop()
ans.append(cur.val)
```

就结束了，但真正下一步还必须：

```python
cur = cur.right
```

否则右子树根本不会被遍历到。

---

#### 错误 3：外层循环条件写错

必须写成：

```python
while cur is not None or stack:
```

因为：

- `cur` 负责当前正在往下走的路径
- `stack` 负责保存以后要回来的节点

只要两者还有一个非空，遍历就还没结束。

---

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(h)`；最坏链式树时可到 `O(n)`

## 解法 3：Morris 中序遍历

对应文件：

```text
10_二叉树/02_q94/solution_3_morris.py
```

### 这版在优化什么

递归和显式栈，本质上都在用额外空间记“回来的路”。

Morris 的思路是：

```text
既然中序遍历完左子树后还要回到当前节点，
那我能不能临时在树里造一条返回路径？
```

答案是可以。

它会找到当前节点左子树里最右边的节点 `pred`，临时把：

```python
pred.right = cur
```

这样左子树遍历完后，就能顺着这条线索回到当前节点。

等回来的时候，再把这条线索恢复成 `None`。

---

### 中序版的关键逻辑

对于当前节点 `cur`：

#### 情况 1：没有左子树

说明可以直接访问自己：

```python
ans.append(cur.val)
cur = cur.right
```

#### 情况 2：有左子树

找到左子树中最右边的节点 `pred`。

如果：

```python
pred.right is None
```

说明第一次到这里，还没建线索。

这时：

1. 建立 `pred.right = cur`
2. 往左走：`cur = cur.left`

注意，这一步还不能访问 `cur`，因为左子树还没遍历完。

如果：

```python
pred.right is cur
```

说明左子树已经遍历完了，这是顺着线索回来的。

这时才：

1. 恢复 `pred.right = None`
2. 访问 `cur`
3. 转去 `cur.right`

---

### 为什么它不是面试首选

因为它虽然空间最优，但现场更容易写错：

1. 容易把访问时机写错
2. 容易忘记恢复指针
3. 容易把中序 Morris 和前序 Morris 混掉

所以更稳的策略通常是：

1. 先写显式栈迭代版
2. 如果面试官继续追问空间还能不能优化，再补 Morris

---

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(1)`，不算结果数组

---
