# LeetCode 199 - 二叉树的右视图（Binary Tree Right Side View）

## 题目

给你一棵二叉树的根节点 `root`，想象自己站在树的右侧。

返回从右侧能看到的节点值，按照从上到下的顺序。

例如：

```text
      1
     / \
    2   3
     \   \
      5   4
```

从右边看，能看到：

```text
[1, 3, 4]
```

因为：

```text
第 1 层最右边是 1
第 2 层最右边是 3
第 3 层最右边是 4
```

---

## 先说结论

第 199 题也是层序系列题。

它和第 102 题的关系是：

```text
102：返回每一层所有节点
199：只返回每一层最右边的节点
```

从第一性原理出发，真正自然的解法是：

1. 朴素 BFS：完整收集每一层，再取这一层最后一个值
2. BFS 优化：分层遍历时只记录每层最后一个节点，**最适合面试**
3. DFS 右优先：每层第一次访问到的节点就是右视图节点

这题不需要为了凑方法写很绕的东西。

因为题目的本质就是：

```text
每一层只要一个节点：最右边那个。
```

---

## 这题本质是什么

站在右边看一棵树，并不是说只能走右孩子。

关键是：

```text
每一层最右侧能挡住后面节点的那个节点
```

比如：

```text
      1
     /
    2
   /
  3
```

虽然整棵树没有右孩子，但右视图仍然是：

```text
[1, 2, 3]
```

所以不能简单写成：

```text
一直往右走
```

正确理解是：

```text
每一层取最右边的节点。
```

---

## 最容易错的地方

### 1. 误以为只看右子树

错误思路：

```text
root -> root.right -> root.right.right
```

这个思路会漏掉没有右孩子但仍然能从右边看到的节点。

例如：

```text
    1
   /
  2
```

右视图是：

```text
[1, 2]
```

不是：

```text
[1]
```

### 2. 忘记按层处理

右视图的单位是“层”。

所以 BFS 写法里仍然要先固定当前层长度：

```python
level_size = len(queue)
```

然后在这一层内部判断谁是最后一个节点。

### 3. DFS 写法必须右子树先走

DFS 也可以做，但必须先访问右子树：

```python
dfs(node.right, depth + 1)
dfs(node.left, depth + 1)
```

这样每一层第一次遇到的节点，才是从右边看到的节点。

---

## 解法一：朴素 BFS，完整收集每一层

对应文件：

```text
10_二叉树/15_q199/solution_1_bfs_level_values.py
```

### 思路

最直观的想法是：

```text
既然我要每层最右边，那我先把每一层完整拿出来。
```

比如普通层序遍历得到：

```text
[
  [1],
  [2, 3],
  [5, 4]
]
```

每一层最后一个值就是：

```text
[1, 3, 4]
```

也就是：

```python
ans.append(level[-1])
```

### 为什么它不是面试最优写法

这个方法完全正确，也很好理解。

但它会保存当前层所有节点值：

```python
level = []
```

而第 199 题其实只需要当前层最后一个值。

所以可以再优化一步：不保存整层列表，只在每层最后一个节点出现时加入答案。

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(w)`，队列最多保存一层左右的节点，不计返回结果

## 解法二：BFS 分层，只记录每层最后一个节点

对应文件：

```text
10_二叉树/15_q199/solution_2_bfs_last_node.py
```

## 这是最适合面试的方法

如果面试里写第 199 题，我最推荐这一版。

原因是：

1. 它直接复用第 102 题 BFS 分层模板
2. 它准确表达“每层取最右节点”这个题意
3. 它不需要额外保存整层 `level`
4. 代码非常稳定，现场手写不容易出错

一句话总结：

```text
BFS 一层一层遍历；每一层中最后弹出的节点，就是这一层的右视图节点。
```

---

### 从第一性原理推导

队列初始只有根节点：

```python
queue = deque([root])
```

每一轮 `while` 开始时，队列里正好是当前层的所有节点。

先固定当前层数量：

```python
level_size = len(queue)
```

然后处理这一层的 `level_size` 个节点：

```python
for i in range(level_size):
    node = queue.popleft()
```

因为节点按从左到右出队，所以：

```text
i == level_size - 1
```

时，当前节点就是这一层最右边的节点。

于是加入答案：

```python
if i == level_size - 1:
    ans.append(node.val)
```

这一步建议紧跟在 `node = queue.popleft()` 后面写。

这样和前几道层序题保持同一个模板：

```text
弹出当前节点
处理当前节点
加入左右孩子
```

孩子仍然按左到右入队：

```python
if node.left:
    queue.append(node.left)
if node.right:
    queue.append(node.right)
```

这样下一层的节点顺序仍然是自然的从左到右。

---

### 为什么这版最适合面试

这版把三个关键点都写得很明确：

```text
1. 层序遍历：用 queue
2. 分层：用 level_size
3. 右视图：每层最后一个节点
```

它不像 DFS 右优先那样需要额外解释“为什么第一次到某层就是答案”。

对面试官来说，这版更直接。

如果你已经刷完 102、107、103，那么第 199 题可以直接记成：

```text
199 = 102 的 BFS 模板 + 每层只取最后一个节点
```

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(w)`，队列最多保存一层左右的节点，不计返回结果

## 解法三：DFS 右优先

对应文件：

```text
10_二叉树/15_q199/solution_3_dfs_right_first.py
```

### 思路

DFS 也能做右视图。

关键是访问顺序：

```text
根节点 -> 右子树 -> 左子树
```

如果我们先走右边，那么每一层第一次遇到的节点，就是这一层最右边能看到的节点。

定义：

```python
dfs(node, depth)
```

其中 `depth` 表示当前节点所在层数。

如果：

```python
depth == len(ans)
```

说明这一层还没有放过节点。

因为我们是右优先遍历，所以这个节点就是这一层右视图节点：

```python
ans.append(node.val)
```

然后先递归右子树，再递归左子树：

```python
dfs(node.right, depth + 1)
dfs(node.left, depth + 1)
```

### 什么时候写 DFS 版

DFS 版很优雅，也经常出现在题解里。

如果面试官问：

```text
能不能不用 BFS？
```

就写这一版。

但默认主写我仍然推荐 BFS，因为第 199 题本质是“每层取一个节点”，层序思路更自然。

### 复杂度

- 时间复杂度：`O(n)`，每个节点访问一次
- 空间复杂度：`O(h)`，递归调用栈高度，不计返回结果

## 面试推荐

第 199 题最适合面试的方法是：

```text
BFS 分层，只记录每层最后一个节点
```

面试时可以这样讲：

```text
右视图本质上是每一层最右边的节点。
我用 BFS 队列做层序遍历。
每一轮 while 代表一层，先用 level_size = len(queue) 固定当前层节点数。
因为队列中当前层节点是从左到右弹出的，
所以这一层最后一次弹出的节点就是最右节点。
当 i == level_size - 1 时，把 node.val 加入答案。
孩子仍然按 left -> right 入队，保证下一层顺序稳定。
```

复杂度：

```text
每个节点访问一次，所以时间复杂度 O(n)。
队列最多保存一层左右的节点，所以空间复杂度 O(w)，不计返回结果。
```

## 和第 102、103、107 题的关系

可以这样记：

```text
102：返回每一层所有节点
107：返回每一层所有节点，但层顺序反过来
103：返回每一层所有节点，但层内方向交替
199：每一层只返回最右边那个节点
```

共同核心仍然是：

```python
level_size = len(queue)
```

区别只在于：

```text
处理完当前层以后，怎么把这一层的信息放进答案。
```
