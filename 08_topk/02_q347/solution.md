# LeetCode 347 — 前 K 个高频元素（Top K Frequent Elements）

## 题目

给定一个非空的整数数组 `nums`，返回出现频率前 `k` 高的元素。可以按任意顺序返回答案。

### 示例

```
输入: nums = [1,1,1,2,2,3], k = 2
输出: [1, 2]
解释: 1 出现 3 次，2 出现 2 次，3 出现 1 次
```

```
输入: nums = [1], k = 1
输出: [1]
```

---

## 解法一览

| 解法 | 时间 | 空间 | 是否推荐面试 |
|---|---|---|---|
| 1. 哈希+排序 | O(n log n) | O(n) | 可接受，但不最优 |
| 2. 哈希+最小堆 | O(n log k) | O(n + k) | 常规做法 |
| 3. 桶排序 | O(n) | O(n) | **最佳 ✅** |

---

## 解法 1：哈希表 + 排序

**思路：** 用哈希表统计频率，按频率排序，取前 k 个。

Python 中一行搞定统计 + 排序：`Counter(nums).most_common(k)`。

**缺点：** 排序 O(n log n)，不是最优。

---

## 解法 2：哈希表 + 最小堆

**思路：** 先统计频率，再用最小堆维护前 k 个最高频的元素。和解法 215 类似，但比较的是频率而不是值。

**复杂度：** 时间 O(n log k)，空间 O(n + k)。

---

## 解法 3：桶排序 ⭐ 面试推荐

**思路：** 频率范围是 1 到 n（数组长度），创建 n+1 个桶，第 i 个桶存频率为 i 的所有元素。

```
nums = [1, 1, 1, 2, 2, 3], k = 2

统计频率: {1: 3, 2: 2, 3: 1}

桶:
桶[1]: [3]    (频率1)
桶[2]: [2]    (频率2)
桶[3]: [1]    (频率3)

从后往前取前 k 个: [1, 2]
```

**复杂度：** 时间 O(n)，空间 O(n)。

---

## 面试建议

**解法 3（桶排序）是面试最优解。**

面试时这样展开：
1. 先说排序法："统计频率后排序，O(n log n)"
2. 再提堆法："维护 k 个最高频，O(n log k)"
3. 最后给出桶排序："频率范围有限（1~n），用桶排序可以 O(n)"
4. 注意提到：同一频率可能有多个元素，桶内是列表
