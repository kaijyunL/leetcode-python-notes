# LeetCode 692 — 前K个高频单词（Top K Frequent Words）

## 题目

给定一个单词列表 `words` 和一个整数 `k`，请返回出现频率前 `k` 高的单词。

返回结果按以下规则排序：
1. 频率高的排前面
2. 频率相同时，按**字典序**排序

### 示例

```
输入: words = ["i", "love", "leetcode", "i", "love", "coding"], k = 2
输出: ["i", "love"]
解释: "i" 和 "love" 都出现 2 次，频率相同，按字典序 "i" < "love"
```

```
输入: words = ["the", "day", "is", "sunny", "the", "the", "the", "sunny", "is", "is"], k = 4
输出: ["the", "is", "sunny", "day"]
解释: 频率分别为 4, 3, 2, 1
```

---

## 解法一览

| 解法 | 时间 | 空间 | 是否推荐面试 |
|---|---|---|---|
| 1. 哈希+排序 | O(n log n) | O(n) | 可接受 |
| 2. 哈希+堆 | O(n log k) | O(n + k) | **推荐 ✅** |

---

## 解法 1：哈希表 + 排序

**思路：** 统计频率后，按"频率降序 + 字典序升序"排序，取前 k 个。

Python 一行：`sorted(freq.items(), key=lambda x: (-x[1], x[0]))[:k]`

注意排序的 key：`(-频率, 单词)`，频率取负是为了降序，单词是默认升序（字典序）。

---

## 解法 2：哈希表 + 堆 ⭐ 面试推荐

**思路：** 维护大小为 k 的最小堆。堆的比较规则是：
- 先比频率，频率小的在堆顶（弹出）
- 频率相同时，字典序大的在堆顶（弹出）

这样堆中始终保留的是"最好的" k 个单词。

最后从堆中取出，需要反转（因为堆顶是最差的）。

---

## 面试建议

**解法 2（堆）是面试推荐答案。**

面试时重点说明：
1. 排序法简单但不是最优
2. 堆法的核心是比较规则：`(-频率, 字典序倒序)`
3. Python 的 `heapq` 是最小堆，所以要"反向"保留前 k 个好的元素
4. 最后记得反转结果
