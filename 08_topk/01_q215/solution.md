# LeetCode 215 — 数组中的第K个最大元素（Kth Largest Element in an Array）

## 题目

给定整数数组 `nums` 和整数 `k`，请返回数组中第 `k` 个最大的元素。请注意，你需要找的是数组排序后的第 `k` 个最大的元素，而不是第 `k` 个不同的元素。

**要求：** 设计并实现时间复杂度为 O(n) 的算法。

### 示例

```
输入: nums = [3, 2, 1, 5, 6, 4], k = 2
输出: 5
解释: 排序后 [1,2,3,4,5,6]，第 2 大是 5
```

```
输入: nums = [3, 2, 3, 1, 2, 4, 5, 5, 6], k = 4
输出: 4
```

---

## 解法一览

| 解法 | 时间 | 空间 | 是否推荐面试 |
|---|---|---|---|
| 1. 排序后取值 | O(n log n) | O(1) | 可接受，但不满足进阶要求 |
| 2. 最小堆（维护 k 个最大） | O(n log k) | O(k) | 常规做法 |
| 3. 快速选择（Quick Select） | O(n) 平均 | O(1) | **最佳 ✅** |

---

## 解法 1：排序后取值

**思路：** 排序后取倒数第 k 个元素。

Python 一行搞定：`return sorted(nums)[-k]` 或 `return nums.sort()` 后取。

**缺点：** O(n log n)，不满足题目 O(n) 的要求。

---

## 解法 2：最小堆（维护 k 个最大元素）

**思路：** 维护一个大小为 k 的最小堆。遍历数组：
- 堆不满时直接加入
- 堆满后，如果当前元素 > 堆顶（堆中最小），则弹出堆顶、加入当前元素
- 最终堆顶就是第 k 大的元素

```
nums = [3, 2, 1, 5, 6, 4], k = 2

遍历:
3 → 堆: [3]
2 → 堆: [2, 3]  (满了，size=2)
1 → 1 < 堆顶2，跳过
5 → 5 > 堆顶2，弹出2，加入5 → 堆: [3, 5]
6 → 6 > 堆顶3，弹出3，加入6 → 堆: [5, 6]
4 → 4 < 堆顶5，跳过

结果: 堆顶 = 5
```

**复杂度：** 时间 O(n log k)，空间 O(k)。

---

## 解法 3：快速选择（Quick Select） ⭐ 面试推荐

**思路：** 快速排序的 partition 思想，但不需要完全排序，只定位第 k 大的位置。

每次 partition 后，pivot 会落在某个位置 `p`：
- 如果 `p` 正好是目标位置 → 找到了
- 如果 `p` 在目标左边 → 只处理右半部分
- 如果 `p` 在目标右边 → 只处理左半部分

这样每次把搜索范围减半，平均时间 O(n)。

### 逐步演示

```
nums = [3, 2, 1, 5, 6, 4], k = 2
目标位置（排序后）: n - k = 6 - 2 = 4

Partition(pivot=3):
[3, 2, 1, 5, 6, 4] → 小于3: [2, 1]  等于3: [3]  大于3: [5, 6, 4]
pivot 落在 index 2
2 < 4 → 去右边 [5, 6, 4] 找

Partition(pivot=5):
[5, 6, 4] → 小于5: [4]  等于5: [5]  大于5: [6]
pivot 落在 index 4
4 == 4 → 找到！返回 5
```

**复杂度：** 平均 O(n)，最坏 O(n²)（随机 pivot 可避免）；空间 O(1)。

---

## 面试建议

**解法 3（快速选择）是面试最优解。**

面试时这样展开：
1. 先说排序法："排序后取，O(n log n)，简单但不满足 O(n)"
2. 再提堆法："维护 k 个最大，O(n log k)，Python 有 heapq 很方便"
3. 最后给出快速选择："借鉴快排的 partition，只处理一半，平均 O(n)"
4. 如果面试官追问最坏情况，提到可以用随机 pivot 或中位数的中位数（BFPRT）
