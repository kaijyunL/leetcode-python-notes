# LeetCode Python Notes

一个以 **Python 刷 LeetCode** 为主的学习型仓库，按专题整理题目，并保留多种解法、题解笔记、复盘记录和少量可视化辅助材料。

这个仓库的目标不是只存“最终答案”，而是尽量保留从朴素思路到优化解法的推导过程，方便后续复习、对比和补坑。

## 仓库内容

- 按专题分类刷题，目前包含数组、字符串、链表、栈和队列、双指针
- 每道题通常会保留一种或多种 Python 解法
- 部分题目附带 Markdown 讲解、思路总结、复盘记录
- 少量题目附带 HTML 可视化页面，帮助理解过程
- 额外包含阶段性学习计划

## 目录结构

```text
leetcode-python-notes/
├── 01_数组
├── 02_字符串
├── 03_链表
├── 04_栈和队列
├── 05_双指针
└── plan
```

每个专题目录下再按题目组织，例如：

```text
01_数组/
└── 05_q169/
    ├── brute_force.py
    ├── hash_map.py
    ├── boyer_moore.py
    └── solution_explanation.md
```

常见文件类型说明：

- `solution.py` / `optimal.py` / `solution_v*.py`：不同版本的 Python 解法
- `brute_force.py`：朴素解法，通常用于和优化方案对比
- `*.md`：题解、总结、复盘、学习笔记
- `*.html`：可视化或交互式讲解页面

## 如何阅读

如果你是第一次看这个仓库，建议按下面的顺序使用：

1. 先按专题进入目录，比如 `03_链表`
2. 再按题号进入对应题目目录，比如 `03_链表/01_q206`
3. 先看 `brute_force.py` 或基础版本，理解原始思路
4. 再看 `optimal.py`、`solution_v2.py`、`solution_v3.py` 等优化版本
5. 最后结合对应的 `*.md` 讲解做总结

## 命名规则

- 顶层专题目录使用 `序号_专题名`，例如 `01_数组`
- 题目目录通常使用 `序号_q题号`，例如 `05_q169`
- 如果同一道题保留多种方案，会用 `solution_v1`、`solution_v2`、`optimal`、`brute_force` 等方式区分

## 学习计划

`plan/` 目录下保存阶段性的刷题计划，例如：

- `plan/leetcode_study_planA.md`
- `plan/leetcode_study_planB.md`

## 说明

- 这个仓库当前更偏向个人学习整理，而不是统一封装的算法模板库
- 代码风格会随着刷题阶段和理解深度逐步演进，因此不同目录下的写法可能不完全一致
- 部分题目保留了“从错误思路到正确思路”的痕迹，这是刻意保留的学习过程
