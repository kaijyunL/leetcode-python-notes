# LeetCode 169. 多数元素 (Majority Element) 循序渐进解析

## 题目描述
给定一个大小为 `n` 的数组 `nums` ，返回其中的多数元素。多数元素是指在数组中出现次数 **大于** `⌊ n/2 ⌋` 的元素。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

---

## 1. 暴力解法 (Brute Force)
### 思路
遍历数组中的每个元素，统计它在数组中出现的次数。如果某个元素的次数大于 `n/2`，则它就是多数元素。

### 复杂度分析
- **时间复杂度**: $O(n^2)$。两层嵌套循环，外层遍历数组，内层统计出现次数。
- **空间复杂度**: $O(1)$。只使用了常数级别的额外空间。

---

## 2. 哈希表法 (Hash Map)
### 思路
利用哈希表（在 Python 中是 `dict`）来存储数组中每个元素及其出现的次数。遍历数组并更新哈希表，当发现某个元素的计数值超过 `n/2` 时，直接返回结果。

### 复杂度分析
- **时间复杂度**: $O(n)$。我们只遍历了一遍数组，哈希表的插入和查找操作平均时间复杂度为 $O(1)$。
- **空间复杂度**: $O(n)$。最坏情况下，数组中有一半不重复的元素，哈希表需要存储这些元素。

---

## 3. 排序法 (Sorting)
### 思路
因为题目保证多数元素出现的次数大于 `n/2`，那么如果我们将数组排序，**位于数组中间位置的元素一定是多数元素**。
无论多数元素是最小的还是最大的，或者分布在中间，排序后它必然会跨越下标为 `n/2` 的位置。

### 复杂度分析
- **时间复杂度**: $O(n \log n)$。排序操作的平均时间复杂度。
- **空间复杂度**: $O(1)$ 或 $O(n)$。取决于所使用的排序算法。Python 的 `sort()` (Timsort) 空间复杂度为 $O(n)$。

---

## 4. 摩尔投票法 (Boyer-Moore Voting Algorithm) - 最优解
### 思路
这是一道非常巧妙的算法。我们可以形象地把它理解为“同归于尽”：
1. 我们维护一个候选人 `candidate` 和一个计数器 `count`。
2. 遍历数组：
   - 如果 `count` 为 0，我们将当前元素设为 `candidate`。
   - 如果当前元素等于 `candidate`，`count` 加 1。
   - 如果当前元素不等于 `candidate`，`count` 减 1（代表通过抵消掉一个非多数元素来消耗多数元素）。
3. 由于多数元素的数量大于 `n/2`，它在抵消之后一定还能剩下。

### 复杂度分析
- **时间复杂度**: $O(n)$。只需线性扫描一遍数组。
- **空间复杂度**: $O(1)$。只使用了两个变量来存储当前候选人和计数值。
