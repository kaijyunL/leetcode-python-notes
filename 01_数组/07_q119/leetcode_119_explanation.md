# LeetCode 119. 杨辉三角 II (Pascal's Triangle II)

杨辉三角（Pascal's Triangle）是一个经典的数学模型。第 $n$ 行的第 $k$ 个数（从 0 开始计数）对应于组合数 $C(n, k)$。

在这道题中，给定一个非负索引 `rowIndex`，我们需要返回杨辉三角中该索引对应的行。

---

## 方法一：生成所有行 (基础动态规划)

### 思路
最直观的方法是模拟杨辉三角的生成过程。从第 0 行开始，逐行计算，直到达到 `rowIndex`。
每一行（除了首尾的 1）的每个数字都是由它上一行的两个相邻数字相加得到的。

### 复杂度分析
- **时间复杂度**: $O(n^2)$，其中 $n$ 是 `rowIndex`。我们需要计算总共约 $n(n+1)/2$ 个元素。
- **空间复杂度**: $O(n^2)$，因为我们存储了所有已经生成的行。

## 方法二：空间优化 (原地更新)

### 思路
在方法一中，我们发现计算某一行时只需要上一行的信息。因此，我们不需要存储所有行。
我们可以只使用一个长度为 `rowIndex + 1` 的数组。为了防止计算时覆盖掉还需要使用的旧值，我们采用**从后往前**更新的策略。

例如计算第 3 行 `[1, 3, 3, 1]` 时：
1. 初始为第 2 行：`[1, 2, 1, 0]`
2. 从后往前：
   - `res[3] = res[3] + res[2]` (即 0 + 1 = 1)
   - `res[2] = res[2] + res[1]` (即 1 + 2 = 3)
   - `res[1] = res[1] + res[0]` (即 2 + 1 = 3)
   - `res[0]` 保持不变为 1。

### 复杂度分析
- **时间复杂度**: $O(n^2)$。
- **空间复杂度**: $O(n)$，只使用了输出所需的空间。

## 方法三：线性时间 (数学公式法)

### 思路
利用组合数公式 $C(n, k) = \binom{n}{k} = \frac{n!}{k!(n-k)!}$。
我们可以推导出相邻两个组合数的关系：
$$C(n, k) = C(n, k-1) \times \frac{n - k + 1}{k}$$
这样我们就可以在 $O(n)$ 时间内直接算出这一行的所有元素。

### 复杂度分析
- **时间复杂度**: $O(n)$。
- **空间复杂度**: $O(n)$ (不计入返回数组则为 $O(1)$)。
