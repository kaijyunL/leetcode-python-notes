# LeetCode 第 41 题：缺失的第一个正数 (First Missing Positive)

## 1. 题目描述
给你一个未排序的整数数组 `nums` ，请你找出其中没有出现的最小的正整数。

**要求：**
- 时间复杂度：$O(n)$
- 辅助空间：$O(1)$

---

## 2. 循序渐进法

### 方法一：排序法 (Sorting)
这是最直观的方法。我们将数组排序，然后从 1 开始寻找缺失的数字。

**思路：**
1. 先对数组进行升序排序。
2. 定义一个变量 `res = 1`，表示我们期望找到的下一个最小正整数。
3. 遍历数组，如果遇到当前遍历的数 `num == res`，说明 `res` 出现了，我们将 `res` 加 1（寻找下一个目标）。
4. 注意：要过滤掉小于 1 的数和重复的数。

**代码实现：** `solution_v1_sort.py`

**复杂度分析：**
- **时间复杂度：** $O(n \log n)$，主要取决于排序。
- **辅助空间：** $O(1)$ 或 $O(n)$，取决于排序算法。

---

### 方法二：哈希表法 (Hash Set)
为了提升查找效率，我们可以使用哈希表。

**思路：**
1. 将数组中所有的数存入一个哈希集合 (Set) 中。
2. 从正整数 1 开始，依次检查 1, 2, 3... 是否在集合中。
3. 第一个不在集合中的正整数即为结果。

**代码实现：** `solution_v2_set.py`

**复杂度分析：**
- **时间复杂度：** $O(n)$，存入集合 $O(n)$，查找最多 $O(n)$ 次。
- **辅助空间：** $O(n)$，需要额外的集合存储所有数字。

---

### 方法三：置换法 (Cyclic Sort / Index as Hash) - 最优解
题目要求 $O(n)$ 时间和 $O(1)$ 空间。我们需要利用原数组作为“哈希表”。

**思路：**
1. 对于一个长度为 $n$ 的数组，我们要找的缺失的正整数一定在 $[1, n+1]$ 范围内。
2. 我们可以尝试将数组中的每个数字 $x$（如果在 $[1, n]$ 范围内）放到它原本应该在的位置上，即下标为 $x-1$ 的位置。
3. 遍历数组，对于 `nums[i]`：
   - 如果 `1 <= nums[i] <= n` 且 `nums[nums[i] - 1] != nums[i]`，我们就将 `nums[i]` 和 `nums[nums[i]-1]` 进行交换。
   - 重复这个过程，直到 `nums[i]` 不满足条件（例如超出范围，或者已经在正确位置上了）。
4. 第二次遍历数组，找到第一个 `nums[i] != i + 1` 的位置，返回 `i + 1`。
5. 如果全部都符合，说明缺失的是 `n + 1`。

**核心逻辑：** `让数字 x 去到下标为 x-1 的位置`。

**代码实现：** `solution_v3_inplace.py`

**复杂度分析：**
- **时间复杂度：** $O(n)$。虽然有 `while` 循环，但每个数字最多被交换到正确位置一次，总交换次数不超过 $n$。
- **辅助空间：** $O(1)$，直接在原数组上操作。
