# LeetCode 189. 轮转数组 (Rotate Array) 循序渐进解析

## 题目描述
给定一个整数数组 `nums`，将数组中的元素向右轮转 `k` 个位置，其中 `k` 是非负数。

---

## 方法一：暴力旋转 (Brute Force)

### 思路
最直接的方法是每次将数组向右移动一位，重复 `k` 次。
1. 每次移动时，先保存最后一个元素。
2. 将数组前面的所有元素向右移动一位。
3. 将保存的最后一个元素放到数组的最前面。

### 复杂度分析
- **时间复杂度**: $O(n \times k)$。其中 $n$ 是数组长度。如果 $k$ 很大，这可能会导致超时。
- **空间复杂度**: $O(1)$。只使用了常数级别的额外空间。

---

## 方法二：使用额外数组 (Extra Array)

### 思路
我们可以创建一个同样大小的临时数组，计算出每个元素轮转后的新位置，直接放进去。
公式：新下标 = `(旧下标 + k) % 数组长度`。
最后将临时数组的内容复制回原数组。

### 复杂度分析
- **时间复杂度**: $O(n)$。我们需要遍历数组两次（一次计算位置，一次复制回原数组）。
- **空间复杂度**: $O(n)$。需要一个额外的数组来存储元素。

---

## 方法三：数组反转 (Reverse - 最优解)

### 思路
这是一个非常巧妙的技巧，不需要额外空间。
假设数组为 `[1,2,3,4,5,6,7]`，`k = 3`。
1. 首先处理 `k`，使其等于 `k % n`（以防 `k` 大于数组长度）。
2. 反转整个数组：`[7,6,5,4,3,2,1]`
3. 反转前 `k` 个元素：`[5,6,7,4,3,2,1]`
4. 反转剩余的 `n-k` 个元素：`[5,6,7,1,2,3,4]`

### 复杂度分析
- **时间复杂度**: $O(n)$。每个元素被翻转了两次。
- **空间复杂度**: $O(1)$。原地操作，不占用额外空间。

---

## 方法四：Python 特色切片 (Python Slice)

### 思路
利用 Python 强大的切片功能，可以将数组分为两部分并重新拼接。
`nums[:] = nums[n-k:] + nums[:n-k]`
注意：必须使用 `nums[:]` 进行赋值，才能在原地修改原数组的内容（修改引用的指向在 LeetCode 中通常不起作用）。

### 复杂度分析
- **时间复杂度**: $O(n)$。切片和拼接操作都会涉及到数组元素的复制。
- **空间复杂度**: $O(n)$。切片操作会产生新的列表对象。

---

## 总结
| 方法 | 时间复杂度 | 空间复杂度 | 备注 |
| :--- | :--- | :--- | :--- |
| 暴力旋转 | $O(n \times k)$ | $O(1)$ | 简单但效率低，易超时 |
| 额外数组 | $O(n)$ | $O(n)$ | 空间换时间，逻辑清晰 |
| 数组反转 | $O(n)$ | $O(1)$ | **面试推荐最优解**，技巧性强 |
| Python 切片 | $O(n)$ | $O(n)$ | Python 特色写法，非常简洁 |
