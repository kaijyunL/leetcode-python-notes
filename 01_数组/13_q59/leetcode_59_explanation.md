# LeetCode 59 - 螺旋矩阵 II (Spiral Matrix II)

这道题要求我们生成一个包含 `1` 到 `n^2` 所有元素，且元素按顺时针顺序螺旋排列的 `n x n` 正方形矩阵。

其实对于螺旋矩阵这类问题，无论是给定矩阵打印元素，还是生成矩阵，本质上都是**模拟**过程。我们不需要去寻找复杂的数学规律，而是要把“一圈一圈往里填数字”这个直观的行为忠实地用代码翻译出来。

下面我们循序渐进地介绍两种理解方式：从**方向数组模拟法**，到更清晰的**边界收缩法（最优解）**。

## 方法一：方向模拟法（基础模拟）

**思路：**
最直观的想法是，我们想象有一个机器人在矩阵里走。它一开始位于左上角 `(0, 0)`，面向右边。
它每走一步就把当前的数字填入格子。当它遇到**矩阵边缘**或者**已经填过数字的格子**时，它就需要进行**顺时针转弯**。
顺时针的转弯顺序总是固定的：向右 -> 向下 -> 向左 -> 向上 -> 向右...

我们可以定义一个方向数组来表示这个过程：
- 向右：行不变，列加1
- 向下：行加1，列不变
- 向左：行不变，列减1
- 向上：行减1，列不变

**复杂度：**
- 时间复杂度：$O(n^2)$，我们需要遍历并填充矩阵中的每一个元素。
- 空间复杂度：$O(1)$，除了用于输出的矩阵，不需要额外的空间。

## 方法二：边界收缩法（按层模拟，最优解）

**思路：**
在方向模拟法中，我们需要不断去判断机器人下一步是否越界或者是否碰到了已经填过的格子，逻辑上稍显细碎。
一种更宏观、也是在面试中最推荐写的方法是**定义四个边界**：上边界 (`top`)、下边界 (`bottom`)、左边界 (`left`)、右边界 (`right`)。

我们按照宏观的圈数来填数字：
1. **向右填（上边缘）**：从 `left` 填到 `right`。填完后，最上面一行的任务完成了，所以上边界下移（`top += 1`）。
2. **向下填（右边缘）**：从 `top` 填到 `bottom`。填完后，最右边一列的任务完成了，所以右边界左移（`right -= 1`）。
3. **向左填（下边缘）**：从 `right` 填到 `left`。填完后，最下面一行的任务完成了，所以下边界上移（`bottom -= 1`）。
4. **向上填（左边缘）**：从 `bottom` 填到 `top`。填完后，最左边一列的任务完成了，所以左边界右移（`left += 1`）。

在这个过程中，我们用一个变量 `num` 记录当前要填的数字，只要 `num` 还没有超过目标值（即 $n^2$），我们就一直执行这四步操作。这种方法思路极其清晰，非常容易写出 bug-free 的代码。

**复杂度：**
- 时间复杂度：$O(n^2)$，同样需要填充全矩阵的每一个元素。
- 空间复杂度：$O(1)$，只使用了四个边界变量以及计数变量。
