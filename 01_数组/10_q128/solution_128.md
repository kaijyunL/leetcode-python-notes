# LeetCode 128. 最长连续序列 (Longest Consecutive Sequence)

## 题目描述
给定一个未排序的整数数组 `nums` ，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

---

## 循序渐进理解

### 1. 暴力解法 (Brute Force)
**思路：**
对于数组中的每一个数字，我们都尝试寻找以它为起点的最长连续序列。
例如，如果当前数字是 `x`，我们就检查数组中是否存在 `x+1`，如果存在，再检查 `x+2`，以此类推。

**复杂度分析：**
- **时间复杂度：** $O(n^3)$。外层循环遍历所有数字 $O(n)$，内层 `while` 循环在最坏情况下遍历 $O(n)$，而在 Python 中使用 `item in list` 的检查操作也是 $O(n)$。
- **空间复杂度：** $O(1)$。

---

### 2. 排序解法 (Sorting)
**思路：**
既然我们要找连续的数字，如果数组是有序的，那么连续的数字就会排在一起。
我们可以先对数组进行排序，然后遍历排序后的数组，记录当前连续序列的长度。

**注意点：**
- 需要处理重复数字的情况（重复数字不增加长度，也不中断连续性）。
- 空数组的情况。

**复杂度分析：**
- **时间复杂度：** $O(n \log n)$。主要取决于排序算法的时间复杂度。
- **空间复杂度：** $O(1)$ 或 $O(n)$，取决于排序算法的实现。

---

### 3. 哈希表最优解 (Hash Set) - $O(n)$
**思路：**
为了达到 $O(n)$ 的复杂度，我们需要在 $O(1)$ 时间内判断某个数字是否存在。使用哈希集合（`set`）非常合适。

**核心优化技巧：**
如果我们想找一个序列 `[x, x+1, x+2, ..., x+k]`，我们只需要从该序列的**起点** `x` 开始计数即可。
如何判断 `x` 是起点？只需检查 `x-1` 是否在集合中。
- 如果 `x-1` 在集合中，说明 `x` 不是起点（它是一个更长序列的一部分），我们跳过它。
- 如果 `x-1` 不在集合中，说明 `x` 是起点，我们开始向后匹配 `x+1, x+2...` 并更新最大长度。

这样，每个数字最多只会被访问两次（一次在集合中被检查，一次作为序列的一部分被计数），总时间复杂度为 $O(n)$。

**复杂度分析：**
- **时间复杂度：** $O(n)$。
- **空间复杂度：** $O(n)$。用于存储哈希集合。
