# LeetCode 118. 杨辉三角 (Pascal's Triangle)

杨辉三角（Pascal's Triangle）是数学中一个经典的结构。题目要求给定一个非负整数 `numRows`，生成杨辉三角的前 `numRows` 行。

在杨辉三角中，每个数是它左上方和右上方两数之和。

---

## 1. 简单暴力法：递归思路 (Recursive Approach)

### 思路讲解
最直观的想法是利用杨辉三角的定义：
- 第 `i` 行第 `j` 列的数字等同于第 `i-1` 行第 `j-1` 列 和 第 `i-1` 行第 `j` 列数字之和。
- 边界条件：每一行的首位和末位都是 `1`。

虽然直接递归会有大量重复计算，但它是理解杨辉三角生成逻辑的基础。

### 算法步骤
1. 定义一个函数 `get_value(i, j)` 表示第 `i` 行第 `j` 列的值。
2. 如果 `j == 0` 或 `j == i`，返回 `1`。
3. 否则，返回 `get_value(i-1, j-1) + get_value(i-1, j)`。
4. 循环调用生成整个三角形。

---

## 2. 最优解法：动态规划 / 迭代法 (Iterative Approach)

### 思路讲解
递归法存在大量的重复计算（可以用记忆化改进），但在生成整个三角形时，**迭代法**是最优的，时间复杂度和空间复杂度都达到了理论下限 $O(numRows^2)$。

我们直接利用前一行来构建当前行：
1. 第一行固定为 `[1]`。
2. 对于后续的每一行：
   - 首尾始终为 `1`。
   - 中间的元素 `row[j] = prev_row[j-1] + prev_row[j]`。

### 优化点
无需递归开销，直接在循环中完成数组构建。

---

## 3. 复杂度分析

- **时间复杂度**：$O(numRows^2)$。我们需要计算每一行的每个元素，总元素个数为 $1 + 2 + ... + numRows = \frac{numRows(numRows+1)}{2}$。
- **空间复杂度**：$O(numRows^2)$。如果不考虑返回值的空间，额外空间复杂度为 $O(1)$ 或 $O(numRows)$（取决于实现细节），但通常我们需要存储整个三角。
