# LeetCode 48. 旋转图像

## 题目描述

给定一个 `n × n` 的二维矩阵 `matrix` 表示一个图像，请你将图像顺时针旋转 90 度。

**你必须在原地旋转图像**，即直接修改输入的二维矩阵，不能使用额外的矩阵。

### 示例

```
输入：matrix = [[1,2,3],
                [4,5,6],
                [7,8,9]]

输出：        [[7,4,1],
               [8,5,2],
               [9,6,3]]
```

```
输入：matrix = [[ 5, 1, 9,11],
                [ 2, 4, 8,10],
                [13, 3, 6, 7],
                [15,14,12,16]]

输出：        [[15,13, 2, 5],
               [14, 3, 4, 1],
               [12, 6, 8, 9],
               [16, 7,10,11]]
```

### 约束条件

- `n == matrix.length == matrix[i].length`
- `1 <= n <= 20`
- `-1000 <= matrix[i][j] <= 1000`

---

## 核心规律

顺时针旋转 90° 时，元素的位置变换规律为：

```
原位置 (i, j) → 新位置 (j, n-1-i)
```

以 3×3 矩阵为例：

```
(0,0)→(0,2)   (0,1)→(1,2)   (0,2)→(2,2)
(1,0)→(0,1)   (1,1)→(1,1)   (1,2)→(2,1)
(2,0)→(0,0)   (2,1)→(1,0)   (2,2)→(2,0)
```

---

## 解法一：辅助矩阵（暴力法）

### 思路

最直观的方法：创建一个新矩阵，按照旋转规律 `(i, j) → (j, n-1-i)` 把每个元素放到新位置，最后将结果复制回原矩阵。

> ⚠️ 严格来说这不满足"原地旋转"的要求（使用了 O(n²) 额外空间），但它是理解旋转规律的最好起点。

### 复杂度

| | 复杂度 |
|---|---|
| 时间 | O(n²) |
| 空间 | O(n²) — 需要额外矩阵 |

### 图解

```
原矩阵:          辅助矩阵(按规律填入):
1  2  3          7  4  1
4  5  6    →     8  5  2
7  8  9          9  6  3

规则: new[j][n-1-i] = old[i][j]
  old[0][0]=1 → new[0][2]=1
  old[0][1]=2 → new[1][2]=2
  old[2][0]=7 → new[0][0]=7
  ...
```

### 代码

```python
def rotate(matrix):
    n = len(matrix)
    # 创建一个同样大小的新矩阵
    new_matrix = [[0] * n for _ in range(n)]
    # 按照旋转规律填入新矩阵
    for i in range(n):
        for j in range(n):
            new_matrix[j][n - 1 - i] = matrix[i][j]
    # 将新矩阵的值复制回原矩阵
    for i in range(n):
        for j in range(n):
            matrix[i][j] = new_matrix[i][j]
```

## 解法二：逐层旋转（四元素交换）

### 思路

我们可以把矩阵看成一圈一圈的"洋葱层"，从最外层到最内层，逐层处理。

对于每一层，我们将四个对应位置的元素进行**循环交换**：

```
top-left → top-right → bottom-right → bottom-left → top-left
```

具体来说，四个位置的交换关系为：

```
(top, left+i) 
    → (top+i, right)
        → (bottom, right-i)
            → (bottom-i, left)
                → 回到 (top, left+i)
```

### 复杂度

| | 复杂度 |
|---|---|
| 时间 | O(n²) |
| 空间 | O(1) — 真正的原地旋转！|

### 图解

以 4×4 矩阵为例，第一层（最外层）的第一组交换：

```
 [A] .  .  [B]        [D] .  .  [A]
  .  .  .   .    →     .  .  .   .
  .  .  .   .          .  .  .   .
 [D] .  .  [C]        [C] .  .  [B]

四个角元素循环交换：A→B→C→D→A（用一个临时变量保存）
然后继续处理同一层的下一组四元素...
```

分层处理示意（4×4 矩阵有 2 层）：

```
第 0 层（外层）:       第 1 层（内层）:
 X  X  X  X            .  .  .  .
 X  .  .  X            .  X  X  .
 X  .  .  X            .  X  X  .
 X  X  X  X            .  .  .  .
```

### 代码

```python
def rotate(matrix):
    n = len(matrix)
    # 逐层处理，从最外层到最内层
    for layer in range(n // 2):
        first = layer          # 当前层的起始索引
        last = n - 1 - layer   # 当前层的结束索引
        for i in range(first, last):
            offset = i - first
            # 保存上边的值
            top = matrix[first][i]
            # 左边 → 上边
            matrix[first][i] = matrix[last - offset][first]
            # 下边 → 左边
            matrix[last - offset][first] = matrix[last][last - offset]
            # 右边 → 下边
            matrix[last][last - offset] = matrix[i][last]
            # 上边（已保存）→ 右边
            matrix[i][last] = top
```

## 解法三：转置 + 翻转（最优雅解法）⭐

### 思路

这是最巧妙也最常用的解法，将旋转分解为两步简单操作：

1. **转置矩阵**：沿主对角线翻转，即 `matrix[i][j]` 与 `matrix[j][i]` 互换
2. **水平翻转每一行**：将每行左右颠倒

**数学证明**：
- 转置：`(i, j) → (j, i)`
- 再翻转行：`(j, i) → (j, n-1-i)`
- 合起来：`(i, j) → (j, n-1-i)` ✅ 正好是顺时针旋转 90°！

### 复杂度

| | 复杂度 |
|---|---|
| 时间 | O(n²) |
| 空间 | O(1) — 原地操作 |

### 图解

```
   原矩阵           第一步：转置           第二步：翻转每行
                   (行列互换)              (左右颠倒)
 1  2  3          1  4  7               7  4  1
 4  5  6    →     2  5  8         →     8  5  2
 7  8  9          3  6  9               9  6  3
                                        ✅ 旋转完成！
```

### 代码

```python
def rotate(matrix):
    n = len(matrix)
    # 第一步：转置矩阵（沿主对角线翻转）
    for i in range(n):
        for j in range(i + 1, n):  # 注意 j 从 i+1 开始，避免重复交换
            matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]
    # 第二步：翻转每一行
    for row in matrix:
        row.reverse()
```

### 为什么这个解法最优雅？

1. **代码最简洁**：不到 10 行，逻辑清晰
2. **不需要复杂的索引计算**：相比逐层旋转，不容易出错
3. **易于记忆**：只需记住"先转置，再翻转"
4. **面试中推荐使用**：既展示了思维能力，又简洁高效

## 三种解法对比

| 解法 | 时间复杂度 | 空间复杂度 | 原地操作 | 代码难度 | 推荐度 |
|------|-----------|-----------|---------|---------|-------|
| 辅助矩阵 | O(n²) | O(n²) | ❌ | ⭐ | 理解用 |
| 逐层旋转 | O(n²) | O(1) | ✅ | ⭐⭐⭐ | 了解即可 |
| 转置+翻转 | O(n²) | O(1) | ✅ | ⭐ | ⭐⭐⭐ 强烈推荐 |

## 拓展：其他旋转方向

| 旋转方向 | 操作 |
|---------|------|
| 顺时针 90° | 转置 → 翻转每行 |
| 逆时针 90° | 转置 → 翻转每列（或：翻转每行 → 转置） |
| 180° | 翻转每行 → 翻转每列（或：顺时针旋转两次） |

掌握了"转置 + 翻转"的思路，所有旋转方向都能轻松应对！
