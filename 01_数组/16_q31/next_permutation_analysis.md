# LeetCode 第 31 题：下一个排列 (Next Permutation) - 循序渐进解法

## 1. 题目描述
[LeetCode 31. Next Permutation](https://leetcode.cn/problems/next-permutation/)

**目标**：实现“下一个排列”函数，将给定数字序列重新排列成字典序中下一个更大的排列。
- 如果不存在下一个更大的排列，则将数字重新排列成最小的排列（即升序排列）。
- 必须**原地**修改，只允许使用额外常数空间。

---

## 2. 解法一：暴力法 (Brute Force) - 思路理解

### 思路
暴力法的核心思想是“我全都要”：
1. 生成该数组所有可能的排列组合。
2. 将这些排列按字典序排序。
3. 在排序后的列表中找到当前的排列，并返回它的下一个。
4. 如果当前排列是最后一个，则返回排序列表的第一个（最小值）。

### 为什么暴力法不行？
- **时间复杂度**：生成 $n$ 个数字的全排列需要 $O(n!)$ 的时间。当 $n=100$ 时（题目给定范围），$100!$ 是一个天文数字，计算机无法在有限时间内算完。
- **空间复杂度**：存储这些排列需要 $O(n! \times n)$ 的空间，会直接导致内容溢出 (Out of Memory)。

> 虽然此方法在 LeetCode 提交会超时，但它是理解“字典序”的基石。

---

## 3. 解法二：最优解 - 两遍扫描法 (Two-pass Path)

### 核心观察
我们需要找的是一个比当前排列**稍微大一点点**的排列。要让一个数变大，我们需要：
1. 把右边的一个“大数”和左边的一个“小数”交换。
2. 交换的位置越靠右，变大的幅度越小。
3. 交换后，我们要让右边的部分尽可能保持最小（升序）。

### 算法步骤
1. **寻找转折点**：从右向左遍历，找到第一个相邻的升序对 `(i, i+1)`，使得 `nums[i] < nums[i+1]`。此时，`nums[i+1:]` 必然是降序排列的。
2. **寻找略大的数**：如果在步骤 1 找到了 `i`，则再次从右向左遍历，在 `nums[i+1:]` 中找到第一个大于 `nums[i]` 的数 `nums[j]`。
3. **交换**：交换 `nums[i]` 和 `nums[j]`。
4. **反转**：此时 `nums[i+1:]` 依然是降序的。为了让增幅最小，我们需要将其反转为升序。
5. **特殊情况**：如果在步骤 1 没找到 `i`（即整个数组都是降序），直接反转整个数组即可。

### 复杂度分析
- **时间复杂度**：$O(n)$。我们最多遍历两遍数组。
- **空间复杂度**：$O(1)$。我们只进行了原地修改，没有使用额外空间。

---

## 4. 总结与直观理解

想象一个数字 `123654`：
1. 从右往左找，`3` 是第一个比它右边数字小的数（转折点）。
2. 在 `3` 的右边 `654` 中，找比 `3` 大的最小数，是 `4`。
3. 交换 `3` 和 `4` $\rightarrow$ `124653`。
4. 把 `4` 后面的 `653` 反转成升序 $\rightarrow$ `124356`。

结果就是 `124356`。
