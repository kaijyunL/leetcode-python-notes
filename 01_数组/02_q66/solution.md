# LeetCode 66. 加一 (Plus One) 循序渐进题解

这一题的要求是将一个由数字组成的数组（代表一个非负整数）加 1。

---

## 方法一：暴力/直观解法 (类型转换)

### 思路
1. 将数组中的所有数字拼接成一个数字或字符串。
2. 将其转换为整数类型。
3. 执行 `+ 1` 操作。
4. 将得到的结果再转换回数组形式。

### 优缺点
- **优点**：代码逻辑非常直观，利用了 Python 对大整数的自动处理能力。
- **缺点**：在不支持大整数的语言（如 C++/Java）中，数组长度较长时会发生溢出。此外，频繁的类型转换在处理超大数组时效率较低。

---

## 方法二：最优解法 (逆序遍历/进位处理)

### 思路
直接在数组上模拟手工加法的逻辑：
1. **从后往前**遍历数组。
2. 将当前位数字加 1。
3. 如果加 1 后的数字 **小于 10**，说明没有进位，直接返回当前数组即可。
4. 如果加 1 后的数字 **等于 10**，说明有进位，将当前位设为 0，然后继续处理前一位。
5. **特殊情况**：如果循环结束还没有返回，说明原数组像 `[9, 9, 9]` 这样，加 1 后变成了 `[0, 0, 0]`，此时需要在数组开头补一个 `1`，结果变为 `[1, 0, 0, 0]`。

### 优缺点
- **优点**：内存消耗低（$O(1)$ 额外空间，如果原地修改），时间复杂度优秀（$O(n)$）。
- **缺点**：需要处理数组扩容的边缘情况。

---

## 方法三：While 循环解法

### 思路
`while` 循环的思路与 `for` 循环其实是一致的，只是我们手动控制指针（索引）：
1. 初始化指针 `i = len(digits) - 1`，指向数组最后一位。
2. 只要指针 `i` 大于等于 0，就执行以下逻辑：
   - 将 `digits[i]` 加 1。
   - 如果加完后不等于 10，直接返回。
   - 如果加完后等于 10，将 `digits[i]` 置为 0，并将指针 `i` 向左移动一位（`i -= 1`）。
3. 如果 `while` 循环自然结束（指针超出了左边界），说明发生了全局进位（如 `[9, 9]` 变成 `[0, 0]`），需要在最前面加入 `1`。

### 优缺点
- **优点**：相比 `for ... range`，`while` 循环在某些开发者看来控制感更强，且逻辑非常清晰地展现了“只要还在进位，就继续循环”的过程。
