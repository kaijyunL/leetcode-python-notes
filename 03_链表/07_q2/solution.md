# LeetCode 2. 两数相加 (Add Two Numbers)

## 题目理解
给定两个**非空**链表，表示两个非负整数。它们每位数字都是按照**逆序**的方式存储的，并且每个节点只能存储**一位**数字。
请你将两个数相加，并以相同形式返回一个表示和的链表。
你可以假设除了数字 0 之外，这两个数都不会以 0 开头。

---

## 1. 简单暴力法 (转成数字相加后再转换为链表 - 不推荐)

### 思路
最直观、最暴力的想法是：把两个链表分别转化成正常的整数，然后将这两个整数相加，最后把加得的和再转回链表形式。
比如链表 `2 -> 4 -> 3` 也就是 `342`。

### 缺点
在很多语言（如 C++、Java）中，链表可能非常长（比如题目限制最多 100 个节点），这就意味着这个数有一百位，远远超过了标准整型变量（如 `int`, `long long`）所能表示的最大范围，会发生严重的**数值溢出（Overflow）**。
虽然在 Python 中整数可以无限大，能用这个取巧的方法通过，但这违反了考察底层数据结构（链表操作）的初衷，在面试中不推荐使用。

---

## 2. 最优解法 (模拟进位加法)

### 思路
既然链表已经是逆序存储的了（即链表头就是个位，第二个节点是十位...），这非常契合我们在草稿纸上列竖式做加法的过程：从个位开始算，逐位相加；如果有进位（carry），就留给下一位去加。

具体步骤设计如下：
1. **定义虚拟头节点** `dummy`：用来帮我们轻松找到最终返回链表的头部，而不需要处理复杂的头节点初始化逻辑。
2. **定义游标指针** `curr`：初始指向 `dummy`，用来不断往下移动构建新链表。
3. **定义进位变量** `carry`：初始为 0。主要应对类似 `5+6=11` 会向高位进1 的情况。
4. **遍历两个链表** `l1` 和 `l2`，只要它们中至少有一个链表还没走完，**或者** `carry` 还有值（不为0），就继续循环求和：
    - 获取当前两位数字的值：如果某个链表已经走完了，就把它那里的值当做 0。
    - 求本位总和：`total = val1 + val2 + carry`。
    - 计算本位留下来的数字：`total % 10`，将其作为一个新节点挂在 `curr.next` 上。
    - 计算传给下一位的进位：`total // 10`。
    - 将 `curr` 还有 `l1`, `l2` 的指针分别向后移动。
5. 当一切都处理完后，返回 `dummy.next` 即可。

### 复杂度分析
- **时间复杂度**：$O(\max(m, n))$，其中 $m$ 和 $n$ 是两个链表的长度。我们最多只遍历较长那个链表的长度次。
- **空间复杂度**：$O(1)$。我们除了返回需要构建的新链表节点外，只需要常数级别的额外空间来存三个变量（指针 $curr$、指针 $dummy$、进位 $carry$）。

此方法即是该题的标准最优解法。
