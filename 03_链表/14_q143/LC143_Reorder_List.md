# LeetCode 143: 重排链表 (Reorder List)

## 题目描述
给定一个单链表 `L` 的头节点 `head` ，单链表 `L` 表示为：
`L0 → L1 → … → Ln - 1 → Ln`
请将其重新排列后变为：
`L0 → Ln → L1 → Ln - 1 → L2 → Ln - 2 → …`

**注意**：不能只是单纯的改变节点内部的值，而是需要实际的进行节点交换。

---

## 方法一：线性表（数组法）

### 💡 核心思路
对于数组或者列表，我们可以通过索引 $O(1)$ 的时间去访问任何一个元素（包括首尾）。但对于单链表，如果要找到末尾的节点，只能从头遍历，时间开销大。
最简单直观（即暴力、退而求其次）的解法是：
1. **转存**：遍历整个单链表，将所有节点按顺序保存到一个数组/列表中。
2. **双指针交替连接**：有了数组之后，我们设置两个指针 `left` 和 `right`，分别指向数组的首和尾。
3. 把 `left` 指向的节点指向 `right` 所在的节点，接着 `left` 前进一步；再根据题意交错指回 `left` 所在的下一个节点，`right` 退后一步。
4. **处理尾部防环**：反复上述操作直到两个指针相遇，最后将最后一个节点的 `next` 置为 `None`，防止产生环路。

### ⏳ 复杂度分析
- **时间复杂度**：$O(N)$，其中 $N$ 是链表节点的个数。我们需要遍历一次链表存入数组，再通过双指针遍历一半的数组重建连接。
- **空间复杂度**：$O(N)$，利用了一个额外的列表来保存链表的所有节点。

---

## 方法二：找中点 + 反转后半段 + 交替合并 （最优解法）

### 💡 核心思路
如果你不想使用 $O(N)$ 的额外辅助空间，如何做到就地（In-place）修改呢？
我们可以观察重新排列的规律：第一半的链表节点是正序的，而后半部分的链表节点是倒序插入的。
举个例子：`1 -> 2 -> 3 -> 4 -> 5`
重排后：`1 -> 5 -> 2 -> 4 -> 3`
可以拆分成三个经典步骤：
1. **寻找链表中点**：利用**快慢指针法**（快指针一次走两步，慢指针一次走一步），当快指针走到结尾时，慢指针刚好处在中点位置。这可以将链表分为前、后两半。
2. **反转后半部分链表**：切断前面和后面的连接，将后面的半个链表（比如上面的 `3->4->5` 变为 `5->4->3`）直接就地反转。
3. **交替合并两个链表**：现在你有两个链表，头分别是原链表头（前一半）和反转过头节点（后一半），利用两个指针，一步步交替穿插把它们重新连接起来即可。

**这是一个非常精妙的综合题，它把找中点、反转链表、以及合并链表三个基础考核点结合在了一起。**

### ⏳ 复杂度分析
- **时间复杂度**：$O(N)$。寻找中点耗费 $O(N)$，反转耗费 $O(N)$，合并耗费 $O(N)$，总时间还是 $O(N)$。
- **空间复杂度**：$O(1)$，只有几个指针变量，完全是原地修改链表，没有创建额外的数组。
