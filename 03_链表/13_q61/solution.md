# LeetCode 第 61 题：旋转链表 (Rotate List)

## 题目描述
给你一个链表的头节点 `head` ，旋转链表，将链表每个节点向右移动 `k` 个位置。

**示例 1：**
- 输入：`head = [1,2,3,4,5], k = 2`
- 输出：`[4,5,1,2,3]`

**示例 2：**
- 输入：`head = [0,1,2], k = 4`
- 输出：`[2,0,1]`

---

## 方法一：简单暴力 (Brute Force)

### 思路
最直观的想法是：每次将链表向右移动 1 个位置，重复 `k` 次。
移动 1 个位置的操作如下：
1. 找到链表的倒数第二个节点。
2. 将倒数第一个节点提升为新的头节点。
3. 断开原倒数第二个节点与最后一个节点的连接（使其成为新的尾节点）。

### 算法步骤
- 如果链表为空或只有一个节点，直接返回。
- 重复 `k` 次以下操作：
    - 找到尾节点 `tail` 和倒数第二个节点 `prev`。
    - 将 `tail.next` 指向当前的 `head`。
    - 将 `head` 更新为 `tail`。
    - 将 `prev.next` 置为 `None`。

### 复杂度分析
- **时间复杂度**：$O(k \times n)$，其中 $n$ 是链表长度。如果 $k$ 非常大，效率会极低。
- **空间复杂度**：$O(1)$，只使用了常数级的额外空间。

---

## 方法二：最优解法 (闭成环后再断开)

### 思路
观察发现，旋转 $k$ 次实际上是将链表的后 $k \pmod n$ 个节点移动到前面。
与其一次次移动，不如直接找到新的切分点。

### 算法步骤
1. **处理边界**：如果链表为空、只有一个节点或 `k=0`，直接返回。
2. **计算长度并成环**：
    - 遍历链表计算长度 $n$，并定位到原本的尾节点 `tail`。
    - 将 `tail.next` 指向 `head`，使链表形成一个**环**。
3. **确定切分位置**：
    - 实际需要旋转的位移为 `k % n`。
    - 向后移动 `n - (k % n)` 个位置会到达新的尾节点位置。
4. **断开环**：
    - 在新尾节点处断开，其下一个节点即为新的头节点。

### 核心逻辑图解
假设链表为 `1 -> 2 -> 3 -> 4 -> 5`，`n=5`, `k=2`。
1. 成环：`1 -> 2 -> 3 -> 4 -> 5 -> (回到1)`
2. 新尾节点：从原来的尾节点 5 出发，走 `n - (k % n) = 5 - (2 % 5) = 3` 步回到位置 3。
3. 位置 3 是新尾，位置 4 是新头。
4. 断开：`3.next = None`，返回节点 4。

### 复杂度分析
- **时间复杂度**：$O(n)$，我们只需要遍历链表两次（第一次计算长度并成环，第二次找到新尾部）。
- **空间复杂度**：$O(1)$。
