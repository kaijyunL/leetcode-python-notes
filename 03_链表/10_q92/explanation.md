# LeetCode 92. 反转链表 II (Reverse Linked List II)

## 题目描述
给你单链表的头指针 `head` 和两个整数 `left` 和 `right` ，其中 `left <= right` 。请你反转从位置 `left` 到位置 `right` 的链表节点，返回 **反转后的链表** 。

**示例 1：**
![img](https://assets.leetcode.com/uploads/2021/02/19/rev2node.jpg)
- 输入：head = [1,2,3,4,5], left = 2, right = 4
- 输出：[1,4,3,2,5]

---

## 解法一：暴力法（数组辅助）
这是最直观的方法，适合面试时如果思路卡壳作为保底方案。

### 思路
1. 遍历一遍链表，将所有节点的值存入一个动态数组（或列表）。
2. 在数组中找到 `left` 到 `right` 的区间，使用双指针进行反转。
3. 再次遍历链表，将反转后的数组值依次填回链表节点中。

### 优缺点
- **优点**：逻辑极其简单，不容易出错。
- **缺点**：使用了额外的线性空间 $O(N)$，且需要多次遍历。

---

## 解法二：切断与反转（三步走）
这种方法利用了“反转整个链表”的已有经验。

### 思路
1. **定位区间**：找到待反转区间的第一个节点的前驱节点 `pre` 和最后一个节点 `right_node`。
2. **切断**：将该区间从原链表中切断，记录该区间的头节点 `left_node` 和后继节点 `curr`。
3. **反转并接回**：反转该子区间，最后将反转后的部分重新接回原链表。

### 关键点
- 使用**虚拟头节点 (dummy node)** 处理 `left=1` 的边界情况。

---

## 解法三：头插法（最优解，一次遍历）
这是官方推荐的一步到位解法，通过调整指针指向实现。

### 思路
在遍历到 `left` 的位置后，逐个将 `left+1` 到 `right` 的节点“搬”到 `left` 的前面。

1. 设置 `pre` 节点，使其始终指向待反转区域的前驱。
2. 设置 `curr` 节点，指向待反转区域的第一个元素。
3. 每次循环，将 `curr` 的下一个节点 `nxt` 抽取出来，插入到 `pre` 和 `pre.next` 之间。

### 示例图解 (left=2, right=4)
- 初始：`1(pre) -> 2(curr) -> 3(nxt) -> 4 -> 5`
- 第一步（搬 3）：`1(pre) -> 3 -> 2(curr) -> 4 -> 5`
- 第二步（搬 4）：`1(pre) -> 4 -> 3 -> 2(curr) -> 5`

### 优点
- **一次遍历**，无需额外空间（$O(1)$ 空间复杂度）。
- 逻辑优雅，是面试中最受青睐的方案。

## 解法四：206 式风格（逻辑延续性最好）
如果你已经背熟了第 206 题（全量反转链表）的四步走口诀，那么这种解法对你来说几乎没有记忆负担。

### 思路
1. **定位**：找到反转区间的前驱 `pre`。
2. **复用 206 逻辑**：在指定的 `for` 循环次数内，执行 206 题的“暂存-反转-前移-前移”四步走。
3. **缝合**：将反转后的局部链表重新接入原链表。

### 与 206 题的异同点
| 维度 | 206 题 | 92 题 |
| :--- | :--- | :--- |
| **初始状态** | `prev = None`, `curr = head` | `prev = None`, `curr = start` |
| **循环条件** | `while curr` | `for _ in range(区间长度)` |
| **收尾工作** | 直接返回 `prev` | 需要 `pre.next = prev` 和 `start.next = curr` |

这种解法既保留了最优的空间复杂度，又极大程度上复用了旧知识，强烈推荐！
