# LeetCode 142. 环形链表 II (Linked List Cycle II)

这道题是「环形链表Ⅰ」的进阶版，不仅要判断链表是否有环，还要**找出环的入口节点**。

我们将从简单暴力的哈希表方法开始，逐步深入到最优的快慢指针解法。

## 方法一：哈希表法 (简单暴力)

### 思路
最直观的想法就是“走过的地方做个记号”。
我们遍历链表，每走到一个节点，就把它存入到一个集合（Set）中。
- 如果遍历过程中，发现当前节点**已经存在于集合中**，说明我们绕了一圈又回到了走过的地方，这个节点就是**环的入口**。
- 如果一路走到 `null`，说明链表没有环，返回 `null`。

### 复杂度分析
- **时间复杂度**：$O(N)$，其中 $N$ 是链表中的节点数。我们只需遍历一次链表。
- **空间复杂度**：$O(N)$，我们需要一个哈希集合来存储所有访问过的节点。

---

## 方法二：快慢指针法 (最优解)

为了将空间复杂度降低到 $O(1)$，我们可以使用经典的**Floyd 判圈算法（龟兔赛跑算法）**。这个方法分为两个阶段：

### 阶段一：判断是否有环
正如第 141 题所做的那样，设定两个指针 `slow` 和 `fast`，都指向头节点：
- `slow` 每次走一步。
- `fast` 每次走两步。

如果链表有环，`fast` 终会追上 `slow`（两者相遇）。如果 `fast` 走到尽头，说明无环。

### 阶段二：寻找环的入口 （关键！）
**为什么相遇后，再派一个指针从头开始走，两者相遇的地方就是入口？**

我们来推导一下背后的数学原理：
假设：
- 链表头部到环入口的距离为 $a$。
- 环入口到相遇点的距离为 $b$。
- 相遇点再顺着环走到环入口的距离为 $c$。
- 那么环的周长就是 $b + c$。

当 `slow` 和 `fast` 相遇时：
- `slow` 走过的路程为 $a + b$。
- `fast` 走过的路程为 $a + n(b + c) + b$，其中 $n$ 是 `fast` 在环里绕的圈数（$n \ge 1$）。

因为 `fast` 的速度是 `slow` 的两倍，同一段时间内，`fast` 走过的总距离是 `slow` 的两倍：

$$
2 \times (a + b) = a + n(b + c) + b
$$

**化简一下方程（更直观的推导）：**

我们先整理一下等式右边的项：

$$
2a + 2b = a + b + n(b + c)
$$

然后等式两边同时减去 $a$ 和 $b$：

$$
a + b = n(b + c)
$$

因为我们需要求起点到入口的距离 $a$，所以把 $b$ 移到等式右边：

$$
a = n(b + c) - b
$$

为了把 $-b$ 消掉，我们将 $n$ 圈拆成 $(n-1)$ 圈加 $1$ 圈：

$$
a = (n - 1)(b + c) + (b + c) - b
$$

显然，后面的 $+b$ 和 $-b$ 抵消了，最后一步得到：

$$
a = (n - 1)(b + c) + c
$$

这个等式 $a = (n - 1)(b + c) + c$ 告诉我们什么呢？
- $a$ 是从头节点到入口的距离。
- $c$ 是从相遇点到入口的距离。
- $(n-1)(b+c)$ 是跑了 $n-1$ 圈环上的长度。

这意味着，**从头节点到入口的距离 $a$**，刚好等于 **从相遇点继续走到入口的距离 $c$** 加上整数圈的环长。

**所以结论很简单：**
当 `slow` 和 `fast` 在环中相遇时，我们让其中的任何一个指针（比如 `fast`回到头部的话也可以，不一定非得新建变量）回到链表头部 `head`，然后两个指针此时都**每次只走一步**。
当它们再次相遇时，刚好就在**环的入口节点**处碰头！

### 复杂度分析
- **时间复杂度**：$O(N)$。在阶段一，快慢指针相遇时时间复杂度为 $O(N)$。在阶段二，两个指针相遇的时间复杂度最大也为 $O(N)$，总体还是 $O(N)$。
- **空间复杂度**：$O(1)$。只使用了常数个指针变量。
