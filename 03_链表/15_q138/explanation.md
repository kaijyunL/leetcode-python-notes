# LeetCode 第 138 题：复制带随机指针的链表 (Copy List with Random Pointer)

## 题目描述

给你一个长度为 `n` 的链表，每个节点包含一个额外增加的随机指针 `random` ，该指针可以指向链表中的任何节点或空节点 `null`。

请构造这个链表的 **深拷贝**。深拷贝应该正好由 `n` 个 **全新** 节点组成，其中每个新节点的值都设为其对应的原节点的值。新节点的 `next` 指针和 `random` 指针也都应该指向复制链表中的新节点，并使原链表和复制链表中的这些指针能够表示相同的链表状态。**复制链表中的指针都不应指向原链表中的节点**。

例如，如果原链表中有两个节点 `X` 和 `Y`，其中 `X.random --> Y` 。那么在复制链表中对应的两个节点 `x` 和 `y` ，同样有 `x.random --> y` 。

返回复制链表的头节点。

---

## 解法一：哈希表法（简单直观）

这是最容易想到的一种方法。我们可以使用一个哈希表来建立「原节点」到「新节点」之间的映射关系。

### 核心步骤：
1. **第一次遍历**：遍历原链表，为每个节点创建一个对应的新节点（仅复制 `val`），并将 `原节点 -> 新节点` 的映射关系存入哈希表中。
2. **第二次遍历**：再次遍历原链表，根据哈希表的映射，将新节点的 `next` 和 `random` 指针连接到正确的新节点上。

### 算法复杂度：
- **时间复杂度**：$O(N)$，其中 $N$ 是链表的长度。我们需要遍历两遍链表。
- **空间复杂度**：$O(N)$，我们需要一个哈希表来存储 $N$ 个原节点到新节点的映射。

---

## 解法二：节点交替连接法（空间优化）

如果我们希望将空间复杂度优化到 $O(1)$（不计返回结果所需的空间），我们可以利用链表本身的结构来存储映射关系，而不需要额外的哈希表。

### 核心步骤：
1. **第一步：原地复制节点**
   - 遍历原链表，在每个原节点 `A` 后面插入一个新的拷贝节点 `A'`。
   - 此时链表结构变为：`A -> A' -> B -> B' -> C -> C' -> ...`

2. **第二步：设置拷贝节点的 random 指针**
   - 如果原节点 `A` 的 `random` 指向 `C`，那么 `A'` 的 `random` 应该指向 `C'`（即 `C.next`）。
   - 表达式：`node.next.random = node.random.next` (注意 `node.random` 为空的情况)。

3. **第三步：拆分链表**
   - 将交替连接的链表拆分成「原链表」和「拷贝后的链表」。
   - 例如：把 `A -> A' -> B -> B' -> ...` 拆回原来的 `A -> B -> ...` 和目标 `A' -> B' -> ...`。

### 算法复杂度：
- **时间复杂度**：$O(N)$，我们需要遍历三遍链表。
- **空间复杂度**：$O(1)$，不需要额外的辅助空间（除了存储返回结果的空间）。

---

## 总结

- **哈希表法**：代码逻辑清晰，易于实现，适合在面试中快速给出准确答案。副作用是需要额外的线性空间。
- **交替连接法**：这种方法的精髓在于「用链表自己的 next 指针代替哈希表」来存储映射关系。这是一种非常高级的链表处理技巧，能够通过空间换取时间，在资源受限的环境下非常有用。
