# LeetCode 82. 删除排序链表中的重复元素 II

## 题目描述
给定一个已排序的链表的头 `head` ，删除所有含有重复数字的节点，只保留原始链表中 **只出现一次** 的数字。返回同样按分类排序的链表。

**示例 1：**
- 输入：`head = [1,2,3,3,4,4,5]`
- 输出：`[1,2,5]`

**示例 2：**
- 输入：`head = [1,1,1,2,3]`
- 输出：`[2,3]`

---

## 方法一：哈希计数（简单暴力）

### 核心思路
这是最直观的方法。因为我们需要找出只出现一次的节点，所以可以：
1. **第一次遍历**：使用哈希表（如 Python 的 `Counter`）记录链表中每个元素出现的次数。
2. **第二次遍历**：根据计数结果，构建一个新的链表，只包含那些计数为 1 的数值。

### 优缺点
- **优点**：思路简单，不容易在指针操作上出错。
- **缺点**：需要额外的 $O(N)$ 空间存储计数值，且需要遍历两次。

## 方法二：一次遍历 + 哑节点（最优解）

### 核心思路
利用链表已排序的特性，我们可以在一次遍历中直接跳过重复节点。

1. **哑节点 (Dummy Node)**：
   由于头节点也可能被删除（比如 `[1, 1, 2]`），我们创建一个 `dummy` 节点指向 `head`。最后返回 `dummy.next`。
2. **三指针思想（或前驱指针）**：
   - 我们维护一个指针 `pre`，它始终指向“当前已确定的不重复部分的最后一个节点”。
   - 每次检查 `pre.next` 指向的节点 `cur` 是否与其后继节点值相同。
   - **如果相同**：说明 `cur` 是重复段的开始。我们用一个循环找到这个重复段的终点，然后让 `pre.next` 直接指向终点之后的那个节点。
   - **如果不相同**：说明 `cur` 是唯一的，我们将 `pre` 向后移动一位。

### 关键点：为什么不立即移动 `pre`？
在删除重复段后，`pre.next` 指向了新的节点，但这个新节点可能又是另一段重复的开始。因此，只有在确定 `pre.next` 是唯一节点时，才移动 `pre`。

### 复杂度分析
- **时间复杂度**：$O(N)$，每个节点最多被访问两次。
- **空间复杂度**：$O(1)$，只使用了常数个指针。

## 📝 进阶：从 83 题到 82 题的风格演变

如果你已经掌握了第 83 题，你会发现 82 题其实只是逻辑上的“小幅变迁”：

### 逻辑对比表

| 特性 | 83 题 (保留重复的一个) | 82 题 (重复的一个不留) |
| :--- | :--- | :--- |
| **判定指针** | `curr` 是安全的，检查 `curr.next` | `pre` 是安全的，检查 `pre.next.next` |
| **删除目标** | 删掉重复的 `next` | 删掉整个重复段 (while 循环) |
| **指针移动** | 只有不重复时移动 `curr` | 只有不重复时移动 `pre` |
| **哑节点** | 不需要 (head 绝对安全) | **必须** (head 也可能被铲除) |

### 统一的代码模板
```python
# 82 题风格（延续自 83）
while pre.next and pre.next.next:
    if pre.next.val == pre.next.next.val:
        # 重复了 -> 铲除逻辑
        val = pre.next.val
        while pre.next and pre.next.val == val:
            pre.next = pre.next.next
    else:
        # 安全 -> 指针后移
        pre = pre.next
```
