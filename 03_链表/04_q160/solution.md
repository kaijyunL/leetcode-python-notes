# LeetCode 160. 相交链表 (Intersection of Two Linked Lists)

## 题目描述
给你两个单链表的头节点 `headA` 和 `headB` ，请你找出并返回两个单链表相交的起始节点。如果两个链表没有交点，返回 `null` 。

---

## 解法一：暴力法 (Brute Force)

### 思路
这是最直观的方法。对于链表 A 中的每一个节点，我们都去遍历一遍链表 B，检查链表 B 中是否存在相同的节点实例。

### 步骤
1. 遍历链表 A。
2. 对于 A 中的每一个节点 `nodeA`，再启动一个循环遍历链表 B。
3. 如果在 B 中找到了一个节点 `nodeB` 使得 `nodeA == nodeB`，则该节点就是交点。
4. 如果遍历完两个链表都没有找到，则返回空。

### 复杂度分析
- **时间复杂度**: $O(M \times N)$，其中 $M$ 和 $N$ 分别是两个链表的长度。
- **空间复杂度**: $O(1)$。

---

## 解法二：哈希集合 (Hash Set)

### 思路
暴力法慢在重复遍历。我们可以利用哈希表来存储已经访问过的节点，从而将查找时间降低到 $O(1)$。

### 步骤
1. 遍历链表 A，将链表 A 的所有节点引用存入一个哈希集合（HashSet）中。
2. 遍历链表 B，对于 B 中的每一个节点，检查它是否已经存在于哈希集合中。
3. 第一个在哈希集合中出现的节点就是它们的交点。
4. 如果遍历完 B 都没有找到，说明不相交。

### 复杂度分析
- **时间复杂度**: $O(M + N)$。
- **空间复杂度**: $O(M)$ 或 $O(N)$，取决于你存哪条链表。

---

## 解法三：双指针法 (Optimal Solution - Two Pointers)

### 思路 (浪漫的双指针)
这是一个非常巧妙的方法。假设链表 A 的非公共部分长度为 $a$，链表 B 的非公共部分长度为 $b$，公共部分长度为 $c$。
- 指针 `pA` 走完 A 后，去走 B 的头：总路程 $a + c + b$。
- 指针 `pB` 走完 B 后，去走 A 的头：总路程 $b + c + a$。

因为 $a + c + b = b + c + a$，所以如果两个链表相交，它们一定会在交点处相遇。如果不相交，$c=0$，它们最终会同时指向 `null`。

### 步骤
1. 创建两个指针 `pA` 和 `pB`，分别指向 `headA` 和 `headB`。
2. 同时移动 `pA` 和 `pB`。
3. 当 `pA` 到达末尾时，将其重置为 `headB`。
4. 当 `pB` 到达末尾时，将其重置为 `headA`。
5. 当 `pA == pB` 时，返回 `pA`。

### 复杂度分析
- **时间复杂度**: $O(M + N)$。
- **空间复杂度**: $O(1)$。
