# LeetCode 141. 环形链表 (Linked List Cycle)

给你一个链表的头节点 `head` ，判断链表中是否有环。

如果链表中有某个节点，可以通过连续跟踪 `next` 指针再次到达，则链表中存在环。 为了表示给定链表中的环，评测系统内部使用整数 `pos` 来表示链表尾连接到链表中的位置（索引从 0 开始）。注意：`pos` 不作为参数进行传递 。仅仅是为了标识链表的实际情况。

如果链表中存在环 ，则返回 `true` 。 否则，返回 `false` 。

---

## 方法一：哈希表 (简单直观)

这是最容易想到的一种方法。当我们遍历链表时，我们可以把遍历过的节点存入一个哈希表。

### 思路
1. 遍历链表中的每个节点。
2. 检查当前节点是否已经在哈希表中。
3. 如果在，说明我们回到了之前访问过的节点，因此链表有环。
4. 如果不在，将当前节点存入哈希表。
5. 如果遍历完整个链表（到达 `null`），说明链表没有环。

### 复杂度分析
- **时间复杂度**：O(n)，其中 n 是链表中的节点数。我们需要访问每个节点一次。
- **空间复杂度**：O(n)，其中 n 是链表中的节点数。我们需要将每个节点都存入哈希表。

---

## 方法二：快慢指针 (最优解法 - Floyd 判圈算法)

为了降低空间复杂度，我们可以使用两个指针，一个快指针，一个慢指针。

### 思路 (龟兔赛跑)
1. 定义两个指针：`slow`（慢指针）和 `fast`（快指针）。
2. 让 `slow` 每次走一步，`fast` 每次走两步。
3. 如果链表中没有环，快指针最终会指向 `null`。
4. 如果链表中有环，快指针最终会追上慢指针。

### 两种起始位置的写法：

#### 写法 A：`fast` 从 `head.next` 开始
*   **写法**：`slow = head`, `fast = head.next`。
*   **跳转条件**：`while slow != fast:`。
*   **特点**：因为初始位置不同，可以直接用不相等作为进入循环的条件。

#### 写法 B：`slow` 和 `fast` 都从 `head` 开始 (推荐)
*   **写法**：`slow = fast = head`。
*   **跳转条件**：`while fast and fast.next:`。
*   **特点**：更加符合直觉（同一起跑线出发）。**注意必须要在循环体内部先移动再判断 `if slow == fast`**。这种写法在处理进阶题（如 142 题找环入口）时数学推导更简单，建议优先掌握。

### 为什么一定会相遇？
假设快指针和慢指针都在环里，快指针相对于慢指针的速度是每次多走 1 步。这意味着每一回合，快慢指针之间的距离就会缩短 1 步。因此，只要有环，它们最终一定会相遇在同一个节点。

### 复杂度分析
- **时间复杂度**：O(n)。如果链表无环，快指针走 n/2 步到达终点。如果链表有环，最坏情况下快指针比慢指针多走一圈也会相遇，总时间仍与 n 成正比。
- **空间复杂度**：O(1)。我们只需要两个指针的额外空间。

---

## 总结
- **初学者**：建议先掌握哈希表法，它符合逻辑思考过程。
- **面试考核**：通常要求 O(1) 的空间复杂度，因此快慢指针是必须要掌握的核心解法。
