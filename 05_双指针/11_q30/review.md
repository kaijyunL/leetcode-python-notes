# LeetCode 30. 串联所有单词的子串

## 题目理解

给你一个字符串 `s` 和一个单词数组 `words`。

`words` 里的每个单词长度相同。要求找出 `s` 中所有起始下标，使得从这个下标开始的一段子串，刚好由 `words` 中所有单词拼接而成。

单词顺序可以任意，但每个单词必须使用一次。

例如：

```text
s = "barfoothefoobarman"
words = ["foo", "bar"]
```

每个单词长度是 `3`，总长度是 `6`。

满足条件的子串有：

```text
"barfoo"，起点 0
"foobar"，起点 9
```

答案：

```text
[0, 9]
```

---

## 方法一：暴力枚举 + 排序比较

### 思路

枚举每个可能的起点 `start`。

从 `start` 开始截取总长度为：

```text
len(words) * len(words[0])
```

的子串。

再按单词长度切分成若干单词，排序后和 `words` 排序结果比较。

如果相等，说明这个起点有效。

### 复杂度

设：

```text
n = len(s)
k = len(words)
w = len(words[0])
```

时间复杂度：

```text
O(n * k log k)
```

空间复杂度：

```text
O(k)
```

## 方法二：暴力枚举 + Counter

### 思路

排序比较可以换成词频比较。

先统计 `words`：

```text
need = Counter(words)
```

每次枚举起点后，按单词长度切分子串，并统计当前窗口里的单词频次。

如果频次和 `need` 相同，就记录起点。

这个方法比排序更贴近最优解，因为这题本质是：

```text
单词频次是否一致
```

### 复杂度

时间复杂度：

```text
O(n * k)
```

空间复杂度：

```text
O(k)
```

代码见：

```text

```

## 方法三：按单词长度分组滑动窗口

这是最适合面试的方法。

### 核心思路

普通字符滑动窗口每次移动 1 个字符。

这题更特殊：所有单词长度都一样，所以窗口可以按“单词”为单位移动。

假设：

```text
word_len = 3
```

那么有效单词边界可能从这些位置开始：

```text
offset = 0
offset = 1
offset = 2
```

分别处理：

```text
0, 3, 6, 9, ...
1, 4, 7, 10, ...
2, 5, 8, 11, ...
```

这样就不会漏掉任何可能起点。

### 窗口维护

用两个计数器：

```text
need    words 的单词频次
window  当前窗口的单词频次
```

右指针每次加入一个单词。

如果加入的单词不在 `need` 里：

```text
当前窗口直接清空，从下一个位置重新开始
```

如果某个单词出现次数超过了 `need`：

```text
移动 left，直到这个单词次数合法
```

当窗口里单词数量等于 `len(words)` 时，说明找到一个答案。

### 面试推荐讲法

面试时可以这样讲：

1. 暴力法枚举每个起点，再检查切出来的单词是否和 `words` 匹配。
2. 匹配本质是单词频次相同，所以用 `Counter`。
3. 因为所有单词长度相同，可以按单词长度做滑动窗口。
4. 需要按 `0 ~ word_len - 1` 的不同偏移量分别滑动，避免漏掉起点。
5. 窗口右端每次加入一个单词。
6. 遇到非法单词就清空窗口。
7. 遇到某个单词次数过多，就移动左端收缩窗口。
8. 窗口单词数等于 `len(words)` 时，记录 `left`。

### 复杂度

时间复杂度：

```text
O(n)
```

每个偏移量下，`left` 和 `right` 都只向右移动。

空间复杂度：

```text
O(k)
```

```text

```

## 方法对比

| 方法 | 思路 | 时间复杂度 | 面试推荐 |
|---|---|---:|---|
| 暴力排序 | 切分单词后排序比较 | `O(n * k log k)` | 不推荐 |
| 暴力 Counter | 切分单词后比较词频 | `O(n * k)` | 过渡理解 |
| 分组滑动窗口 | 按单词长度移动窗口 | `O(n)` | 最推荐 |

## 最终建议

面试最推荐：

```text
按 word_len 分组的滑动窗口
```

关键一句话：

```text
所有单词长度相同，所以窗口可以按一个单词一个单词地滑动。
```
