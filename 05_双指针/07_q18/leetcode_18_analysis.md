# LeetCode 18: 四数之和 (4Sum) 深度解析

本案例文档旨在通过循序渐进的方式，带你从最原始的编码思路进化到最优的双指针解法。

---

## 1. 题目背景
**目标**：给定一个数组 `nums` 和一个目标值 `target`，找出所有唯一的四元组，使得它们的和等于 `target`。
**难点**：
- **去重**：结果集中不能有重复的组合。
- **性能**：暴力解法在数据量较大时会超时。

---

## 2. 解法演进

### 2.1 暴力枚举 (Brute Force)
最简单的方法是四层循环。

*   **思路**：`for i -> for j -> for k -> for l`。
*   **去重**：通常利用 `set` 或排序后跳过重复元素。
*   **复杂度**：$O(N^4)$。
*   **结论**：在 LeetCode 上会 TLE（超出时间限制）。

### 2.2 双指针优化 (Optimal - Two Pointers)
这是解决“n数之和”问题的标准最优方案。

*   **核心逻辑**：
    1.  **排序**：排序是关键，它使得我们可以根据和的大小有目的地移动指针。
    2.  **固定两数**：外层两层循环固定 `nums[i]` 和 `nums[j]`。
    3.  **内层双指针**：在剩余区间内，使用 `left` 和 `right` 指针寻找另外两个数。
*   **去重策略**：
    - 在每一层循环开始时，检查 `if nums[x] == nums[x-1]: continue`。
    - 在找到一个答案后，`left` 和 `right` 也要分别跳过重复值。
*   **复杂度**：$O(N^3)$。

---

## 3. 核心优化：剪枝 (Pruning)
为了达到极致性能，我们需要在每层循环中判断“当前路径是否还有可能成功”。

1.  **最小和判断**：如果当前最小的四个数相加都大于 `target`，后面更大的数更不可能满足，直接 `break` 结束循环。
    ```python
    if nums[i] + nums[i+1] + nums[i+2] + nums[i+3] > target: break
    ```
2.  **最大和判断**：如果当前固定数加上最大的三个数仍小于 `target`，说明当前的固定数太小，直接 `continue` 进入下一次循环寻找更大的数。
    ```python
    if nums[i] + nums[n-3] + nums[n-2] + nums[n-1] < target: continue
    ```

## 4. 代码抽象与工程化 (Abstraction)

在进阶代码实现中，我们将 **两数之和 (Two Sum)** 的逻辑独立出来。这样做有三大好处：
1.  **逻辑解耦**：外层循环只负责“降维”（将 4Sum 变成 3Sum，再变成 2Sum），核心的双指针逻辑在 `twoSum` 函数中。
2.  **易于扩展**：这种结构非常容易改造成通用的 `nSum` 递归函数。
3.  **可读性增强**：代码行数虽然略多，但每一层的职责非常明确。

---

## 5. 重点回顾与通用模型 (K-Sum)
理解了 4Sum 的这种抽象结构，你其实就掌握了解决所有 $K$ 数之和问题的钥匙：
- **2Sum**：直接双指针，$O(N)$。
- **3Sum**：固定 1 个数，调用「2Sum」逻辑，$O(N^2)$。
- **4Sum**：固定 2 个数，调用「2Sum」逻辑，$O(N^3)$。
- **K-Sum**：固定 $K-2$ 个数，最后调用「2Sum」逻辑，$O(N^{K-1})$。
