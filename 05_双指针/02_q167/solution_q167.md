# LeetCode 167. 两数之和 II - 输入有序数组

## 题目描述
给你一个下标从 **1** 开始的整数数组 `numbers` ，该数组已按 **非递减顺序排列** ，请你从数组中找出满足相加之和等于目标数 `target` 的两个数。

返回这两个数的下标 `index1` 和 `index2` ，以长度为 2 的整数数组 `[index1, index2]` 的形式返回。

**注意：**
- 每个输入 **恰好对应一个答案**。
- 你 **不能** 重复使用相同的元素。
- 你的解决方案必须只使用 **常量级额外空间**。

---

## 1. 暴力解法 (Brute Force)
### 思路
最直接的方法是枚举数组中所有可能的两数组合，检查它们的和是否等于目标值。

### 复杂度分析
- **时间复杂度**: $O(n^2)$，其中 $n$ 是数组的长度。我们需要嵌套循环遍历数组。
- **空间复杂度**: $O(1)$，只使用了常数个额外变量。

---

## 2. 二分查找解法 (Binary Search)
### 思路
既然数组是有序的，我们可以固定第一个数 `numbers[i]`，然后在它的右侧部分利用二分查找来寻找是否存在 `target - numbers[i]`。

### 复杂度分析
- **时间复杂度**: $O(n \log n)$。遍历数组需要 $O(n)$，每次遍历进行一次二分查找需要 $O(\log n)$。
- **空间复杂度**: $O(1)$。

---

## 3. 双指针解法 (Two Pointers) —— **面试推荐最优解**
### 思路 (适合小白理解)
这是本题的最优解法，充分利用了数组**有序**这一核心特征。

想象你在一个天平上找两个东西凑成特定重量：
1. 我们放两个指针：`left` 指向数组最左边（最小的数），`right` 指向数组最右边（最大的数）。
2. 计算这两个数的和 `sum = numbers[left] + numbers[right]`。
3. **关键逻辑**：
   - 如果 `sum == target`：找到了！直接返回结果。
   - 如果 `sum < target`：说明和太小了。由于数组是有序的，要想让和变大，我们只能让 `left` 向右移动（换一个更大的数）。
   - 如果 `sum > target`：说明和太大了。要想让和变小，我们只能让 `right` 向左移动（换一个更小的数）。
4. 重复这个过程，直到找到答案。因为题目保证有且仅有一个答案，所以一定能找到。

### 为什么这个方法最适合面试？
1. **效率最高**：时间复杂度达到了线性级别 $O(n)$。
2. **空间节省**：完全符合题目 $O(1)$ 空间的要求。
3. **思路清晰**：双指针是处理有序数组/链表的经典技巧，面试官非常看重对这类模式的掌握。
4. **易于扩展**：掌握了双指针，你可以轻松应对很多变体题目（如三数之和、接雨水等）。

### 复杂度分析
- **时间复杂度**: $O(n)$。每个指针最多走 $n$ 步，两个指针合起来也就遍历了一次数组。
- **空间复杂度**: $O(1)$。
