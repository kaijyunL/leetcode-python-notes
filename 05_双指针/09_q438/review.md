# LeetCode 438. 找到字符串中所有字母异位词

## 题目理解

给你两个字符串：

```text
s
p
```

请在 `s` 中找到所有 `p` 的字母异位词的起始下标。

字母异位词的意思是：

```text
字符种类和每个字符出现次数都一样，只是顺序可以不同。
```

例如：

```text
"abc"
"bac"
"cba"
```

它们互相都是字母异位词。

但：

```text
"abc"
"abb"
```

不是，因为字符次数不同。

---

## 示例

```text
s = "cbaebabacd"
p = "abc"
```

`p` 的长度是 `3`，所以我们只需要看 `s` 中所有长度为 `3` 的子串：

```text
cba  是 abc 的异位词，起点 0
bae  不是
aeb  不是
eba  不是
bab  不是
aba  不是
bac  是 abc 的异位词，起点 6
acd  不是
```

答案是：

```text
[0, 6]
```

这题的关键点是：

```text
只检查长度等于 len(p) 的窗口。
```

---

## 方法一：暴力枚举 + 排序比较

### 思路

最直接的方法是：

1. 枚举 `s` 中每一个长度为 `len(p)` 的子串
2. 把这个子串排序
3. 把 `p` 排序
4. 如果两个排序结果相同，说明它们是字母异位词

例如：

```text
p = "abc"
```

排序后是：

```text
"abc"
```

窗口：

```text
"cba"
```

排序后也是：

```text
"abc"
```

所以 `"cba"` 是 `"abc"` 的异位词。

### 复杂度

设：

```text
n = len(s)
m = len(p)
```

时间复杂度：

```text
O(n * m log m)
```

因为有大约 `n` 个窗口，每个窗口排序需要 `O(m log m)`。

空间复杂度：

```text
O(m)
```

排序窗口时需要额外空间。

### 是否适合面试

不适合作为最终答案。

它适合帮你理解题目：

```text
异位词可以通过排序后是否相等来判断。
```

代码见：

```text
05_双指针/09_q438/brute_force_sort.py
```

## 方法二：暴力枚举 + 计数比较

### 思路

排序不是唯一判断异位词的方法。

异位词更本质的判断是：

```text
每个字符出现次数是否相同。
```

所以我们可以：

1. 先统计 `p` 的字符次数
2. 枚举 `s` 中每个长度为 `m` 的窗口
3. 统计当前窗口的字符次数
4. 如果两个计数字典相等，就记录起点

例如：

```text
p = "abc"
```

计数是：

```text
{"a": 1, "b": 1, "c": 1}
```

窗口：

```text
"cba"
```

计数也是：

```text
{"a": 1, "b": 1, "c": 1}
```

所以命中。

### 复杂度

时间复杂度：

```text
O(n * m)
```

每个窗口重新统计一次，窗口长度是 `m`。

空间复杂度：

```text
O(m)
```

### 是否适合面试

比排序暴力法更接近最优思路，但仍然不是最终推荐。

它的价值是帮你从：

```text
排序比较
```

过渡到：

```text
字符频次比较
```

代码见：

```text
05_双指针/09_q438/brute_force_count.py
```

## 方法三：滑动窗口 + Counter

### 核心思路

方法二的问题是：每次窗口右移，都重新统计整个窗口。

但其实相邻窗口高度重叠。

例如：

```text
s = "cbaebabacd"
p = "abc"
m = 3
```

第一个窗口：

```text
cba
```

第二个窗口：

```text
bae
```

它们只变化了两个字符：

```text
移出 c
加入 e
```

所以我们不需要重新统计整个窗口。

我们只需要维护一个长度为 `m` 的滑动窗口：

```text
窗口右移时：
加入右边新字符
移出左边旧字符
```

然后比较当前窗口计数是否等于 `p` 的计数。

### 为什么窗口长度固定

因为异位词必须和 `p` 使用完全相同的字符。

所以长度一定等于：

```text
len(p)
```

如果窗口长度不同，不可能是异位词。

这就是第 438 题和第 3 题的区别：

```text
第 3 题：窗口长度不固定，要找最长无重复子串
第 438 题：窗口长度固定，就是 len(p)
```

### 复杂度

如果使用 `collections.Counter`，每次比较两个 Counter，在字符集固定为小写字母时，可以看成 `O(26)`，也就是常数。

时间复杂度：

```text
O(n)
```

空间复杂度：

```text
O(1)
```

因为题目通常限定小写英文字母，最多 26 个字符。

代码见：

```text
05_双指针/09_q438/sliding_window_counter.py
```

## 方法四：滑动窗口 + 长度为 26 的数组

这是最适合面试的方法。

### 为什么用数组

题目里的字符串通常只包含小写英文字母。

小写字母只有 26 个：

```text
a 到 z
```

所以可以用长度为 26 的数组记录字符次数。

例如：

```text
count[0] 表示 a 的次数
count[1] 表示 b 的次数
count[2] 表示 c 的次数
...
```

字符转下标：

```python
index = ord(ch) - ord("a")
```

### 做法

设：

```text
m = len(p)
```

我们维护两个数组：

```text
p_count      p 的字符频次
window_count 当前窗口的字符频次
```

先统计：

```text
p 的频次
s 前 m 个字符的频次
```

如果两个数组相同，说明起点 `0` 是答案。

然后窗口每次向右移动一格：

```text
移出 s[i - m]
加入 s[i]
```

每次移动后，如果两个频次数组相同，就记录窗口起点：

```text
i - m + 1
```

### 详细例子

```text
s = "cbaebabacd"
p = "abc"
```

`m = 3`。

先看前 3 个字符：

```text
窗口 = "cba"
```

`"cba"` 的频次和 `"abc"` 一样，所以记录：

```text
0
```

窗口右移一格：

```text
旧窗口: cba
新窗口: bae
```

变化是：

```text
移出 c
加入 e
```

频次不同，不记录。

继续右移，直到窗口：

```text
bac
```

它的频次又和 `"abc"` 一样，所以记录：

```text
6
```

最终答案：

```text
[0, 6]
```

### 面试推荐讲法

面试时可以这样讲：

1. 暴力做法是枚举每个长度为 `len(p)` 的子串，排序后比较，复杂度是 `O(n * m log m)`。
2. 异位词的本质是字符频次相同，所以可以改成频次比较。
3. 每个窗口重新统计频次仍然有重复计算。
4. 因为窗口长度固定为 `len(p)`，可以用滑动窗口。
5. 窗口右移时，只需要移出一个旧字符、加入一个新字符。
6. 题目是小写字母，可以用长度为 26 的数组保存频次。
7. 每次窗口频次和 `p` 频次相等，就把当前窗口起点加入结果。

这套讲法覆盖了：

```text
为什么固定窗口
为什么比较频次
为什么能滑动
为什么数组比字典更适合
复杂度是多少
```

### 复杂度

时间复杂度：

```text
O(n)
```

每个字符最多被加入窗口一次、移出窗口一次。每次比较 26 个位置，可以看作常数。

空间复杂度：

```text
O(1)
```

因为数组长度固定为 26。

代码见：

```text
05_双指针/09_q438/optimal_solution.py
```

## 方法对比

| 方法 | 思路 | 时间复杂度 | 空间复杂度 | 面试推荐 |
|---|---|---:|---:|---|
| 暴力排序 | 每个窗口排序后和 `p` 比较 | `O(n * m log m)` | `O(m)` | 不推荐 |
| 暴力计数 | 每个窗口重新统计频次 | `O(n * m)` | `O(m)` | 过渡理解 |
| 滑动窗口 + Counter | 维护窗口计数 | `O(n)` | `O(1)` | 推荐 |
| 滑动窗口 + 26 数组 | 维护固定数组计数 | `O(n)` | `O(1)` | 最推荐 |

---

## 最终建议

学习时按这个顺序理解：

```text
排序判断异位词
字符频次判断异位词
固定长度滑动窗口
用 26 数组优化计数
```

面试时最推荐写：

```text
滑动窗口 + 长度为 26 的数组
```

因为它直接利用了题目“小写字母”的条件，复杂度清晰，代码也不长。

最关键的一句话是：

```text
异位词看频次，窗口长度固定为 len(p)，每次右移只更新移出和加入的两个字符。
```
