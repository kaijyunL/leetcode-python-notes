# LeetCode 88. 合并两个有序数组 (Merge Sorted Array)

## 题目描述
给你两个按 **非递减顺序** 排列的整数数组 `nums1` 和 `nums2`，另有两个整数 `m` 和 `n` ，分别表示 `nums1` 和 `nums2` 中的元素数目。

请你 **合并** `nums2` 到 `nums1` 中，使合并后的数组同样按 **非递减顺序** 排列。

**注意：** 最终，合并后数组不应由函数返回，而是存储在数组 `nums1` 中。为了应对这种情况，`nums1` 的初始长度为 `m + n`，其中前 `m` 个元素表示应合并的元素，后 `n` 个元素为 0 ，应忽略。`nums2` 的长度为 `n` 。

---

## 解法一：暴力合并后排序 (Naive Approach)

### 思路
最直接的方法是先将 `nums2` 的所有元素复制到 `nums1` 后半部分的空闲位置（即 `m` 之后的位置），然后对整个 `nums1` 进行排序。

### 复杂度分析
- **时间复杂度**: $O((m+n)\log(m+n))$。主要是排序消耗的时间。
- **空间复杂度**: $O(\log(m+n))$。排序算法通常需要一定的辅助栈空间。

---

## 解法二：正向双指针 (Two Pointers - Forward)

### 思路
由于两个数组都已经是有序的，我们可以使用两个指针分别指向 `nums1` 和 `nums2` 的开头。每次比较两个指针指向的数值，将较小的那个放入一个**辅助数组**中。

### 复杂度分析
- **时间复杂度**: $O(m+n)$。只需要遍历两个数组各一次。
- **空间复杂度**: $O(m+n)$。需要一个额外的数组来存储结果。

---

## 解法三：逆向双指针 (Two Pointers - Backward) —— **面试推荐**

### 为什么这个最适合面试？
1. **时间最优**: $O(m+n)$ 的线性时间。
2. **空间最优**: $O(1)$ 的额外空间。它利用了 `nums1` 后半部分原本就是“空闲”的特性，避免了额外空间。
3. **考察点**: 这种“从后往前”处理的思想在很多数组类题目中非常经典，能体现你对题目限制条件的深度利用。

### 详细解析（小白也能懂）
想象一下，如果你从前往后填，你会担心把 `nums1` 还没处理到的数据给覆盖掉。比如 `nums1 = [1, 2, 3, 0, 0, 0]`，`nums2 = [2, 5, 6]`。如果你把 `nums2` 的 `2` 往 `nums1` 的第一个位置放，原本的 `1` 怎么办？

**但是，如果你从后往前填：**
- `nums1` 的最后面（索引为 `m+n-1` 的位置）是空的，随便填。
- 我们比较 `nums1[m-1]`（当前 `nums1` 最大的数）和 `nums2[n-1]`（当前 `nums2` 最大的数）。
- 谁大，谁就去填在 `nums1` 的最后面。
- 填完一个，对应的指针就往前移一步，目标位置的指针也往前移一步。
- **关键点**: 这样永远不会覆盖到 `nums1` 还没比较过的有效数字，因为后面永远有足够的坑位等你去填。

### 特殊情况处理
如果 `nums2` 还没填完，而 `nums1` 的有效数字已经用完了，直接把 `nums2` 剩余部分抄到 `nums1` 前面即可。
如果 `nums1` 还没填完，而 `nums2` 已经用完了，由于 `nums1` 本身就是有序的，剩下的位置就是它们该在的位置，不需要动。

### 复杂度分析
- **时间复杂度**: $O(m+n)$。
- **空间复杂度**: $O(1)$。直接在原数组上操作。
