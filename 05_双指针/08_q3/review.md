# LeetCode 3. 无重复字符的最长子串

## 题目理解

给你一个字符串 `s`，请你找出其中不含重复字符的最长子串长度。

注意是：

```text
子串
```

不是子序列。

子串必须是连续的。

例如：

```text
s = "abcabcbb"
```

其中 `"abc"` 是一个没有重复字符的子串，长度是 `3`。

答案是：

```text
3
```

再比如：

```text
s = "bbbbb"
```

最长无重复子串只能是 `"b"`，长度是：

```text
1
```

---

## 方法一：暴力枚举所有子串

### 思路

最直接的方法是枚举所有子串。

一个子串由两个边界决定：

```text
left  起点
right 终点
```

我们可以枚举所有 `left` 和 `right`，得到每一个子串，然后判断这个子串里有没有重复字符。

如果没有重复，就更新最大长度。

### 判断子串是否重复

可以用集合 `seen`。

从 `left` 到 `right` 遍历每个字符：

1. 如果字符不在 `seen` 中，就加入
2. 如果字符已经在 `seen` 中，说明这个子串有重复字符

例如：

```text
"abc"
```

没有重复。

```text
"abca"
```

第二个 `"a"` 出现时，发现 `"a"` 已经在集合里，说明重复。

### 复杂度

时间复杂度：

```text
O(n^3)
```

因为有两层循环枚举子串，再用一层循环检查子串是否重复。

空间复杂度：

```text
O(min(n, m))
```

其中 `m` 是字符集大小，集合里最多存放一个无重复子串中的字符。

### 是否适合面试

不适合作为最终答案。

它适合用来理解题目本质：

```text
在所有连续子串里，找没有重复字符的最长长度。
```

但 `O(n^3)` 太慢。

代码见：

```text

```

## 方法二：枚举起点，向右扩展

### 思路

暴力法慢，是因为它会重复检查很多子串。

我们可以稍微优化一下：

固定一个起点 `left` 后，往右扩展 `right`。

如果当前字符没出现过，就继续扩展。

如果当前字符已经出现过，说明从这个 `left` 出发，再往右也不可能形成无重复子串，可以直接停止当前起点。

例如：

```text
s = "abcabcbb"
```

当 `left = 0` 时：

```text
a
ab
abc
abca  出现重复 a，停止
```

这时不需要继续看：

```text
abcab
abcabc
...
```

因为已经重复了。

### 复杂度

时间复杂度：

```text
O(n^2)
```

空间复杂度：

```text
O(min(n, m))
```

### 是否适合面试

这个方法比纯暴力好，但仍然不是最优。

它适合当作从暴力法过渡到滑动窗口的中间思路。

代码见：


## 方法三：滑动窗口 + 哈希集合

这是最适合面试的方法。

### 核心思想

维护一个窗口：

```text
s[left:right + 1]
```

让这个窗口始终保持：

```text
没有重复字符
```

我们用两个指针：

```text
left  窗口左边界
right 窗口右边界
```

再用一个集合 `seen` 记录当前窗口里有哪些字符。

### 怎么移动

遍历 `right`，把字符一个个尝试加入窗口。

如果：

```text
s[right] 不在 seen 中
```

说明加入后仍然没有重复，可以直接加入窗口，然后更新答案。

如果：

```text
s[right] 已经在 seen 中
```

说明当前窗口里已经有这个字符了。

这时不能直接加入它，需要先移动 `left`，把窗口左边的字符一个个移出去，直到 `s[right]` 不再重复。

然后再加入 `s[right]`。

### 例子

以：

```text
s = "abcabcbb"
```

开始：

```text
left = 0
seen = {}
res = 0
```

`right = 0`，字符是 `"a"`：

```text
窗口: "a"
seen = {"a"}
res = 1
```

`right = 1`，字符是 `"b"`：

```text
窗口: "ab"
seen = {"a", "b"}
res = 2
```

`right = 2`，字符是 `"c"`：

```text
窗口: "abc"
seen = {"a", "b", "c"}
res = 3
```

`right = 3`，字符又是 `"a"`。

此时 `"a"` 已经在窗口 `"abc"` 里了，不能直接加入。

所以移动 `left`：

```text
移出 s[left] = "a"
left 从 0 变成 1
```

现在窗口里没有 `"a"` 了，再加入当前 `"a"`：

```text
窗口: "bca"
seen = {"b", "c", "a"}
res = 3
```

继续这样移动，最终答案是 `3`。

### 为什么这是双指针

`left` 和 `right` 都只会向右移动，不会回头。

每个字符最多：

```text
被 right 加入窗口一次
被 left 移出窗口一次
```

所以整体是线性的。

### 复杂度

时间复杂度：

```text
O(n)
```

空间复杂度：

```text
O(min(n, m))
```

其中 `m` 是字符集大小。

### 面试推荐讲法

面试时可以这样讲：

1. 暴力法是枚举所有子串，再判断是否有重复，复杂度是 `O(n^3)`。
2. 进一步可以固定起点向右扩展，遇到重复就停止，复杂度降到 `O(n^2)`。
3. 最优解使用滑动窗口。
4. 用 `left` 和 `right` 表示当前无重复窗口。
5. 用集合 `seen` 保存窗口里的字符。
6. 当 `s[right]` 不重复时，加入窗口并更新答案。
7. 当 `s[right]` 重复时，不断移动 `left`，直到窗口里不再包含 `s[right]`。
8. 因为每个字符最多进窗口一次、出窗口一次，所以时间复杂度是 `O(n)`。

## 方法四：滑动窗口 + 哈希表记录下标

### 思路

第三种方法遇到重复字符时，会一个一个移动 `left`。

我们还可以更直接一点：

用字典 `last_index` 记录每个字符上一次出现的位置。

当遍历到 `right` 时，如果 `s[right]` 之前出现过，并且它上一次出现的位置还在当前窗口里：

```text
last_index[s[right]] >= left
```

那么可以直接把 `left` 跳到：

```text
last_index[s[right]] + 1
```

这样窗口就跳过了重复字符。

### 例子

```text
s = "abba"
```

遍历到第二个 `"b"` 时：

```text
窗口是 "ab"
left = 0
right = 2
```

`"b"` 上一次出现位置是 `1`，还在当前窗口里。

所以：

```text
left = 1 + 1 = 2
```

窗口变成：

```text
"b"
```

然后继续处理当前字符。

注意最后一个 `"a"`：

它上一次出现位置是 `0`，但当前 `left = 2`。

位置 `0` 已经不在当前窗口里，所以不能把 `left` 退回去。

因此要写：

```python
left = max(left, last_index[ch] + 1)
```

### 是否最适合面试

这个版本也非常适合面试。

如果你刚开始刷题，我更建议先掌握“滑动窗口 + 集合”版本，因为它更直观。

如果你已经理解窗口不变量，可以写“滑动窗口 + 哈希表下标”版本，代码更短。

代码见：


## 方法对比

| 方法 | 思路 | 时间复杂度 | 空间复杂度 | 面试推荐 |
|---|---|---:|---:|---|
| 暴力枚举 | 枚举所有子串并检查重复 | `O(n^3)` | `O(min(n, m))` | 不推荐 |
| 起点扩展 | 固定起点，向右扩展到重复为止 | `O(n^2)` | `O(min(n, m))` | 过渡理解 |
| 滑动窗口 + 集合 | 维护无重复窗口 | `O(n)` | `O(min(n, m))` | 推荐 |
| 滑动窗口 + 下标 | 重复时直接跳过旧位置 | `O(n)` | `O(min(n, m))` | 推荐，熟练后使用 |

## 最终建议

学习时按这个顺序理解：

```text
暴力枚举所有子串
固定起点向右扩展
滑动窗口维护无重复区间
用下标表直接跳 left
```

面试时优先讲：

```text
滑动窗口 + 集合
```

因为它最容易解释清楚窗口含义。

如果面试官追问能否写得更短，再给出：

```text
滑动窗口 + 哈希表记录下标
```

最关键的一句话是：

```text
窗口里始终没有重复字符，right 负责扩张，left 负责在重复时收缩。
```
