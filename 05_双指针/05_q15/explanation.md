# LeetCode 第 15 题：三数之和 (3Sum)

## 题目描述
给你一个整数数组 `nums` ，判断是否存在三元组 `[nums[i], nums[j], nums[k]]` 满足 `i != j`、`i != k` 且 `j != k` ，同时还满足 `nums[i] + nums[j] + nums[k] == 0` 。请你返回所有和为 `0` 且不重复的三元组。

**注意：** 答案中不可以包含重复的三元组。

---

## 1. 暴力解法 (Brute Force)

### 思路
最直接的想法是使用三层嵌套循环，遍历数组中所有可能的三元组，并检查它们的和是否为 0。为了避免重复的三元组，我们可以先对数组进行排序，或者在找到三元组后将其排序并放入一个集合（Set）中去重。

### 复杂度分析
- **时间复杂度：** $O(N^3)$，其中 $N$ 是数组长度。三层循环嵌套。
- **空间复杂度：** $O(N)$，用于存储结果和去重集合。

---

## 2. 最优解法：排序 + 双指针 (Sorting + Two Pointers) —— **面试推荐**

### 核心思路 (适合面试细讲)

这是本题的最优解法，也是面试官最希望看到的思路。

1.  **预处理：排序**
    - 首先将数组按升序排列。排序的好处是：
        - 方便去重。
        - 我们可以根据当前和的大小，决定如何移动指针（变大或变小）。

2.  **固定一个，寻找两个**
    - 遍历数组，每次固定一个数 `nums[i]`。
    - 在 `nums[i]` 之后的区间内，使用 **左指针 `L`** 和 **右指针 `R`** 寻找两个数，使得 `nums[i] + nums[L] + nums[R] == 0`。

3.  **双指针移动逻辑**
    - 计算当前和 `total = nums[i] + nums[L] + nums[R]`。
    - 如果 `total == 0`：
        - 找到一个结果，存入列表。
        - **关键点：** 为了去重，左右指针需要跳过所有重复的数值。
        - `L` 向右移，`R` 向左移。
    - 如果 `total < 0`：
        - 说明和太小了，需要增大数值。由于数组已排序，应将左指针 `L` 向右移。
    - 如果 `total > 0`：
        - 说明和太大了，需要减小数值。应将右指针 `R` 向左移。

4.  **关键：去重逻辑 (小白必读)**
    - **外层循环去重：** 如果当前的 `nums[i]` 和上一次的 `nums[i-1]` 相同，直接跳过，因为以这个数开头的组合已经找过了。
    - **双指针内部去重：** 在找到一个和为 0 的组合后，如果 `nums[L]` 和下一个数相同，`L` 继续移动；如果 `nums[R]` 和上一个数相同，`R` 继续移动。

### 为什么这个方法最适合面试？
- **效率高：** 将 $O(N^3)$ 降到了 $O(N^2)$。
- **展示基本功：** 考察了排序、双指针和细节处理（去重逻辑）。
- **逻辑清晰：** 相比于使用哈希表去重，双指针的空间复杂度更低（$O(1)$ 额外空间，不计输出），且逻辑更容易口头表述。

### 复杂度分析
- **时间复杂度：** $O(N^2)$，外层循环 $O(N)$，内层双指针 $O(N)$。
- **空间复杂度：** $O(1)$ 或 $O(log N)$，取决于排序算法的实现空间，不需要额外的哈希表去重。

---

## 总结
面试时，先提到排序，再解释如何通过固定一个数将问题转化为“两数之和”，最后重点讲解如何通过指针移动和跳过重复项来保证结果的唯一性和高效性。
