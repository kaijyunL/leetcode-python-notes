# LeetCode 80. 删除有序数组中的重复项 II

## 题目描述

给你一个有序数组 `nums` ，请你 **原地** 删除重复出现的元素，使得出现次数超过两次的元素只出现两次 ，返回删除后数组的新长度。

不要使用额外的数组空间，你必须在 **原地** 修改输入数组并在使用 $O(1)$ 额外空间的条件下完成。

**示例 1：**
> 输入：`nums = [1,1,1,2,2,3]`
> 输出：`5, nums = [1,1,2,2,3]`
> 解释：函数应返回新长度 `length = 5`，并且原数组的前五个元素被修改为 `1, 1, 2, 2, 3`。

---

## 解法一：简单暴力法（使用辅助空间或频繁移动）

### 思路
最直观的想法是遍历数组，统计每个数字出现的次数。如果某个数字出现次数超过 2 次，就将其后面的所有元素向前移动一位。或者为了简单理解，先统计频率，再重构数组。

由于题目要求原地修改，暴力法通常涉及大量的元素移动，时间复杂度较高。

### 复杂度分析
- **时间复杂度**：$O(n^2)$，因为每次删除元素都需要移动后续所有元素。
- **空间复杂度**：$O(1)$（如果原地移动）或 $O(n)$（如果使用辅助哈希表）。

---

## 解法二：双指针法（最优解 & 面试推荐）

这是最适合面试的解法，逻辑清晰且效率最高。

### 核心思想
利用数组**已经有序**的特点。我们可以维护两个指针：
1. `slow` 指针：代表处理后的数组的末尾（即下一个待插入的位置）。
2. `fast` 指针：用于遍历整个原数组。

**关键判断条件：**
我们需要判断 `fast` 指向的当前元素是否可以放入 `slow` 位置。
规则是：只要当前元素 `nums[fast]` 与 `nums[slow - 2]` 不相同，说明这个元素还没有出现超过两次，可以放入。

### 详细步骤说明（小白版）
1. **初始化**：如果数组长度小于等于 2，直接返回长度即可，因为最多允许重复两次。
2. **起始位置**：设置 `slow = 2`。因为前两个元素（下标 0 和 1）无论是否重复，都一定符合“最多重复两次”的规则。
3. **遍历**：从 `fast = 2` 开始向后扫描：
   - 比较 `nums[fast]` 和 `nums[slow - 2]`。
   - **为什么要比 `slow - 2`？**
     - 因为 `slow` 是我们要填入的位置。
     - 如果 `nums[fast] == nums[slow - 2]`，说明在 `slow` 之前已经有两个相同的数字了（即 `nums[slow-2]` 和 `nums[slow-1]` 都和 `nums[fast]` 相等，因为数组是有序的）。
     - 如果 `nums[fast] != nums[slow - 2]`，说明把 `nums[fast]` 填在 `slow` 位置后，不会违反“最多重复两次”的规则。
4. **赋值与移动**：如果符合条件，就执行 `nums[slow] = nums[fast]`，然后 `slow` 往前走一步。

### 复杂度分析
- **时间复杂度**：$O(n)$，只需一次遍历。
- **空间复杂度**：$O(1)$，只使用了两个变量。

---

## 为什么解法二最适合面试？
1. **效率极高**：$O(n)$ 时间复杂度和 $O(1)$ 空间复杂度是该题的极限性能。
2. **代码优雅**：逻辑精炼，利用了有序数组的特性。
3. **通用性强**：这种“快慢指针+跨步比较”的方法可以推广到“允许重复 K 次”的通用场景（只需将 `2` 替换为 `K`）。
