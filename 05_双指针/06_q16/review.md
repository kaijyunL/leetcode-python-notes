# LeetCode 16. 最接近的三数之和

## 题目理解

给你一个整数数组 `nums` 和一个目标值 `target`，请你从数组中选出三个数，使它们的和最接近 `target`。

返回这个最接近的三数之和。

例如：

```text
nums = [-1, 2, 1, -4], target = 1
```

可以选：

```text
[-1, 2, 1]，和为 2
```

`2` 和 `target = 1` 的距离是 `1`，这是最接近的结果，所以返回：

```text
2
```

这题和 LeetCode 15「三数之和」很像，但目标不一样：

```text
第 15 题：找所有和为 0 的三元组
第 16 题：找一个最接近 target 的三数之和
```

因此第 16 题不需要返回所有组合，也不需要专门做结果去重。我们只需要维护一个当前最接近的 `res`。

---

## 方法一：暴力枚举

### 思路

最直接的想法是：

1. 枚举第一个数 `i`
2. 枚举第二个数 `j`
3. 枚举第三个数 `k`
4. 计算三数之和 `total`
5. 如果 `total` 比之前记录的答案更接近 `target`，就更新答案

判断哪个和更接近 `target`，看绝对差：

```python
abs(total - target)
```

如果这个差值更小，说明更接近。

代码见：

```text
05_双指针/06_q16/brute_force.py
```

### 复杂度

时间复杂度：

```text
O(n^3)
```

因为三层循环枚举所有三元组。

空间复杂度：

```text
O(1)
```

只用了几个变量。

### 是否适合面试

暴力法适合用来解释最原始的思路，但不适合作为最终面试答案。

它的价值是帮助你先想清楚：

```text
这题本质是在所有三元组里面，找一个 sum，使 abs(sum - target) 最小。
```

但如果数组长度比较大，`O(n^3)` 会超时。

## 方法二：排序 + 双指针

这是最适合面试的方法。

### 核心思路

先排序数组。

排序后，我们固定一个数 `nums[i]`，然后在它右边用两个指针找另外两个数：

```text
i       left                         right
↓       ↓                            ↓
[-4,   -1,    1,    2]
```

每次计算：

```python
total = nums[i] + nums[left] + nums[right]
```

然后和 `target` 比较。

如果：

```text
total == target
```

说明已经完全命中目标，距离是 `0`，不可能有更好的答案，可以直接返回。

如果：

```text
total < target
```

说明当前三数之和偏小。因为数组已经排序，想让和变大，就应该移动 `left`：

```python
left += 1
```

如果：

```text
total > target
```

说明当前三数之和偏大。想让和变小，就应该移动 `right`：

```python
right -= 1
```

这就是双指针能成立的关键：排序之后，移动左指针会让和变大，移动右指针会让和变小。

### 为什么要先排序

不排序时，`left` 和 `right` 的移动没有明确意义。

例如 `total` 偏小，如果数组是乱的，你不知道移动哪个指针能让和变大。

排序后就有了单调性：

```text
nums[left + 1] >= nums[left]
nums[right - 1] <= nums[right]
```

所以：

```text
left 右移：三数之和可能变大
right 左移：三数之和可能变小
```

这让我们可以从暴力枚举的 `O(n^3)` 优化到 `O(n^2)`。

### 详细过程

以：

```text
nums = [-1, 2, 1, -4], target = 1
```

排序后：

```text
[-4, -1, 1, 2]
```

初始化一个答案：

```text
res = nums[0] + nums[1] + nums[2] = -4 + -1 + 1 = -4
```

当前答案和目标的距离是：

```text
abs(-4 - 1) = 5
```

开始枚举固定数。

### 第一轮：固定 `nums[0] = -4`

```text
i = 0
left = 1
right = 3

[-4, -1, 1, 2]
  ↑   ↑     ↑
  i  left  right
```

当前三数之和：

```text
-4 + -1 + 2 = -3
```

距离：

```text
abs(-3 - 1) = 4
```

比之前的距离 `5` 更小，所以更新：

```text
res = -3
```

因为 `-3 < target`，说明和偏小，所以 `left` 右移。

```text
[-4, -1, 1, 2]
  ↑      ↑  ↑
  i    left right
```

当前三数之和：

```text
-4 + 1 + 2 = -1
```

距离：

```text
abs(-1 - 1) = 2
```

继续更新：

```text
res = -1
```

还是偏小，`left` 继续右移，此时 `left == right`，这一轮结束。

### 第二轮：固定 `nums[1] = -1`

```text
i = 1
left = 2
right = 3

[-4, -1, 1, 2]
      ↑  ↑  ↑
      i left right
```

当前三数之和：

```text
-1 + 1 + 2 = 2
```

距离：

```text
abs(2 - 1) = 1
```

比当前答案 `-1` 的距离 `2` 更接近，所以更新：

```text
res = 2
```

因为 `2 > target`，说明和偏大，所以 `right` 左移。

此时 `left == right`，结束。

最终返回：

```text
2
```

### 为什么不需要像第 15 题那样去重

第 15 题要返回所有不重复的三元组，所以必须去重。

第 16 题只返回一个数字：

```text
最接近 target 的三数之和
```

重复的三元组最多只是让我们重复计算同一个和，不会影响最终答案。

所以为了小白更容易理解，也为了面试时更稳定，代码里可以不写去重逻辑。

当然，如果你已经很熟练，也可以跳过重复的固定值来减少一点无效计算，但这不是本题的核心。

### 面试推荐讲法

面试时可以这样讲：

1. 暴力做法是枚举所有三元组，时间复杂度是 `O(n^3)`。
2. 为了优化，我先对数组排序。
3. 排序后固定一个数，把问题变成在右侧区间找两个数。
4. 用 `left` 和 `right` 两个指针维护另外两个数。
5. 每次计算三数之和，并更新当前最接近 `target` 的答案。
6. 如果当前和小于 `target`，说明和偏小，移动 `left` 让和变大。
7. 如果当前和大于 `target`，说明和偏大，移动 `right` 让和变小。
8. 如果刚好等于 `target`，直接返回，因为距离已经是 `0`。

这套讲法能覆盖：

```text
为什么排序
为什么双指针
为什么这样移动指针
为什么可以提前返回
复杂度是多少
```

### 复杂度

时间复杂度：

```text
O(n^2)
```

排序是 `O(n log n)`，外层循环是 `O(n)`，内层双指针整体是 `O(n)`，所以主导复杂度是：

```text
O(n^2)
```

空间复杂度：

```text
O(1)
```

如果不把排序消耗的内部空间算进去，只用了常数个变量。

## 两种方法对比

| 方法 | 思路 | 时间复杂度 | 空间复杂度 | 是否推荐面试 |
|---|---|---:|---:|---|
| 暴力枚举 | 三层循环枚举所有三元组 | `O(n^3)` | `O(1)` | 不推荐作为最终答案 |
| 排序 + 双指针 | 固定一个数，另外两个数用双指针找 | `O(n^2)` | `O(1)` | 推荐 |

## 最终建议

学习时先理解暴力法，因为它能帮你看清题目本质：

```text
找一个三数之和，让它和 target 的差值最小。
```

面试时重点写排序加双指针。

最关键的一句话是：

```text
排序后，当前和小了就移动 left 变大，当前和大了就移动 right 变小。
```

只要这句话想明白了，第 16 题就和第 15 题一样，都是典型的双指针题。
