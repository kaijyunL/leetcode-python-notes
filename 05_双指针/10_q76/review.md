# LeetCode 76. 最小覆盖子串

## 题目理解

给你两个字符串：

```text
s
t
```

要求在 `s` 中找到一个最短子串，使这个子串包含 `t` 中所有字符，并且字符次数也要满足。

例如：

```text
s = "ADOBECODEBANC"
t = "ABC"
```

答案是：

```text
"BANC"
```

因为 `"BANC"` 包含了：

```text
A: 1
B: 1
C: 1
```

并且是满足条件的最短子串。

注意：

```text
t = "AABC"
```

窗口里必须至少有：

```text
A: 2
B: 1
C: 1
```

不是只看字符种类，还要看次数。

---

## 方法一：暴力枚举所有子串

### 思路

枚举 `s` 的所有子串，判断它是否覆盖 `t`。

判断覆盖的方法：

1. 统计 `t` 的字符频次
2. 统计当前子串的字符频次
3. 对 `t` 中每个字符，检查当前子串是否数量足够

如果当前子串覆盖 `t`，就更新最短答案。

### 复杂度

设：

```text
n = len(s)
m = len(t)
```

时间复杂度：

```text
O(n^3)
```

因为有 `O(n^2)` 个子串，每次统计和检查需要额外时间。

空间复杂度：

```text
O(m)
```

不适合最终答案，只适合理解题意。

## 方法二：枚举左端点，向右扩展

### 思路

固定左端点 `left`，向右移动 `right`。

当窗口第一次覆盖 `t` 时，记录答案，然后停止当前 `left`。

因为继续扩大 `right` 只会让窗口更长，不可能得到以当前 `left` 开头的更短答案。

### 复杂度

时间复杂度：

```text
O(n^2)
```

空间复杂度：

```text
O(m)
```

### 是否适合面试

比纯暴力好，但仍然不是最优。

它适合作为过渡：先理解“窗口覆盖 `t`”这个概念。

代码见：

```text

```

## 方法三：滑动窗口

这是最适合面试的方法。

### 核心思路

用两个指针维护窗口：

```text
s[left:right + 1]
```

右指针 `right` 负责扩大窗口，让窗口尽量覆盖 `t`。

当窗口已经覆盖 `t` 后，左指针 `left` 负责收缩窗口，尝试让答案更短。

整体过程：

```text
right 向右扩张，直到窗口满足条件
left 向右收缩，直到窗口不再满足条件
继续扩张 right
```

### 需要维护什么

用两个计数器：

```text
need      t 中每个字符需要多少个
window    当前窗口中这些字符有多少个
```

再用一个变量：

```text
valid
```

表示当前窗口中，已经有多少种字符满足了需要的次数。

当：

```text
valid == len(need)
```

说明窗口已经覆盖了 `t`。

### 为什么 `valid` 有用

如果每次都完整比较两个 Counter，会有额外开销。

`valid` 可以让我们快速知道当前窗口是否满足条件。

例如：

```text
t = "AABC"
need = {"A": 2, "B": 1, "C": 1}
```

当窗口里：

```text
A >= 2
B >= 1
C >= 1
```

这三种字符都满足，`valid == 3`，窗口有效。

### 详细过程

```text
s = "ADOBECODEBANC"
t = "ABC"
```

先统计：

```text
need = {"A": 1, "B": 1, "C": 1}
```

右指针不断扩张。

当窗口第一次包含 `"A" "B" "C"` 时，窗口是：

```text
"ADOBEC"
```

这是一个有效窗口，但不一定最短。

于是开始移动 `left`，缩小窗口。

如果移出某个关键字符导致窗口不再覆盖 `t`，就停止收缩，再继续移动 `right`。

最终会找到：

```text
"BANC"
```

### 面试推荐讲法

面试时可以这样讲：

1. 暴力法枚举所有子串，再判断是否覆盖 `t`，复杂度高。
2. 这题适合滑动窗口，因为我们要找一个连续子串。
3. 用 `right` 扩大窗口，直到窗口覆盖 `t`。
4. 一旦覆盖，就用 `left` 收缩窗口，尝试更新最短答案。
5. 用 `need` 记录目标字符频次，用 `window` 记录窗口频次。
6. 用 `valid` 记录有多少种字符已经满足频次要求。
7. 当 `valid == len(need)` 时，当前窗口有效。
8. 每个字符最多进窗口一次、出窗口一次，所以时间复杂度是 `O(n)`。

### 复杂度

时间复杂度：

```text
O(n + m)
```

空间复杂度：

```text
O(m)
```

## 方法对比

| 方法 | 思路 | 时间复杂度 | 面试推荐 |
|---|---|---:|---|
| 暴力枚举 | 枚举所有子串并检查覆盖 | `O(n^3)` | 不推荐 |
| 枚举左端点 | 固定左端点，右端扩到覆盖 | `O(n^2)` | 过渡理解 |
| 滑动窗口 | 右扩到覆盖，左收缩到不覆盖 | `O(n + m)` | 最推荐 |

## 最终建议

面试最推荐：

```text
滑动窗口 + need/window 计数 + valid
```

最关键的一句话：

```text
right 负责让窗口满足条件，left 负责在满足条件时尽量缩短窗口。
```
