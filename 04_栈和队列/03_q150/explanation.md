# LeetCode 150. 逆波兰表达式求值

## 题目在说什么

给你一个字符串数组 `tokens`，它表示一个**逆波兰表达式**（后缀表达式）。

你需要返回这个表达式的计算结果。

常见运算符只有 4 个：
- `+`
- `-`
- `*`
- `/`

这里最重要的一点是：
- 除法结果要**向 0 截断**
- 比如 `6 / -132 = 0`
- 在 Python 里通常写成 `int(a / b)`

---

## 什么是逆波兰表达式

平时我们写的是中缀表达式：
- `2 + 1`
- `(2 + 1) * 3`

逆波兰表达式会把运算符放到后面：
- `2 1 +`
- `2 1 + 3 *`

也就是：
- **先放数字**
- **等需要计算时再放运算符**

例如：
- `tokens = ["2", "1", "+", "3", "*"]`

计算过程：
1. 先算 `2 + 1 = 3`
2. 再算 `3 * 3 = 9`

答案就是 `9`

---

## 先抓住这题的核心规律

当我们从左到右遍历 `tokens` 时：
- 遇到数字，先记下来
- 遇到运算符，说明现在可以拿最近的两个数字来计算

所以这题天然适合用**后进先出**的数据结构，也就是**栈**。

但为了让你循序渐进理解，我们先从最直接的暴力想法开始。

---

## 方法一：暴力模拟替换

### 思路

我们可以把 `tokens` 当成一个会不断变化的列表。

每一轮从左到右扫描：
- 找到一个运算符
- 它前面的两个位置一定是两个操作数
- 直接算出结果
- 再把这三个元素替换成一个结果

重复这个过程，直到列表只剩一个数字。

### 为什么这个方法容易想到

因为它最接近“手算”：
- 看见一个可计算的小片段
- 先把它算掉
- 再继续往后处理

### 举例

`["2", "1", "+", "3", "*"]`

第一轮找到 `+`：
- `2 + 1 = 3`
- 列表变成 `["3", "3", "*"]`

第二轮找到 `*`：
- `3 * 3 = 9`
- 列表变成 `["9"]`

最终答案是 `9`

### 代码

```python
class Solution:
    def evalRPN_bruteforce(self, tokens):
        arr = tokens[:]

        while len(arr) > 1:
            for i, token in enumerate(arr):
                if token not in {"+", "-", "*", "/"}:
                    continue

                a = int(arr[i - 2])
                b = int(arr[i - 1])

                if token == "+":
                    value = a + b
                elif token == "-":
                    value = a - b
                elif token == "*":
                    value = a * b
                else:
                    value = int(a / b)

                arr = arr[: i - 2] + [str(value)] + arr[i + 1 :]
                break

        return int(arr[0])
```

### 复杂度

- 时间复杂度：`O(n^2)`
- 空间复杂度：`O(n)`

### 缺点

- 每次计算后都要重新拼接列表
- 会反复扫描
- 数据量一大就不划算

### 面试评价

这个方法**可以当成思路起点**，但**不适合作为最终面试答案**。

因为面试官一般希望你进一步想到：

> 既然每次都只关心“最近的两个数字”，那为什么不直接把最近的数字存起来？

这就自然过渡到栈。

---

## 方法二：栈

### 思路

用一个栈 `stack`：
- 遇到数字：入栈
- 遇到运算符：弹出两个数字计算，再把结果压回栈中

为什么栈特别合适？

因为“最近出现、还没参与计算的数字”正好就在栈顶。

### 关键细节

假设当前运算符是 `-` 或 `/`：
- 第一次弹出的是右操作数 `b`
- 第二次弹出的是左操作数 `a`

所以一定要写成：
- `a - b`
- `a / b`

不能写反。

### 过程演示

`tokens = ["4", "13", "5", "/", "+"]`

1. 读到 `4`，入栈：`[4]`
2. 读到 `13`，入栈：`[4, 13]`
3. 读到 `5`，入栈：`[4, 13, 5]`
4. 读到 `/`，弹出 `5` 和 `13`
   - `13 / 5 = 2`
   - 压回栈：`[4, 2]`
5. 读到 `+`，弹出 `2` 和 `4`
   - `4 + 2 = 6`
   - 压回栈：`[6]`

最终答案：`6`

## 方法三：栈 + 运算封装（同样是最优解）

### 思路

本质上还是栈。

只是把不同运算封装一下，让主流程更统一：
- 如果是数字就入栈
- 如果是运算符就调用对应规则

这样代码会更整洁一些。

### 什么时候用这个版本

- 你已经完全理解了栈解法
- 想让代码更易扩展
- 或者你平时写题更喜欢“映射表”风格

### 代码

```python
class Solution:
    def evalRPN_optimized(self, tokens):
        stack = []
        ops = {
            "+": lambda a, b: a + b,
            "-": lambda a, b: a - b,
            "*": lambda a, b: a * b,
            "/": lambda a, b: int(a / b),
        }

        for token in tokens:
            if token not in ops:
                stack.append(int(token))
                continue

            b = stack.pop()
            a = stack.pop()
            stack.append(ops[token](a, b))

        return stack[-1]
```

### 复杂度

- 时间复杂度：`O(n)`
- 空间复杂度：`O(n)`

### 和方法二的区别

- 算法本质一样
- 复杂度一样
- 只是写法更抽象一点

如果是面试，通常**方法二更稳**，因为更直观。

## 最适合面试的是哪一种

**最适合面试的是：方法二，栈。**

原因很简单：
- **思路自然**：从题目规律直接推到栈
- **表达清楚**：数字入栈，运算符出栈计算
- **复杂度优秀**：时间 `O(n)`，空间 `O(n)`
- **代码不花哨**：面试官容易快速看懂
- **细节好展开**：你可以重点解释操作数顺序和除法截断

## 面试版详细讲解（给小白也能看懂）

### 第一步：为什么想到栈

你把题目从左到右看，会发现一个现象：

- 数字先出现
- 运算符后出现
- 一个运算符出现时，只会用到**最近的两个还没被使用的数字**

这句话非常关键。

“最近的两个还没被使用的数字”，其实就是一种**后进先出**关系。

而后进先出，最经典的数据结构就是**栈**。

---

### 第二步：栈里放什么

栈里只放数字。

遍历 `tokens`：
- 如果当前是数字，就转成整数后压栈
- 如果当前是运算符，就从栈里弹出两个数来计算

---

### 第三步：为什么要先弹 `b`，再弹 `a`

这是这题最容易错的地方。

比如表达式：
- `4 2 -`

它真正表示的是：
- `4 - 2`

遍历到 `-` 时，栈顶是 `2`，下面是 `4`。

所以：
- `b = stack.pop()` 得到的是 `2`
- `a = stack.pop()` 得到的是 `4`

真正运算必须写：
- `a - b`

如果你写成 `b - a`，结果就错了。

除法也一样：
- 必须是 `a / b`

---

### 第四步：为什么不能写 `a // b`

因为 Python 的 `//` 是**向下取整**，而题目要求的是**向 0 截断**。

例如：
- `-3 // 2 = -2`
- 但题目想要的是 `-1`

所以这题里要写成：

```python
int(a / b)
```

这样才是向 0 截断。

---

### 第五步：最后为什么返回栈顶

因为每遇到一个运算符，我们都会：
- 取出两个数字
- 算出一个结果
- 再把结果放回去

整个过程会不断把表达式“缩起来”。

遍历结束后，栈里只剩一个数，它就是最终答案。

---

## 面试时你可以这样说

你可以用下面这种话术回答：

> 这题适合用栈。因为逆波兰表达式从左到右遍历时，数字会先出现，运算符后出现；当遇到运算符时，需要取最近的两个数字参与计算，这正好符合栈的后进先出特性。遍历数组时，数字就压栈；运算符就弹出两个数，注意先弹出的是右操作数，再弹出的是左操作数，计算结果再压回栈。最后栈中剩下的唯一元素就是答案。时间复杂度是 `O(n)`，空间复杂度是 `O(n)`。

这段话很适合面试时直接讲。

## 面试版代码

```python
class Solution:
    def evalRPN(self, tokens):
        stack = []

        for token in tokens:
            if token not in {"+", "-", "*", "/"}:
                stack.append(int(token))
                continue

            b = stack.pop()
            a = stack.pop()

            if token == "+":
                stack.append(a + b)
            elif token == "-":
                stack.append(a - b)
            elif token == "*":
                stack.append(a * b)
            else:
                stack.append(int(a / b))

        return stack[-1]
```

## 三种方法怎么选

- **想先理解题目**：先看方法一，知道这题本质是在不断“合并表达式”
- **想通过面试**：重点掌握方法二
- **想让代码更整洁**：可以再看方法三

如果你时间有限，**只要把方法二真正吃透，这题就够用了。**

## 最后总结

这题表面是在算表达式，实际上是在考你能不能识别：

> “最近两个待计算数字” 这一类问题，往往适合用栈。

你只要牢牢记住 3 件事：
- 数字入栈
- 运算符弹两个数再计算
- 注意减法、除法的左右顺序

这题就稳了。
