# LeetCode 32. 最长有效括号 (Longest Valid Parentheses)

## 题目描述
给你一个只包含 '(' 和 ')' 的字符串，找出最长有效（格式正确且连续）括号子串的长度。

---

## 循序渐进解法分析

### 1. 暴力解法 (Brute Force)
**思路**：
检查所有可能的偶数长度子串。对于每个子串，利用栈来判断它是否为有效的括号。
- 时间复杂度：$O(n^3)$。生成所有子串 $O(n^2)$，验证每个子串 $O(n)$。
- 空间复杂度：$O(n)$，验证时需要的栈空间。

### 2. 栈 (Stack) - **面试最推荐解法**
**思路**：
我们不只是用栈来匹配括号，而是用栈来**保存下标**。
1. 首先将 `-1` 压入栈中。这作为一个“参照物”，代表上一个未匹配的右括号的位置（初始化为起点前一个位置）。
2. 遍历字符串：
   - 如果遇到 '('，将其下标压入栈。
   - 如果遇到 ')'，弹出栈顶元素。
     - 如果弹出后栈为空，说明当前这个 ')' 没有匹配的左括号，我们将它的下标压入栈，作为新的“参照物”。
     - 如果弹出后栈不为空，当前有效长度 = `当前下标 - 栈顶下标`。

**为什么适合面试？**
- 逻辑清晰，是栈这种数据结构在括号匹配问题上的升华。
- 时间复杂度 $O(n)$，空间复杂度 $O(n)$。

### 3. 动态规划 (Dynamic Programming)
**思路**：
定义 `dp[i]` 为以第 `i` 个字符结尾的最长有效括号的长度。
- 如果 `s[i] == '('`，`dp[i] = 0`（以左括号结尾不可能有效）。
- 如果 `s[i] == ')'`：
  - 如果 `s[i-1] == '('`，形如 `...()`，则 `dp[i] = dp[i-2] + 2`。
  - 如果 `s[i-1] == ')'` 且 `s[i - dp[i-1] - 1] == '('`，形如 `...((...))`，则 `dp[i] = dp[i-1] + 2 + dp[i - dp[i-1] - 2]`。

- 时间复杂度：$O(n)$。
- 空间复杂度：$O(n)$。

### 4. 贪心 / 双计数器 (Two Counters) - **最优空间解法**
**思路**：
利用两个计数器 `left` 和 `right` 分别记录左括号和右括号的数量。
1. **从左向右遍历**：遇到 '('，`left++`；遇到 ')'，`right++`。
   - 如果 `left == right`，更新最大长度 `max = 2 * right`。
   - 如果 `right > left`，说明右括号多了，无法匹配，将 `left` 和 `right` 重置为 0。
2. **从右向左遍历**（防止 `((()` 这种情况）：遇到 ')'，`right++`；遇到 '('，`left++`。
   - 如果 `left == right`，更新最大长度 `max = 2 * left`。
   - 如果 `left > right`，重置为 0。

- 时间复杂度：$O(n)$（遍历两次）。
- 空间复杂度：**$O(1)$**。

## 面试重点详解：栈解法 (小白保姆级教程)

很多同学能想到用栈匹配 `()`，但不知道怎么算**长度**。这道题的精髓在于：**栈里面存的不是字符，而是下标。**

### 1. 核心逻辑：找“参照物”
想象你在量一段绳子的长度，你需要一个**起点**。
- 我们在栈底预放一个 `-1`，它代表“上一个没被匹配的右括号的位置”。
- 每次匹配成功，`当前下标 - 栈顶下标` 就是这段有效括号的长度。

### 2. 为什么栈底要放个 -1？
如果字符串是 `()`：
- `i=0` 时，`(` 入栈，栈变成 `[-1, 0]`。
- `i=1` 时，`)` 来了，弹出栈顶的 `0`，栈剩下 `[-1]`。
- 长度 = `1 - (-1) = 2`。瞧，刚好算对！如果不放 `-1`，你弹空了之后就没法做减法了。

### 3. 动图级分步模拟
以 `s = ") ( ) )"` 为例：

1.  **初始化**：栈 `stack = [-1]`，`max_ans = 0`。
2.  **i = 0, s[0] = ')'** (右括号)：
    -   弹出栈顶 `-1`。栈空了！
    -   说明这个 `)` 是多余的，没人跟它配对。
    -   **关键动作**：把它的下标 `0` 压入栈。现在 `stack = [0]`。
    -   *意义：从现在起，`0` 成了新的“参照物”，后面的有效长度都从 `0` 之后开始算。*
3.  **i = 1, s[1] = '('** (左括号)：
    -   下标 `1` 入栈。现在 `stack = [0, 1]`。
4.  **i = 2, s[2] = ')'** (右括号)：
    -   弹出栈顶 `1`。栈不为空。
    -   计算长度：`i - 栈顶` = `2 - 0 = 2`。
    -   更新 `max_ans = 2`。
5.  **i = 3, s[3] = ')'** (右括号)：
    -   弹出栈顶 `0`。栈空了！
    -   说明这个 `)` 也是多余的。
    -   **关键动作**：把它的下标 `3` 压入栈。现在 `stack = [3]`。
    -   *意义：以后的长度要从 `3` 之后算起。*

### 4. 总结成口诀
- **遇到左括号**：无脑入栈（记录位置）。
- **遇到右括号**：
    - 先弹出（尝试匹配）。
    - 弹完**栈空了**：说明当前右括号是“显眼包”，没救了，把它存进去当“新地标”。
    - 弹完**还有人**：说明匹配成功！用`当前下标 - 栈顶下标`算出刚才那段有多长。
