# LeetCode 43. 字符串相乘 (Multiply Strings)

这道题要求我们将两个表示为数字的字符串相乘，并返回结果字符串。题目明确要求：**不能使用任何内置的大数类型（如 BigInteger）或直接将输入转换为整数处理。**

我们可以完全模拟平时手算乘法（竖式乘法）的过程来解决这个问题。下面从“普通竖式乘法”到“优化后的竖式乘法”循序渐进地讲解。

---

## 重点注意 (边界情况)
无论哪种解法，都要注意一个特殊情况：如果 `num1` 或 `num2` 中有任何一个为 `"0"`，乘积必然为 `"0"`。应该在一开始特判处理。

---

## 方法一：普通竖式乘法（结合字符串相加）

### 思路
这是最直观的方法，完全模拟我们小学时学过的竖式乘法：
1. 遍历 `num2` 的每一位数字，用这一位去乘整个 `num1`，得到一个局部乘积字符串。
2. 因为 `num2` 的不同位代表不同的量级（个位、十位、百位等），所以我们在得到局部乘积后，需要在它的末尾补充对应数量的 `0`。
3. 将所有的局部乘积字符串累加起来。这是通过 **“字符串相加”** （即另一道经典题 LeetCode 415 题）的逻辑来实现的。因为不能转成大整数，我们需要按位在字符串层面去完成加法的进位。

### 复杂度分析
这种写法思路简单，但是其中涉及到大量的字符串操作（不断的倒序拼接、按位加法计算和反转字符串）。特别是在进行累加时，字符串会变得越来越长，导致时间复杂度和空间复杂度相对较高。
*   时间复杂度为主循环内包含了字符串的加法运算，大约为 $O(M \times N + N^2)$。
*   空间复杂度由于产生了许多较长的临时字符串，大约也偏高。

---

## 方法二：优化竖式乘法（使用数组累加）

### 思路
在方法一中，我们将每一位乘法的结果当成独立的字符串，最后再做加法。但我们可以更聪明地**把加法进位过程直接合并到乘法计算图的位置中**！

如果我们仔细观察两个数字字符串中每一位在相乘后所对应的结果位置，会发现一个绝妙且规律的对应关系：
**乘数 `num1` 的第 `i` 位和乘数 `num2` 的第 `j` 位相乘，它们的乘积必定会直接影响（累加）到最终结果集的第 `i + j` 和第 `i + j + 1` 位！**

具体操作如下：
1. 若 `num1` 长度为 `m`，`num2` 长度为 `n`，它们乘积的总长最多只有 `m + n` 位。我们开辟一个长度为 `m + n` 的整数数组 `res`，初始化各项为 `0`。（这步用数组操作替代了复杂的字符串累积）
2. 从右往左（个位到最高位）进行双重循环，分别遍历 `num1` 和 `num2` 这俩字符串提取数字。
3. 对于从个位提取出的 `num1[i]` 和 `num2[j]`，它们相乘得到的 `mul = x * y` 加上下一位的原进位存留 `res[i + j + 1]`，就是当前位置应该有的**临时总和**。
4. 我们将临时总和的**个位数**（取模 `10`）留在它本来的位置：`res[i + j + 1]`。
5. 将临时总和的**十位数**作为进位（整除 `10`）直接累加给前一个更高位：`res[i + j]`。
6. 最后循环全结束时，处理得到的结果 `res` 数组。它最前面可能含有前导 `0`（例如计算 `2 * 3 = 06` 时，在长度为 $m+n=2$ 的数组里装的会是 `[0, 6]`），我们需要将前导 `0` 剔除，再把剩下数字转回并拼接成字符串。

### 复杂度分析
*   **时间复杂度**：$O(M \times N)$。其中 $M$ 和 $N$ 分别是字符串 `num1` 和 `num2` 的长度。我们需要嵌套两层循环计算它们每位数组合。
*   **空间复杂度**：$O(M + N)$。主要需要一个长度为 $M+N$ 的临时数组存储位置数值。
