# LeetCode 14. 最长公共前缀 (Longest Common Prefix)

## 题目描述
编写一个函数来查找字符串数组中的最长公共前缀。
如果不存在公共前缀，返回空字符串 `""`。

**示例 1：**
- 输入：`strs = ["flower","flow","flight"]`
- 输出：`"fl"`

**示例 2：**
- 输入：`strs = ["dog","racecar","car"]`
- 输出：`""`
- 解释：输入不存在公共前缀。

---

## 解法一：横向扫描 (Horizontal Scanning)

### 核心思路
这是最直观的方法。我们可以先求出前两个字符串的公共前缀，再用这个前缀去和第三个字符串求公共前缀，依此类推。
就像是在做“接力赛”，不断缩小可能的前缀范围。

### 算法步骤
1. 假设第一个字符串 `strs[0]` 就是初始的最长公共前缀 `prefix`。
2. 遍历数组中的后续字符串。
3. 对于每个字符串，不断缩短 `prefix`，直到该字符串以 `prefix` 开头。
4. 如果 `prefix` 变为空字符串，说明没有公共前缀，直接返回。

### 复杂度分析
- **时间复杂度**：$O(S)$，其中 $S$ 是所有字符串中字符数量的总和。
- **空间复杂度**：$O(1)$，只需要常数空间存储前缀。

---

## 解法二：纵向扫描 (Vertical Scanning)

### 核心思路
比起一个一个字符串地对比，我们也可以“一列一列”地对比。即比较所有字符串的第 1 个字符，再比较所有字符串的第 2 个字符，直到出现不匹配。

### 算法步骤
1. 遍历第一个字符串的每一个字符索引 `i`。
2. 检查其他所有字符串在该索引处的字符。
3. 如果索引 `i` 超过了某个字符串的长度，或者某个字符串在索引 `i` 处的字符与第一个字符串不一致，则返回当前截取的前缀。
4. 如果遍历完第一个字符串都匹配，则第一个字符串本身就是最长公共前缀。

### 优点
相比横向扫描，纵向扫描在某些情况下更快。例如：如果数组末尾有一个非常短的字符串，纵向扫描会在早期就发现匹配失败，而横向扫描可能需要处理完前面的长字符串。

---

## 解法三：排序比较法 (Sorting Approach) - 最推荐的 Pythonic 解法

### 核心思路
我们可以先对字符串数组进行排序。排序后，最长公共前缀一定只取决于**字典序最小（第一个）**和**字典序最大（最后一个）**的两个字符串。
因为如果第一个 and 最后一个拥有共同前缀，那么夹在它们中间的所有字符串也一定拥有这个前缀。

### 算法步骤
1. 对数组 `strs` 进行排序。
2. 比较第一个字符串 `first` 和最后一个字符串 `last`。
3. 找出它们两者的最长公共前缀并返回。

### 复杂度分析
- **时间复杂度**：$O(N \log N \cdot L)$，其中 $N$ 是字符串个数，$L$ 是平均长度（排序的开销）。但在 Python 中，对于某些测试用例，这种方法的代码极其简洁高效。

## 解法四：Pythonic 魔法解法 (zip + set)

### 核心思路
利用 Python 的 `zip(*strs)` 将字符串按列对齐打包，再利用 `set` 判断每一列是否只有一种字符。

### 代码实现
```python
def longestCommonPrefix(strs):
    res = ""
    for chars in zip(*strs):
        if len(set(chars)) == 1:
            res += chars[0]
        else:
            break
    return res
```

### 复杂度分析
- **时间复杂度**：$O(S)$，且代码极短。
- **缺点**：`zip` 会在内存中创建元组，对于字符串极多的情况，空间开销略大于纵向扫描。

## 🏆 面试最强策略

在面试中，建议按以下顺序展现：

1. **稳健起手 (纵向扫描)**：先写出纵向扫描的解法。解释它是最优的时间复杂度 $O(S)$ 且具备“早期退出”的特性，非常稳健。
2. **语言进阶 (zip 魔法)**：写完后补充一句：“如果是 Python 开发，利用 `zip` 和 `set` 可以写出更 Pythonic 的精简方案”。
3. **逻辑转换 (排序法)**：如果面试官探讨不同思路，指出排序后只比首尾是有趣的数学视角。

**避坑指南**：不要首选排序法作为唯一答案，除非你能清楚解释为什么 $O(N \log N)$ 在某些场景下可以接受。面试官通常更看重 $O(S)$ 的线性解。

---

## 总结
- **追求性能 & 严谨**：选择 **解法二 (纵向扫描)**。
- **追求代码优雅 & 技巧**：选择 **解法四 (zip+set)**。
