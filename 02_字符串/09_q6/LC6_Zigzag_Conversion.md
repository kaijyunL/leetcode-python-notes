# LeetCode 6: Z 字形变换 (Zigzag Conversion)

这道题考察的是找规律和字符串操作能力。我们在寻找解法时，可以根据“暴力模拟 ➔ 优化存储 ➔ 探究数学规律”的递进思路来理解。

以下将渐进式地为你介绍三种解法：

---

## 解法一：二维矩阵模拟 (最暴力的思路)

### 思路讲解
最直观的想法就是“照猫画虎”。题目是怎么描述 Z 字形（其实是反写的 N 字形）排列的，我们就老老实实创建一个二维矩阵（表格）把它画出来。
1. 首先，我们判断边界状况，如果 `numRows = 1` 或者字符串长度本来就不超过行数，那么形态根本不会变，直接返回原字符串即可。
2. 了解 Z 字形的周期：每次**向下**走到最底，然后**向右上**走到最顶，这就是一个完整的周期。
   - 一个周期的总字符数 `t = 2 * numRows - 2`；
   - 每个周期会占用矩阵的宽度为 `numRows - 1` 列。
3. 我们可以算出一个足够容纳所有字符的二维矩阵大小，全部填充为空字符串。
4. 顺序遍历原始字符串 `s`，模拟指针在矩阵里的运动轨迹（向下填，然后向右上填），一步步放入字符。
5. 最后按行遍历矩阵，把非空的字符拼起来。

**复杂度分析：**
- **时间复杂度:** $O(r \times n)$，$r$ 为行数，$n$ 为字符串长度。因为我们需要遍历整个二维矩阵。
- **空间复杂度:** $O(r \times n)$，二维矩阵带来的高额空间花销。

---

## 解法二：按行排序 (大幅优化空间与逻辑)

### 思路讲解
在第一种方法中，大量的矩阵空间其实是填了空字符串，这是一种极大的浪费。我们最终的目的是**把不同位置的字符，按顺序放入到对应的行里**，最后再把每一行拼接起来！
那么，我们不需要完整的矩阵背景，只需要 `numRows` 个字符串。
1. 创建一个长度为 `numRows` 的字符串数组 `rows`。
2. 我们遍历原字符串，只需要知道当前遍历到的字符属于哪一行（索引为 `i`）。
3. 这个行索引 `i` 的变化规律非常简单：
   - 从第 0 行开始，`i` 逐渐增加（表示向下移动）。
   - 每次触底（碰到最后一行 `numRows - 1`），方向就需要反转为向上，`i` 逐渐减小。
   - 每次触顶（返回到第 0 行），方向也需要反转为向下。
4. 将字符拼接到对应的 `rows[i]` 中。
5. 遍历结束后，把 `rows` 里面所有的字符串按顺序合并。

**复杂度分析：**
- **时间复杂度:** $O(n)$，只遍历了一遍原始字符串，拼接时间也是 $O(n)$。
- **空间复杂度:** $O(n)$，创建了用来存放最终答案同等量级的字符串。比起矩阵，空间节省极其可观！

---

## 解法三：直接构造查数学规律 (找下标的最优解)

### 思路讲解
既然 Z 字形的填写是一个具有**周期性**的物理过程，那么同一个周期的字符、乃至相同行的字符在原字符串中的下标，必定有严谨的数学规律！我们要是能直接算出它最终出现在每一行的哪些下标位置，就能不再模拟过程，而是直接“提取”字符了。

1. 一个完整的周期（一列向下 + 一段向右上的斜线）所包含的字符数量是 `t = 2 * numRows - 2`。
2. 我们直接按要求的最终状态，逐行（`i` 从 `0` 到 `numRows - 1`）来提取：
   - 每个周期的起始下标我们可以用 `j` 来表示（`j = 0, t, 2t, 3t...`）。
   - 对于第 `i` 行：
     - 在当前周期的**向下直线**上，字符的原下标就是 `j + i`。
     - 如果当前不是第一行，也不是最后一行，那么在当前周期的**向上斜线**上，还会穿过一个字符。它的原下标是什么呢？一个周期长度是 `t`，如果从下一个周期起点 `j + t` 往回退 `i` 步，就是我们要找的斜拉位置，即 `j + t - i`。
3. 按照这个坐标规律，直接遍历每一行，把计算出来的下标对应的字符拼好。

**复杂度分析：**
- **时间复杂度:** $O(n)$，刚好每次遍历命中一个最终该排好的字符，没有任何冗余动作。
- **空间复杂度:** $O(1)$，除了最终所需的返回值字符串数组（Python 中的字符合并需求），没有使用额外的空间，非常优雅。
