# LeetCode 125. 验证回文串 (Valid Palindrome)

## 题目描述

如果在将所有大写字符转换为小写字符、并移除所有非字母数字字符之后，短语正着读和反着读都一样。则可以认为该短语是一个 **回文串** 。

字母及数字都属于字母数字字符。

**示例 1：**
- 输入: `s = "A man, a plan, a canal: Panama"`
- 输出: `true`
- 解释: `"amanaplanacanalpanama"` 是回文串。

**示例 2：**
- 输入: `s = "race a car"`
- 输出: `false`
- 解释: `"raceacar"` 不是回文串。

---

## 解法一：筛选后反转（简单暴力）

### 思路
这是最直观的方法：
1. **清洗字符串**：遍历原字符串，只保留字母和数字，并全部转为小写。
2. **判断回文**：利用 Python 的切片功能 `[::-1]` 将清洗后的字符串反转，看是否与原字符串相等。

### 复杂度分析
- **时间复杂度**：$O(n)$，其中 $n$ 是字符串的长度。我们需要遍历一次字符串来构造新字符串，再进行一次反转比较。
- **空间复杂度**：$O(n)$，我们需要创建一个新的字符串来存储处理后的字符。

---

## 解法二：双指针（最优解）

### 思路
为了优化空间复杂度（达到 $O(1)$），我们可以直接在原字符串上使用双指针，而不需要额外存储空间。

1. **设定指针**：定义两个指针 `left` 和 `right`，分别指向字符串的开头和结尾。
2. **跳过非字符**：使用 `left` 向右移，`right` 向左移，直到它们指向字母或数字。
3. **比较字符**：
   - 将两个指针指向的字符转为小写。
   - 如果不相等，则直接返回 `False`。
   - 如果相等，则 `left` 继续向右，`right` 继续向左，移动到下一个位置。
4. **终止条件**：当 `left >= right` 时，说明所有有效字符都已经比较完毕，返回 `True`。

### 复杂度分析
- **时间复杂度**：$O(n)$，每个字符最多被访问一次。
- **空间复杂度**：$O(1)$，只使用了常数级别的额外空间（两个指针）。

---

## 总结
- **方法一** 代码简洁，但在处理极大字符串时会消耗较多内存。
- **方法二** 是面试中的标准答案，体现了对空间效率的追求。
