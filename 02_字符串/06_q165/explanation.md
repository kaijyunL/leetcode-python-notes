# LeetCode 165. 比较版本号 (Compare Version Numbers)

## 题目描述
给你两个版本号 `version1` 和 `version2` ，请你比较它们。

版本号由一个或多个修订号组成，各修订号之间用点 `.` 隔开。每个修订号由多位数字组成，可能包含前导零。每个版本号至少包含一个字符。修订号从左到右编号，下标从 0 开始，最左边的修订号下标为 0 ，下一个修订号下标为 1 ，以此类推。例如，`2.5.33` 和 `0.1` 都是有效的版本号。

比较版本号时，请按从左到右的顺序依次比较它们的修订号。比较修订号时，只需比较忽略前导零后的整数值。也就是说，修订号 `1` 和 `001` 相等。如果版本号没有指定某个下标处的修订号，则该修订号视为 `0` 。例如，版本 `1.0` 小于版本 `1.1` ，因为它们下标为 0 的修订号相同，而下标为 1 的修订号分别为 `0` 和 `1` ，`0 < 1` 。

返回规则如下：
- 如果 `version1 > version2` 返回 `1`
- 如果 `version1 < version2` 返回 `-1`
- 除此之外返回 `0`

---

## 循序渐进解法

### 方法一：分割字符串 (Split & Compare)
这是最直观的方法。我们将字符串按照 `.` 进行分割，得到修订号列表。然后遍历列表进行逐一比较。

#### 核心逻辑：
1. 使用 `split('.')` 将两个字符串切分为字符串列表。
2. 确定两个列表中较长的一个，作为遍历的长度。
3. 在遍历过程中，如果某个版本已经遍历完，则将其修订号补为 `0`。
4. 将字符串形式的修订号转换为整数以忽略前导零。
5. 逐一对比，一旦不相等就返回结果。

#### 复杂度分析：
- **时间复杂度**：$O(n + m)$，其中 $n$ 和 $m$ 分别是两个字符串的长度。我们需要遍历字符串进行分割。
- **空间复杂度**：$O(n + m)$，需要额外的空间来存储分割后的修订号列表。

---

### 方法二：双指针 (Two Pointers) - 最优解
方法一虽然简单，但需要额外的空间来存储分割后的列表。在高性能要求的场景下，我们可以使用双指针直接在原字符串上操作，将空间复杂度降低到 $O(1)$。

#### 核心逻辑：
1. 使用两个指针 `i` 和 `j` 分别指向 `version1` 和 `version2` 的起始位置。
2. 在 `while` 循环中，分别解析出当前的修订号。
3. 解析修订号时，跳过 `.` 字符，将连续的数字字符转换为一个整数。
4. 比较两个整数，若不相等则直接返回结果。
5. 移动指针到下一个修订号的起点，继续循环。

#### 复杂度分析：
- **时间复杂度**：$O(n + m)$，两个指针各遍历一次相应的字符串。
- **空间复杂度**：$O(1)$，只使用了常数个变量，不需要存储分割后的数组。

---

## 总结建议
- **入门理解**：先掌握方法一，这能帮助你快速理解题意，处理好“前导零”和“位数不等”的情况。
- **进阶要求**：在面试中，面试官往往会考察你对空间复杂度的优化意识，此时双指针法是最佳选择。

## 核心细节深度解析 (FAQ)

### Q1：双指针法是如何处理前导零的？
**答**：通过“按位累加”的逻辑自然消除。
在代码中，我们使用 `v1 = v1 * 10 + int(version1[i])` 来解析数字。
- 如果修订号是 `"001"`，前两次循环中 `v1` 始终是 `0 * 10 + 0 = 0`。
- 只有遇到非零数字 `1` 时，`v1` 才会变成 `1`。
- 这种方式无需显式去除零，直接在解析过程中就完成了“去零”和“字符串转整数”的操作。

### Q2：当两个版本号长度不等（修订号缺失）时怎么处理？
**答**：通过“外层 `or` 循环 + 默认值 `0`”来处理。
- **外层循环**：使用 `while i < n1 or j < n2`，只要任意一个版本号还没结束，就继续比较。
- **默认补零**：在每一轮开始前，将 `v1` 和 `v2` 重置为 `0`。如果 `version1` 已经跑完了（`i >= n1`），它就不会进入解析循环，`v1` 就会保持默认值 `0`。
- **效果**：这完美契合了题目中“缺失修订号视为 `0`”的规定（例如 `1.0` 和 `1.0.0` 会被判定为相等）。
