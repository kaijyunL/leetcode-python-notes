# LeetCode 78 — 子集（Subsets）

## 题目

给你一个整数数组 `nums`，数组中的元素互不相同。请返回该数组所有可能的子集。

子集不能包含重复结果，返回顺序不限。

### 示例

```
输入: nums = [1, 2, 3]
输出: [[], [1], [2], [1,2], [3], [1,3], [2,3], [1,2,3]]
```

也可以返回其他顺序，只要包含全部子集即可。

---

## 核心理解

对子集问题来说，每个元素只有两种选择：

```
选
不选
```

如果数组长度是 `n`，每个元素都有 2 种选择，所以一共有：

```
2^n
```

个子集。

比如：

```
nums = [1, 2, 3]
```

每个元素选或不选：

```
1 不选，2 不选，3 不选 -> []
1 选，   2 不选，3 不选 -> [1]
1 不选，2 选，   3 不选 -> [2]
1 选，   2 选，   3 不选 -> [1, 2]
...
```

所以这题的本质不是“找一个答案”，而是“枚举所有选择路径”。

---

## 解法一览

| 解法 | 思路 | 时间复杂度 | 空间复杂度 | 是否推荐面试 |
|---|---|---|---|---|
| 1. 位掩码枚举 | 用二进制表示每个元素选不选 | O(n * 2^n) | O(n * 2^n) | 可作为暴力理解 |
| 2. 迭代扩展 | 每来一个数，把它加到已有子集后面 | O(n * 2^n) | O(n * 2^n) | 简洁好写 |
| 3. 回溯 | DFS 枚举每条选择路径 | O(n * 2^n) | O(n * 2^n) | **面试推荐 ✅** |

> 注意：返回所有子集本身就需要 `2^n` 个结果，所以不可能低于指数级复杂度。这里的“最优”主要指思路通用、可扩展、适合后续回溯题。

---

## 解法 1：位掩码枚举

### 思路

用一个二进制数表示某个子集。

如果 `nums = [1, 2, 3]`，长度 `n = 3`，那么一共有 `2^3 = 8` 个状态：

```
000 -> []
001 -> [1]
010 -> [2]
011 -> [1, 2]
100 -> [3]
101 -> [1, 3]
110 -> [2, 3]
111 -> [1, 2, 3]
```

每一位表示对应元素是否被选择：

```
第 0 位 -> nums[0]
第 1 位 -> nums[1]
第 2 位 -> nums[2]
```

如果某一位是 `1`，就把对应元素加入当前子集。

### 代码

```python
class Solution:
    def subsets(self, nums: list[int]) -> list[list[int]]:
        n = len(nums)
        ans = []

        for mask in range(1 << n):
            subset = []
            for i in range(n):
                if mask & (1 << i):
                    subset.append(nums[i])
            ans.append(subset)

        return ans
```

### 复杂度

一共有 `2^n` 个状态，每个状态最多检查 `n` 个元素。

时间复杂度：

```
O(n * 2^n)
```

结果本身需要存 `2^n` 个子集，每个子集最多长度为 `n`。

空间复杂度：

```
O(n * 2^n)
```

如果不算返回结果，额外空间是 `O(n)`，用于临时 `subset`。

### 评价

这个方法很适合用来理解“每个元素选或不选”的本质。

但面试里它有两个问题：

1. 位运算对部分面试官和读代码的人不如回溯直观
2. 后续遇到组合、排列、去重、剪枝时，位掩码不如回溯通用

## 解法 2：迭代扩展

### 思路

从空集开始：

```
ans = [[]]
```

每遇到一个新数字，就把它加到已有的每个子集后面，形成新的子集。

以 `nums = [1, 2, 3]` 为例：

初始：

```
[[]]
```

加入 `1`：

```
已有: [[]]
新增: [[1]]
结果: [[], [1]]
```

加入 `2`：

```
已有: [[], [1]]
新增: [[2], [1, 2]]
结果: [[], [1], [2], [1, 2]]
```

加入 `3`：

```
已有: [[], [1], [2], [1, 2]]
新增: [[3], [1, 3], [2, 3], [1, 2, 3]]
结果: [[], [1], [2], [1, 2], [3], [1, 3], [2, 3], [1, 2, 3]]
```

### 代码

```python
class Solution:
    def subsets(self, nums: list[int]) -> list[list[int]]:
        ans = [[]]

        for num in nums:
            new_subsets = []
            for subset in ans:
                new_subsets.append(subset + [num])
            ans.extend(new_subsets)

        return ans
```

也可以写成更短的列表推导：

```python
ans += [subset + [num] for subset in ans]
```

但学习阶段建议先写展开版，更容易看清楚过程。

### 复杂度

时间复杂度：

```
O(n * 2^n)
```

空间复杂度：

```
O(n * 2^n)
```

### 评价

这个方法非常简洁，也很适合写在 LeetCode 上。

但它更像“构造规律”，不是标准回溯模板。后续做 #90 子集 II、#77 组合、#39 组合总和时，还是回溯法更容易迁移。

## 解法 3：回溯法 ⭐ 面试推荐

### 思路

回溯可以理解成：用 DFS 枚举所有可能的选择路径。

对 `nums = [1, 2, 3]`，我们从左到右决定每个位置后面还能选哪些数。

核心模板：

```python
def backtrack(start):
    ans.append(path[:])

    for i in range(start, len(nums)):
        path.append(nums[i])
        backtrack(i + 1)
        path.pop()
```

这里有三个关键变量：

| 变量 | 含义 |
|---|---|
| `ans` | 存放所有子集 |
| `path` | 当前正在构造的子集 |
| `start` | 下一轮选择从哪个下标开始 |

### 为什么一进入函数就加入 `path[:]`？

因为子集问题里，每一个中间路径都是一个合法答案。

比如：

```
path = []
path = [1]
path = [1, 2]
path = [1, 2, 3]
```

这些都是合法子集，所以每次进入 `backtrack` 时，都要把当前路径加入结果。

注意必须写：

```python
ans.append(path[:])
```

不能写：

```python
ans.append(path)
```

因为 `path` 后面会继续 `append` 和 `pop`，如果直接存 `path` 本身，结果里的所有元素会引用同一个列表，最后就错了。

### 为什么递归时传 `i + 1`？

因为每个元素只能用一次。

当前选择了 `nums[i]` 后，下一层只能从 `i + 1` 后面继续选，避免重复使用同一个元素，也避免出现 `[2, 1]` 这种和 `[1, 2]` 本质重复的子集。

### 以 `[1, 2, 3]` 为例

递归过程大致是：

```
[]
├── [1]
│   ├── [1, 2]
│   │   └── [1, 2, 3]
│   └── [1, 3]
├── [2]
│   └── [2, 3]
└── [3]
```

对应结果：

```
[[], [1], [1, 2], [1, 2, 3], [1, 3], [2], [2, 3], [3]]
```

顺序和示例可以不同，题目允许任意顺序返回。

### 代码

```python
class Solution:
    def subsets(self, nums: list[int]) -> list[list[int]]:
        ans = []
        path = []

        def backtrack(start: int) -> None:
            ans.append(path[:])

            for i in range(start, len(nums)):
                path.append(nums[i])
                backtrack(i + 1)
                path.pop()

        backtrack(0)
        return ans
```

### 复杂度

一共有 `2^n` 个子集。

每次加入结果时要复制当前路径：

```python
path[:]
```

复制成本最多是 `O(n)`。

所以时间复杂度是：

```
O(n * 2^n)
```

返回结果占用：

```
O(n * 2^n)
```

递归路径 `path` 和递归调用栈最多深度为 `n`，如果不算返回结果，额外空间是：

```
O(n)
```

### 为什么它最适合面试？

回溯法最适合面试，因为它不是只解决这一题，而是后续一整类题的通用模板。

Q78 是回溯入门题。掌握这题之后，可以自然过渡到：

- Q90 Subsets II：子集 + 去重
- Q77 Combinations：限制 path 长度
- Q39 Combination Sum：可重复选择
- Q40 Combination Sum II：不可重复选择 + 去重
- Q46 Permutations：排列问题
- Q47 Permutations II：排列 + 去重

面试里可以这样表达：

1. 这题要返回所有子集，所以至少有 `2^n` 个结果
2. 我用回溯枚举所有选择路径
3. `path` 表示当前子集，`start` 控制下一次从哪里开始选
4. 每次进入递归时，当前 `path` 就是一个合法子集，所以先加入答案
5. 循环中选择一个数，递归，再撤销选择

这就是经典的：

```
做选择 -> 递归 -> 撤销选择
```

---

## 最终推荐

面试最推荐写 **解法 3：回溯法**。

它的代码不长，但能体现回溯的核心框架：

```python
path.append(nums[i])
backtrack(i + 1)
path.pop()
```

这套框架后面可以直接迁移到组合、排列、去重、字符串切割等题型。
