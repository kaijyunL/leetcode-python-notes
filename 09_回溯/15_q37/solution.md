# LeetCode 37 - 解数独（Sudoku Solver）

## 题目

给你一个 `9 x 9` 的数独棋盘 `board`，空格用 `'.'` 表示。

请你原地填充空格，使整个棋盘满足数独规则：

1. 每一行数字 `1-9` 不能重复
2. 每一列数字 `1-9` 不能重复
3. 每个 `3 x 3` 宫内数字 `1-9` 不能重复

题目保证输入一定有解，并且解唯一。

---

## 这题本质是什么

这题本质上不是“推理题”，而是一个很标准的：

```text
回溯 + 约束检查
```

我们要做的事情可以理解成：

```text
找到一个空格
尝试填 1 到 9
如果当前填法不冲突，就继续填下一个空格
如果后面走不通，就撤销这一步，换一个数字继续试
```

所以这题和第 36 题《有效的数独》的关系非常直接：

```text
Q36: 判断一个局面是否合法
Q37: 在合法性判断的基础上，用回溯把空位补出来
```

---

## 解法一览

| 解法 | 思路 | 合法性判断 | 复杂度（按空格数 E 理解） | 是否推荐面试 |
|---|---|---|---|---|
| 1. 暴力回溯 + 每次扫描 | 每填一个数字，都重新扫描行、列、宫 | `O(9 + 9 + 9)` | 最坏约 `O(9^E)` | 适合理解 |
| 2. 回溯 + 行列宫状态表 | 用布尔表记录哪些数字已经用过 | `O(1)` | 最坏约 `O(9^E)`，但常数更小 | **最适合面试 ✅** |
| 3. 位运算 + MRV | 用 bitmask 维护状态，并优先选候选最少的空格 | `O(1)` | 实战里最快 | 进阶最优 |

> 严格说，数独棋盘大小固定为 `9 x 9`，所以 LeetCode 上也可以说是常数规模问题。
> 但在面试中，更推荐按“空格数 `E`”来分析搜索复杂度，这样更能体现你理解了回溯树。

---

## 核心理解

### 1. 为什么这是回溯题

因为我们需要不断做这样的动作：

```text
做选择 -> 递归 -> 撤销选择
```

放到这题里就是：

```text
填一个数字 -> 继续填后面的空格 -> 如果失败就擦掉
```

---

### 2. 为什么递归函数通常返回 `bool`

这题题目保证：

```text
解唯一
```

所以一旦我们找到一个完整解，就可以立刻一路返回 `True`，不需要继续搜别的分支。

这和“求所有解”的回溯题不一样。

这里更像：

```python
if backtrack(...):
    return True
```

找到就停，效率更高。

---

### 3. 怎么定位一个格子属于哪个 3x3 宫

对于格子 `(row, col)`：

```python
box = (row // 3) * 3 + col // 3
```

宫编号会是 `0` 到 `8`：

```text
0 1 2
3 4 5
6 7 8
```

比如：

- `(0, 0)` 在第 `0` 宫
- `(1, 7)` 在第 `2` 宫
- `(5, 4)` 在第 `4` 宫
- `(8, 8)` 在第 `8` 宫

这个公式一定要熟。

## 解法 1：暴力回溯 + 每次扫描判断

对应文件：

```text
09_回溯/15_q37/solution_1_bruteforce_scan.py
```

### 思路

先把所有空格位置收集起来：

```python
empties = [(r, c), ...]
```

然后按顺序去填这些空格。

对于当前空格 `(r, c)`：

1. 依次尝试字符 `'1'` 到 `'9'`
2. 每次都扫描：
   - 第 `r` 行
   - 第 `c` 列
   - 所在 `3x3` 宫
3. 如果不冲突，就填进去，递归处理下一个空格
4. 如果后面失败，就恢复成 `'.'`

### 为什么叫“暴力”

因为它没有额外维护状态。

每次要判断某个数字能不能放，都重新把相关区域扫一遍。

也就是说，很多信息被反复计算了。

### 递归框架

```python
def backtrack(index):
    if index == len(empties):
        return True

    row, col = empties[index]

    for ch in "123456789":
        if is_valid(row, col, ch):
            board[row][col] = ch
            if backtrack(index + 1):
                return True
            board[row][col] = "."

    return False
```

### 优点

1. 最直观
2. 最容易从零想出来
3. 很适合理解回溯到底在干什么

### 缺点

1. 合法性判断成本高
2. 重复扫描很多次
3. 面试里如果直接停在这一步，通常还不够好

### 复杂度

- 时间复杂度：最坏约 `O(9^E)`，每层最多试 9 个数字，并且每次判断还要扫描行列宫
- 空间复杂度：`O(E)`，主要是递归栈和空格列表

## 解法 2：回溯 + 行列宫状态表

对应文件：

```text
09_回溯/15_q37/solution_2_backtracking_state.py
```

## 这是最适合面试的方法

如果面试里让我写第 37 题，我最推荐优先写这个版本。

原因很简单：

1. 它比暴力法明显更优
2. 它的核心优化非常自然，容易讲清楚
3. 它没有位运算门槛，现场出错率更低
4. 代码结构清晰，面试官容易跟上你的思路

一句话总结就是：

```text
性能足够好，逻辑足够清楚，手写足够稳
```

---

### 核心优化点

我们不再每次都去扫描整行、整列、整宫。

而是在一开始就维护 3 份状态：

```python
rows[row][num]
cols[col][num]
boxes[box][num]
```

它们的含义是：

- `rows[row][num] = True` 表示数字 `num` 已经在这一行出现过
- `cols[col][num] = True` 表示数字 `num` 已经在这一列出现过
- `boxes[box][num] = True` 表示数字 `num` 已经在这个宫出现过

这样判断能不能填 `num` 就只需要：

```python
if rows[row][num] or cols[col][num] or boxes[box][num]:
    不能填
```

这就是 `O(1)` 判断。

---

### 为什么这个优化非常重要

暴力法的问题是：

```text
每次尝试一个数字，都要重新扫描
```

状态表做的事情是：

```text
把“是否出现过”这个信息提前记下来
以后直接查表
```

这就是典型的：

```text
空间换时间
```

---

### 详细流程

#### 第一步：预处理原始棋盘

遍历整个棋盘：

1. 如果当前位置不是空格
2. 读出这个数字 `num`
3. 算出它属于哪个宫 `box`
4. 把这三个状态表标记为已使用
5. 如果当前位置是空格，就加入 `empties`

这样递归开始前，所有已知信息都准备好了。

---

#### 第二步：递归填空格

假设现在处理 `empties[index] = (row, col)`。

我们尝试 `1` 到 `9`：

1. 如果该数字在当前行、列、宫已经出现过，跳过
2. 否则把它填进去
3. 同时更新 `rows / cols / boxes`
4. 递归处理下一个空格
5. 如果后面失败，再把棋盘和状态一起撤销

这里最关键的是：

```text
棋盘变了，状态表也要同步变
回溯撤销时，两边也都要一起还原
```

---

### 这版代码最值得背下来的骨架

```python
for num in range(1, 10):
    if rows[row][num] or cols[col][num] or boxes[box][num]:
        continue

    board[row][col] = str(num)
    rows[row][num] = True
    cols[col][num] = True
    boxes[box][num] = True

    if backtrack(index + 1):
        return True

    board[row][col] = "."
    rows[row][num] = False
    cols[col][num] = False
    boxes[box][num] = False
```

这个骨架同时体现了：

1. 剪枝
2. 做选择
3. 递归深入
4. 撤销选择

---

### 面试时怎么讲最顺

你可以按这个顺序说：

1. 我先用回溯，因为本质上是在空格里试数字
2. 暴力写法每次都要扫描行列宫，重复计算很多
3. 所以我用三个状态表分别记录行、列、宫里已经用过的数字
4. 这样判断一个数字能不能放只要 `O(1)` 查表
5. 每次放数字时同步更新状态，回溯时再撤销
6. 因为题目保证唯一解，所以我让递归返回 `bool`，找到就立刻停止

这套说法很完整，也很像“真正理解后说出来的话”。

---

### 复杂度

- 时间复杂度：最坏约 `O(9^E)`，但每次合法性检查降到了 `O(1)`
- 空间复杂度：`O(E)` 递归栈，外加固定大小的状态表 `O(1)`

## 解法 3：位运算 + MRV（候选最少优先）

对应文件：

```text
09_回溯/15_q37/solution_3_mrv_bitmask.py
```

### 这版为什么更快

它做了两层优化：

1. 用位运算记录状态，比布尔表更轻
2. 每次不随便选下一个空格，而是优先选“候选数字最少”的那个空格

第二点尤其关键，这个策略通常叫：

```text
MRV（Minimum Remaining Values）
```

也就是：

```text
剩余可选值最少优先
```

如果某个空格只剩 1 个或 2 个候选，我们优先处理它，通常能更早发现死路，剪枝效果很好。

---

### 位运算怎么表示状态

用一个 9 位二进制数表示数字 `1-9` 是否已经出现。

比如：

```text
bit 0 -> 数字 1
bit 1 -> 数字 2
...
bit 8 -> 数字 9
```

如果数字 `5` 已经用过，那么就把第 `4` 位设成 `1`。

某个位置不能用的数字集合就是：

```python
rows[row] | cols[col] | boxes[box]
```

能用的数字集合就是：

```python
FULL_MASK & ~(rows[row] | cols[col] | boxes[box])
```

其中：

```python
FULL_MASK = (1 << 9) - 1
```

---

### MRV 怎么做

在当前所有还没填的空格里，逐个计算候选集合大小：

```python
candidate_count = available_mask.bit_count()
```

选择候选数最少的空格作为下一步。

如果某个空格候选数是 `0`，说明当前分支已经死了，直接返回 `False`。

---

### 为什么它不一定是面试首选

因为它虽然快，但现场手写会更容易出错：

1. 要处理 bitmask
2. 要处理 lowbit
3. 要处理候选集合转换成数字
4. 要在回溯里正确撤销位状态

如果面试官更看重“清晰稳定”，那么方法 2 通常更划算。

如果面试官追问：

```text
还能更快吗？
```

这时候再把方法 3 讲出来，会非常加分。

---

### 复杂度

- 时间复杂度：最坏仍然是指数级搜索，但因为 MRV 和位运算，实际运行通常最快
- 空间复杂度：`O(E)` 递归栈，状态数组仍然是固定大小 `O(1)`
