# LeetCode 162. 寻找峰值

## 题目理解

给你一个数组 `nums`，找到任意一个峰值元素，并返回它的下标。

峰值元素满足：

```text
nums[i] > nums[i - 1]
nums[i] > nums[i + 1]
```

题目还规定：

```text
nums[-1] = -∞
nums[n] = -∞
```

也就是说，数组两边外侧都可以看成负无穷。

例如：

```text
nums = [1, 2, 3, 1]
```

`3` 是峰值，返回：

```text
2
```

---

## 方法一：线性扫描

### 思路

从左到右扫描，找到第一个满足：

```text
nums[i] > nums[i + 1]
```

的位置。

这个位置一定是一个峰值。

为什么？

如果前面一直在上升：

```text
1, 2, 3
```

突然在 `3 -> 1` 这里下降，那么 `3` 就是峰值。

如果一直没有下降，说明数组一路上升，最后一个元素就是峰值，因为右侧可以看成 `-∞`。

### 复杂度

时间复杂度：

```text
O(n)
```

空间复杂度：

```text
O(1)
```

## 方法二：二分查找

这是最适合面试的方法。

### 核心思路

看中点 `mid` 和右边相邻元素 `mid + 1` 的关系。

如果：

```text
nums[mid] < nums[mid + 1]
```

说明右边是上坡，右侧一定存在峰值。

所以：

```python
left = mid + 1
```

如果：

```text
nums[mid] > nums[mid + 1]
```

说明从 `mid` 到 `mid + 1` 是下坡，`mid` 自己可能就是峰值，或者左侧存在峰值。

所以：

```python
right = mid
```

注意这里是 `right = mid`，不是 `right = mid - 1`。

因为 `mid` 可能就是答案，不能排除。

### 为什么这题和前面模板略不同

前面第 35、69、34、74 题主要是在找边界：

```text
最后一个满足条件的位置
```

所以可以统一写：

```python
if check(mid):
    left = mid + 1
else:
    right = mid - 1
```

第 162 题不是普通边界题，而是：

```text
根据坡度判断哪一侧一定有峰值
```

当发现左侧有峰值可能时，`mid` 不能丢掉，所以要写：

```python
right = mid
```

### 例子

```text
nums = [1, 2, 3, 1]
```

开始：

```text
left = 0
right = 3
mid = 1
```

比较：

```text
nums[1] = 2
nums[2] = 3
```

因为：

```text
2 < 3
```

说明右边上坡，右边一定有峰值：

```python
left = mid + 1
```

此时：

```text
left = 2
right = 3
```

再比较：

```text
mid = 2
nums[2] = 3
nums[3] = 1
```

因为：

```text
3 > 1
```

说明 `mid = 2` 可能就是峰值，不能排除：

```python
right = mid
```

最后：

```text
left = right = 2
```

返回 `2`。

### 面试推荐讲法

1. 线性扫描可以做，但复杂度是 `O(n)`。
2. 观察 `nums[mid]` 和 `nums[mid + 1]`。
3. 如果 `nums[mid] < nums[mid + 1]`，说明右侧上坡，右侧一定有峰值。
4. 否则说明左侧包含 `mid` 一定有峰值。
5. 每次都能丢掉一半区间，所以复杂度是 `O(log n)`。

### 复杂度

时间复杂度：

```text
O(log n)
```

空间复杂度：

```text
O(1)
```

```text
上坡往右找，下坡保留 mid 往左找。
```
